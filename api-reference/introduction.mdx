The Skygen API is organized around REST. It uses predictable resource-oriented URLs, returns JSON, and uses standard HTTP response codes and verbs. All requests are authenticated with a Bearer JWT access token (see [Authentication](/guides/authentication)) unless the endpoint is explicitly public.

## Base URL

```
https://api.skygen.ai
```

## Authentication

Every authenticated request must include an access token in the `Authorization` header.

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

Access tokens expire after 60 minutes. Use the refresh flow to obtain a new one. See [Authentication](/guides/authentication) for the full flow, including OAuth, email verification, password reset, and device KID rotation.

## Content type

All request bodies must be JSON with `Content-Type: application/json`, except the login endpoint, which uses `application/x-www-form-urlencoded` (OAuth2 password flow).

## Pagination

List endpoints support `limit` and `offset` query parameters. Maximum page size is typically 100. Check the per-endpoint reference for exact bounds.

```http
GET /sessions?limit=20&offset=40
```

## Versioning

Skygen maintains a single stable API. Breaking changes will be announced via the [changelog](https://skygen.ai/changelog) and gated behind a `v2` path prefix (see `/billing/v2` for the first example).

## Errors

See the [Errors reference](/errors) for status codes, response format, and retry strategy.

## Real-time events

Some flows rely on Socket.IO for real-time updates (message streaming, device status, agent progress). Connect to the `/user` namespace with your access token. See [Agents](/concepts/agents) for subscription patterns.

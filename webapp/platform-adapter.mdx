# Platform Adapter

The platform adapter is a dependency injection layer that abstracts platform-specific capabilities (storage, desktop automation, OAuth, analytics) behind a common interface. It is the single point where the web browser and Tauri desktop differ.

---

## Interface (`types/platform.ts`)

```typescript
export interface IPlatformAdapter {
  // Platform identification
  platform: 'macos' | 'windows' | 'web';

  // Capabilities — gating flags for platform-specific features
  capabilities: {
    hasDesktopAutomation: boolean;   // True on Tauri desktop only
    hasSecureStorage: boolean;        // True on Tauri (OS keychain)
    hasOverlay: boolean;
    canOpenOAuthWindow: boolean;
  };

  // Token and preference storage
  storage: {
    get(key: string): Promise<string | null>;
    set(key: string, value: string): Promise<void>;
    remove(key: string): Promise<void>;
  };

  // Desktop-only (optional)
  desktop?: {
    startDeviceWebSocket(): Promise<void>;
    stopDeviceWebSocket(): Promise<void>;
    checkDeviceWebSocketStatus(): Promise<boolean>;
    getDeviceId(): Promise<string | null>;
  };

  // Auto-updater (optional, desktop only)
  updater?: IUpdaterService;

  // Analytics (optional)
  analytics?: {
    identify(userId: string): void;
    track(event: string, properties?: Record<string, unknown>): void;
  };

  // OAuth window opening
  openOAuthWindow(url: string): Promise<void>;
}
```

The adapter is passed to `bootstrap()` at app startup and stored globally. All code that needs platform capabilities calls `getPlatform()` from `init/bootstrap.ts` — there is no React context or prop-drilling.

## Web Implementation (`WebApp/src/platform/webPlatformAdapter.ts`)

<Note>
`webPlatformAdapter.ts` imports `IPlatformAdapter` from `../../ui-shared/src/types/platform` (a relative path that resolves to `WebApp/ui-shared/src/`). `WebApp/ui-shared/` is a **separate git submodule** from `WebApp/ui-shared-tauri/` — the one containing all the active shared UI code. The `ui-shared` submodule contains only the platform type definitions, while `ui-shared-tauri` contains the full component library. Both are wired together via TypeScript path aliases in `tsconfig.json` (`"ui-shared": ["./ui-shared/src"]`).
</Note>

```typescript
export const webPlatformAdapter: IPlatformAdapter = {
  platform: 'web',

  capabilities: {
    hasDesktopAutomation: false,   // No device WebSocket
    hasSecureStorage: false,        // Uses localStorage, not OS keychain
    hasOverlay: false,
    canOpenOAuthWindow: true,
  },

  storage: {
    get:    async (key) => localStorage.getItem(key),
    set:    async (key, value) => localStorage.setItem(key, value),
    remove: async (key) => localStorage.removeItem(key),
  },

  // OAuth: full-page redirect (same-tab)
  openOAuthWindow: async (url: string) => {
    window.location.href = url;
  },

  desktop: undefined,   // No device WebSocket on web

  analytics: {
    identify(userId: string) {
      window.dataLayer = window.dataLayer || [];
      window.dataLayer.push({ event: 'identify', user_id: userId });
    },
    track(event: string, properties?: Record<string, unknown>) {
      window.dataLayer = window.dataLayer || [];
      // Remap internal event names to GTM event names
      const eventName = event === 'signup' ? 'registration_completed' : event;
      window.dataLayer.push({ event: eventName, ...properties });
    },
  },
};
```

### Storage keys used by the application

| Key | Set by | Purpose |
|---|---|---|
| `auth_token` | `authApi.login`, `authApi.register`, `authApi.refreshToken` | JWT Bearer token |
| `refresh_token` | `authApi.login`, `authApi.refreshToken` | JWT refresh token |
| `user_data` | Removed on logout | Legacy user cache (currently only removed, not written) |
| `skygen_onboarding_done` | `billingSlice.markOnboardingComplete()` | Onboarding completion flag |
| `skygen_theme` | `uiSlice.setTheme()` | UI theme preference |
| `skygen_sidebar_open` | `uiSlice.setSidebarOpen()` | Sidebar state |
| `prompt_event_id` | Various | Prompt event tracking (removed on logout) |
| `skygen_active_integrations` | MCP integrations | Active integration set (removed on logout) |

## Desktop Implementation

The Tauri desktop app implements the full `IPlatformAdapter` with:

| Difference | Web | Desktop (Tauri) |
|---|---|---|
| `platform` | `'web'` | `'macos'` or `'windows'` |
| `capabilities.hasDesktopAutomation` | `false` | `true` |
| `capabilities.hasSecureStorage` | `false` | `true` |
| `storage` | `localStorage` | OS keychain via Tauri plugin |
| `desktop` | `undefined` | Device WebSocket management |
| `updater` | `undefined` | `IUpdaterService` implementation |
| `openOAuthWindow` | Page redirect | Opens system browser / Tauri window |

## Capability Gating Pattern

All platform-specific code is gated on capability flags — never on `platform === 'web'` string checks:

```typescript
// CORRECT — capability gating
if (platform.capabilities.hasDesktopAutomation && platform.desktop) {
  await platform.desktop.startDeviceWebSocket();
}

// WRONG — string check
if (platform.platform !== 'web') { ... }
```

This pattern appears in `authSlice.login()`, `authSlice.signup()`, `authSlice.restoreSession()`, and `authSlice.logout()`.

## IUpdaterService (`types/platform.ts`)

The desktop auto-updater interface for Tauri apps:

```typescript
export interface IUpdaterService {
  checkForUpdates(): Promise<UpdateInfo | null>;
  installUpdate(): Promise<void>;
  getAppVersion(): Promise<string>;
  isDevMode(): Promise<boolean>;
  onDownloadProgress(
    callback: (progress: { percentage: number; downloaded: number; total: number }) => void
  ): () => void;    // Returns unsubscribe function
  onDownloadVerifying(callback: () => void): () => void;
}
```

Not implemented on web — `platform.updater` is `undefined`. The `UpdateRequiredError` response from the backend (`types/messages.ts:36`) is handled by `sessionApi.sendMessage` and surfaces an update prompt in the UI regardless of whether `updater` is available.

## Analytics Integration

The web adapter pushes to `window.dataLayer` for Google Tag Manager. Events tracked by the application:

| Internal event | GTM event name | Pushed from |
|---|---|---|
| `identify` | `identify` | `authSlice.login`, `authSlice.restoreSession` |
| `login` | `login` | `authSlice.login` |
| `signup` | `registration_completed` | `authSlice.signup` |
| `logout` | `logout` | `authSlice.logout` |
| `task_created` | `task_created` | `messageSlice.sendMessage` (first message) |
| `message_sent` | `message_sent` | `messageSlice.sendMessage` (every message) |
| `task_completed` | `task_completed` | `useChatWebSocket` (terminal new_message) |
| `paywall_viewed_tok` | `paywall_viewed_tok` | `messageSlice.sendMessage` (free limit reached) |

All analytics calls are wrapped in `try/catch` — analytics must never break UX.

## Bootstrap Configuration

The full bootstrap config type:

```typescript
export interface BootstrapConfig {
  apiConfig: ApiConfig;       // BACKEND_URL, WS_URL, STRIPE_PUBLISHABLE_KEY
  platform: IPlatformAdapter;
  basePath?: string;          // React Router basename. Default: '/'
                              // WebApp sets '/app' (served at skygen.ai/app)
}
```

`basePath` is used both by React Router's `basename` prop and for constructing absolute same-origin URLs (e.g. OAuth redirect URIs).

## Adding a New Platform

To add a new platform (e.g. mobile):

1. Implement `IPlatformAdapter` for the platform.
2. Set `platform` to the appropriate value (or extend the union type in `types/platform.ts`).
3. Implement `storage` using the platform's native secure storage.
4. Set `capabilities` flags appropriately.
5. Pass the adapter to `bootstrap()` at app startup.

No other code changes are required — all capability-gated code paths check `capabilities` flags at runtime.

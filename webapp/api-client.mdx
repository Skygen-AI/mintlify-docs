# API Client

The frontend API layer is split into two clients: an HTTP client for REST calls and a Socket.IO client for real-time events. Both live in `ui-shared-tauri/src/api/`.

---

## HTTP Client (`api/http/httpClient.ts`)

A thin, typed wrapper around the browser `fetch` API.

### Usage

```typescript
import { httpClient } from './api/http/httpClient';

// All methods are typed generics:
const session = await httpClient.get<Session>('/sessions/abc123');
const created = await httpClient.post<Session>('/sessions', { title: 'New chat' });
await httpClient.patch('/sessions/abc123', { title: 'Updated' });
await httpClient.delete('/sessions/abc123');
```

The base URL is read from `getBackendUrl()` which returns the value passed to `bootstrap()`. Never hardcode URLs — they differ between dev and prod.

### Auth header injection

```typescript
// httpClient.ts — called on every non-GET request
async function getAuthHeaders(): Promise<Record<string, string>> {
  const token = await getPlatform().storage.get('auth_token');
  return token
    ? { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` }
    : { 'Content-Type': 'application/json' };
}
```

`credentials: 'include'` is set on every request, so the `skygen_session` cookie is always sent for browser clients regardless of whether a Bearer token is present.

### Error handling

Non-2xx responses throw `HttpClientError`:

```typescript
export class HttpClientError extends Error {
  status: number;      // HTTP status code (0 for network errors)
  statusText: string;
  data?: unknown;      // Raw parsed error response body
}
```

Error messages are extracted by `extractErrorMessage()` which handles:
- `string` detail
- `object` detail with `message` field (billing errors, device errors)
- `array` detail (Pydantic validation errors — joined with `;`)
- Nested `{ error: string }` shape

### Network error detection

`detectNetworkError()` maps `TypeError` (CORS, connection refused, mixed content) to human-readable messages. This fires before `HttpClientError` is thrown.

### Raw fetch usage

Three API modules bypass `httpClient` and call `fetch()` directly:

| API | Reason |
|---|---|
| `authApi.login` | `application/x-www-form-urlencoded` content type |
| `authApi.getCurrentUser`, `authApi.register`, etc. | Fine-grained error handling (401 vs 400 branching) |
| `sessionApi.sendMessage` | Status-based branching on 402/403/409/426 — `HttpClientError` would lose typed discriminators |
| `sessionApi.getMessages`, `getTaskStatus`, `getPlan` | Require `AbortController` timeout (not yet in `httpClient`) |

The `fetchWithTimeout` helper in `sessionApi.ts` wraps raw `fetch` with a 30-second abort controller.

### Mock mode

```typescript
const USE_MOCKS = false;  // flip to true for offline dev
```

When `true`, all requests return `{}` immediately and log to console. Defined independently in `httpClient.ts`, `authApi.ts`, and `billingApi.ts`.

## Socket.IO Client (`api/websocket/wsClient.ts`)

A **singleton** Socket.IO client that connects to the `/user` namespace. All React code subscribes to it via `wsClient.on()` / `wsClient.off()` — never via the raw Socket.IO socket.

### Connection

```typescript
import { wsClient } from './api/websocket/wsClient';

// Connect (called by authSlice after login/restoreSession)
await wsClient.connect(userId, token);

// Set token refresher so reconnects get a fresh token
wsClient.setTokenRefresher(() => platform.storage.get('auth_token'));

// Disconnect (called by authSlice on logout)
wsClient.disconnect();

// Check status
wsClient.isConnected(); // boolean
```

### Auth flow

Socket.IO's `auth` option is a function (not a static value), so it is called on every connection attempt including reconnects:

```typescript
this.socket = io(`${httpUrl}/user`, {
  auth: async (cb) => {
    let currentToken = this.token;
    if (this.tokenRefresher) {
      const fresh = await Promise.race([
        this.tokenRefresher(),
        new Promise<null>((_, rej) => setTimeout(() => rej(new Error('timeout')), 5000))
      ]);
      if (fresh) currentToken = fresh;
    }
    cb({ token: currentToken, user_id: this.userId });
  },
  withCredentials: true,    // Sends skygen_session cookie cross-origin
  transports: ['websocket'], // No polling — prevents session mismatch behind ALB
  reconnectionDelay: 1000,
  reconnectionDelayMax: 30000,
  reconnectionAttempts: Infinity,
});
```

### Subscribing to events

```typescript
// Subscribe (deduplicated — same function reference will not be added twice)
wsClient.on('new_message', (params: NewMessageParams) => {
  // params is typed based on MethodHandlersMap
});

// Unsubscribe specific handler
wsClient.off('new_message', handler);

// Unsubscribe all handlers for an event
wsClient.off('new_message');
```

**Type safety:** For events defined in `MethodHandlersMap` (`types/websocket.ts`), the payload is strongly typed. Synthetic events (`reconnected`, `device_status`, etc.) fall back to `unknown`.

### Durable delivery and deduplication

The backend uses at-least-once durable delivery for important events. After a reconnect, the server replays events that were not ACK'd. The client deduplicates these via a LRU set of 200 `notification_id` values:

```typescript
private dispatchDurable(notificationId: unknown, emit: () => void): void {
  const id = typeof notificationId === 'string' ? notificationId : undefined;
  if (id && this.seenNotificationIds.has(id)) {
    this.sendNotificationAck(id);  // ACK so server stops replaying
    return;                         // Skip emit
  }
  emit();
  if (id) this.markNotificationSeen(id);
  this.sendNotificationAck(id);
}
```

**Events using durable dispatch:** `agent_response`, `display_message`, `confirmation_request`, `task_progress`, `device_status_update`, `admin_notification`, `agent_status_update`, `agent_error`, `cloud_pc_status_changed`, `mcp_connection_completed`.

**Ephemeral events (no ACK):** `cloud_pc_control_changed`, `cloud_pc_provisioned`, `mcp_connection_failed/timeout/expired`, all cross-client session/task sync events.

### Sending events

```typescript
// Send a confirmation response
wsClient.sendSafetyConfirmationResponse(
  confirmationId,
  approved,
  message?,
  onAck?       // Optional Socket.IO ack callback
);
```

### Tab visibility handling

When the page becomes visible after being hidden, the client checks if the socket is disconnected and Socket.IO's own reconnection is not already in progress. If both conditions are met, it forces a reconnect. This covers the case where a laptop wakes from sleep after the socket connection timed out.

### `agent_response` normalisation

The `agent_response` event is the legacy protocol. The client normalises it to a `WsChatMessage` and emits it as `new_message` to subscribers, plus a `task_progress` event derived from the `status` field. The new `display_message` event is the preferred path.

## `sendMessage` Response Handling

`sessionApi.sendMessage()` returns a `SendMessageResult` — it never throws. All error branches are encoded in the result:

```typescript
interface SendMessageResult {
  success: boolean;
  data?: Message;                   // Present on success
  error?: 'device_busy'
        | 'device_unreachable'
        | 'cloud_pc_conflict'
        | 'invalid_otp'
        | 'network'
        | 'update_required'
        | 'payment_required'
        | 'unknown';
  errorMessage?: string;
  deviceBusyInfo?: DeviceBusyError;
  deviceUnreachableInfo?: DeviceUnreachableError;
  cloudPcConflictInfo?: CloudPcSessionConflictError;
  updateRequiredInfo?: UpdateRequiredError;
  paymentRequiredInfo?: PaymentRequiredError;
}
```

HTTP status to error mapping:

| HTTP status | error field | Condition |
|---|---|---|
| 200 | — | `success: true` |
| 402 | `payment_required` | Any billing block |
| 403 | `invalid_otp` | OTP validation failed |
| 409 | `device_busy` | `isDeviceBusyError(detail)` — device locked by another session |
| 409 | `device_unreachable` | `isDeviceUnreachableError(detail)` — device offline or not responding |
| 409 | `cloud_pc_conflict` | `isCloudPcSessionConflict(detail)` — Cloud PC bound to another session |
| 426 | `update_required` | `isUpdateRequiredError(detail)` |
| network error | `network` | `TypeError` from fetch |
| other | `unknown` | All other cases |

## Overview

In this tutorial you will build a Skygen automation that:
1. Reads your Gmail inbox for the past 24 hours
2. Fetches a news summary from a news API
3. Compiles a structured digest
4. Posts it to a Slack channel

You will use Gmail and Slack MCP integrations via Composio, a scheduled trigger, and the Skygen REST API. Total setup time: approximately 20 minutes.

---

## Prerequisites

- A Skygen account with an active subscription (Standard tier or above)
- Gmail connected in Skygen (see [Connecting Integrations](/guides/connecting-integrations))
- Slack connected in Skygen
- A Slack channel to post the digest to (e.g., `#daily-digest`)
- Your Skygen API token

---

## Step 1: Verify integrations are connected

Before creating the automation, confirm both Gmail and Slack are connected:

```python
import httpx

BASE_URL = "https://api.skygen.ai"
TOKEN = "your-skygen-jwt-token"
headers = {"Authorization": f"Bearer {TOKEN}"}

resp = httpx.get(f"{BASE_URL}/mcp/integrations/available", headers=headers)
integrations = resp.json()

for integration in integrations["items"]:
    if integration["toolkit"] in ("gmail", "slack"):
        status = "connected" if integration["is_connected"] else "NOT connected"
        print(f"{integration['toolkit']}: {status}")
```

If either shows "NOT connected", follow the [Connecting Integrations](/guides/connecting-integrations) guide first.

## Step 3: Create a daily trigger

Now automate this by creating a trigger that runs the digest every morning:

```python
import httpx

BASE_URL = "https://api.skygen.ai"
TOKEN = "your-skygen-jwt-token"
headers = {"Authorization": f"Bearer {TOKEN}", "Content-Type": "application/json"}

trigger_payload = {
    "toolkit": "gmail",
    "trigger_name": "new_email",
    "config": {
        "interval_minutes": 1440,  # Check once per day
    },
    "action_prompt": """
Read my Gmail inbox for emails received in the last 24 hours.
Compile a structured daily digest and post it to the #daily-digest Slack channel.

Format the digest as:

**Daily Digest - {today's date}**

**Action Required (emails needing response):**
- List each email with sender and brief subject

**Key Updates:**
- Important updates from colleagues or systems

**News & Newsletters:**
- 2-3 key items from newsletter subscriptions

Keep each section to 5 bullets maximum. If a section is empty, omit it.
Post to Slack only once — do not send duplicate messages.
"""
}

resp = httpx.post(
    f"{BASE_URL}/mcp/triggers",
    headers=headers,
    json=trigger_payload,
)
trigger = resp.json()
print(f"Trigger created: {trigger['trigger_id']}")
print(f"Status: {trigger['status']}")
```

## Step 5: Customize the digest format

You can update the `action_prompt` at any time without recreating the trigger:

```python
TRIGGER_ID = "your-trigger-id"

httpx.patch(
    f"{BASE_URL}/mcp/triggers/{TRIGGER_ID}",
    headers=headers,
    json={
        "action_prompt": """
Read emails from the last 24 hours. Post to #daily-digest on Slack.

Format:
**Morning Digest {date}**

Top 3 things I need to act on today:
1. ...

What happened yesterday I should know about:
- ...

Skip the digest entirely if there are fewer than 3 unread emails.
"""
    }
)
```

## Troubleshooting

**Digest not appearing in Slack**  
Check the trigger event history for errors. The most common cause is the Slack connection being inactive. Open the Connectors panel in Skygen and verify Slack shows as connected.

**Agent asks me to connect Gmail/Slack**  
The connection may have been revoked. Disconnect and reconnect in Settings > Integrations.

**Digest sent multiple times**  
The action prompt includes "do not send duplicate messages" to prevent this, but you can also add an explicit instruction: "Before posting, check if a digest was already posted today by searching for a message starting with 'Daily Digest' in #daily-digest."

**Trigger showing as inactive**  
A Composio event refreshed and the connection was flagged as expired. Reconnect and then re-enable the trigger with `PATCH /mcp/triggers/{id}` with `{"is_active": true}`.

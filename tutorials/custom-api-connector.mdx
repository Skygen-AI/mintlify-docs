## Overview

Skygen custom connectors let you expose any HTTP API — your own backend, a SaaS API, an internal tool — as a set of tools the agent can invoke. In this tutorial you will:

1. Register an OpenAPI-based connector (auto tool discovery)
2. Register a manual REST connector (explicit tool definitions)
3. Test tool execution via the agent
4. Secure the connector with authentication

We will use a public API (JSONPlaceholder) for OpenAPI and a fictional internal CRM for manual definitions.

---

## Prerequisites

- Skygen account with Standard plan
- Python 3.9+ with `httpx`
- Your Skygen API token

## Part 1: OpenAPI connector (auto-discovery)

### Step 1.1: Register the connector

OpenAPI connectors introspect a spec URL and automatically generate tool definitions for every endpoint.

```python
import httpx

BASE_URL = "https://api.skygen.ai"
TOKEN = "your-skygen-jwt-token"
headers = {
    "Authorization": f"Bearer {TOKEN}",
    "Content-Type": "application/json",
}

# Register a connector using JSONPlaceholder's OpenAPI spec
connector_payload = {
    "type": "openapi",
    "name": "JSONPlaceholder API",
    "description": "A fake REST API for testing. Provides posts, comments, users, todos.",
    "config": {
        "base_url": "https://jsonplaceholder.typicode.com",
        "spec_url": "https://jsonplaceholder.typicode.com",  # or actual OpenAPI JSON URL
        "auth_type": "none",
    },
}

resp = httpx.post(f"{BASE_URL}/connectors", headers=headers, json=connector_payload)
connector = resp.json()
CONNECTOR_ID = connector["id"]
print(f"Connector created: {CONNECTOR_ID}")
print(f"Slug: {connector['slug']}")
print(f"Status: {connector['status']}")
```

### Step 1.2: Add an icon (optional)

```python
import requests

# Step A: Get presigned URL
icon_resp = httpx.post(
    f"{BASE_URL}/connectors/icon/presigned",
    headers=headers,
    json={"content_type": "image/png", "file_extension": "png"},
)
presigned = icon_resp.json()

# Step B: Upload icon to S3
with open("my-api-icon.png", "rb") as f:
    requests.post(
        presigned["upload_url"],
        data=presigned["fields"],
        files={"file": f},
    )

# Step C: Link icon to connector
httpx.patch(
    f"{BASE_URL}/connectors/{CONNECTOR_ID}/icon",
    headers=headers,
    json={"icon_key": presigned["icon_key"]},
)
print("Icon uploaded")
```

### Step 1.3: Use it in a task

```python
# Create a session and ask the agent to use the connector
session = httpx.post(
    f"{BASE_URL}/chat/sessions",
    headers=headers,
    json={"title": "JSONPlaceholder test"},
).json()

SESSION_ID = session["id"]

httpx.post(
    f"{BASE_URL}/chat/sessions/{SESSION_ID}/messages",
    headers=headers,
    json={
        "content": "Use the JSONPlaceholder API connector to fetch the first 5 posts "
                   "and summarize their titles.",
    },
)
print(f"Task running at: https://app.skygen.ai/chat/{SESSION_ID}")
```

## Part 2: Manual REST connector

For APIs without OpenAPI specs, define tools explicitly.

### Step 2.1: Define the connector with tools

```python
crm_connector = {
    "type": "rest",
    "name": "Internal CRM",
    "description": "Access customer records, create tickets, and manage accounts.",
    "config": {
        "base_url": "https://crm.internal.example.com/api/v1",
        "auth_type": "bearer",
        "auth_token": "crm-internal-api-key-xyz",
    },
    "tools": [
        {
            "name": "get_customer",
            "description": "Retrieve a customer record by their email address.",
            "method": "GET",
            "path": "/customers",
            "parameters": {
                "email": {
                    "type": "string",
                    "description": "Customer email address",
                    "required": True,
                    "location": "query",
                }
            },
        },
        {
            "name": "create_support_ticket",
            "description": "Create a new support ticket for a customer.",
            "method": "POST",
            "path": "/tickets",
            "body_schema": {
                "customer_id": {
                    "type": "string",
                    "description": "Customer UUID from get_customer",
                    "required": True,
                },
                "subject": {
                    "type": "string",
                    "description": "Ticket subject line",
                    "required": True,
                },
                "priority": {
                    "type": "string",
                    "enum": ["low", "normal", "high", "urgent"],
                    "description": "Ticket priority",
                    "required": False,
                },
                "body": {
                    "type": "string",
                    "description": "Full ticket description",
                    "required": True,
                },
            },
        },
        {
            "name": "list_customer_tickets",
            "description": "List open support tickets for a customer.",
            "method": "GET",
            "path": "/tickets",
            "parameters": {
                "customer_id": {
                    "type": "string",
                    "description": "Customer UUID",
                    "required": True,
                    "location": "query",
                },
                "status": {
                    "type": "string",
                    "enum": ["open", "in_progress", "resolved"],
                    "required": False,
                    "location": "query",
                },
            },
        },
    ],
}

resp = httpx.post(f"{BASE_URL}/connectors", headers=headers, json=crm_connector)
crm = resp.json()
print(f"CRM connector created: {crm['id']}")
print(f"Tools: {len(crm.get('tools', []))}")
```

### Step 2.2: Test the connector

```python
session = httpx.post(
    f"{BASE_URL}/chat/sessions",
    headers=headers,
    json={"title": "CRM automation test"},
).json()

SESSION_ID = session["id"]

httpx.post(
    f"{BASE_URL}/chat/sessions/{SESSION_ID}/messages",
    headers=headers,
    json={
        "content": "Look up the customer with email alice@example.com in the CRM. "
                   "If they have any open tickets, summarize them. "
                   "If they have no tickets, create a new one with subject "
                   "'Welcome check-in' and priority 'low'.",
    },
)
print(f"CRM task running at: https://app.skygen.ai/chat/{SESSION_ID}")
```

## Part 3: Authentication patterns

### Bearer token

```python
{
    "auth_type": "bearer",
    "auth_token": "your-api-token"
}
# Sends: Authorization: Bearer your-api-token
```

### API key in custom header

```python
{
    "auth_type": "api_key",
    "api_key": "your-api-key",
    "api_key_header": "X-API-Key"
}
# Sends: X-API-Key: your-api-key
```

### Basic authentication

```python
{
    "auth_type": "basic",
    "username": "api-user",
    "password": "api-password"
}
# Sends: Authorization: Basic <base64>
```

### No authentication (public API)

```python
{
    "auth_type": "none"
}
```

## Part 4: Managing connectors

### List connectors

```python
resp = httpx.get(f"{BASE_URL}/connectors", headers=headers)
for c in resp.json()["items"]:
    print(f"- {c['name']} ({c['type']}) — {c['status']} — slug: {c['slug']}")
```

### Update a connector

```python
httpx.put(
    f"{BASE_URL}/connectors/{CONNECTOR_ID}",
    headers=headers,
    json={
        "description": "Updated description",
        "config": {
            "base_url": "https://crm.internal.example.com/api/v2",  # Upgrade to v2
            "auth_type": "bearer",
            "auth_token": "new-api-token",
        },
    },
)
```

### Deactivate a connector

```python
httpx.patch(
    f"{BASE_URL}/connectors/{CONNECTOR_ID}/status",
    headers=headers,
    json={"status": "inactive"},
)
```

### Delete a connector

```python
httpx.delete(f"{BASE_URL}/connectors/{CONNECTOR_ID}", headers=headers)
```

## Part 5: Security considerations

**All connector configs are encrypted at rest** using AES with the `MCP_CRYPTO_KEY`. Secrets in API responses are always shown as `[REDACTED]`.

**SSRF protection:** Skygen validates all `base_url` and `spec_url` values against a blocklist before saving. Requests to localhost, link-local addresses (169.254.0.0/16), and private RFC 1918 ranges are blocked.

**Per-user isolation:** Connectors belong to the user who created them. Other users on the platform cannot access or see your connectors.

**Agent access:** The agent can only use connectors belonging to the authenticated user for the current session. It cannot access other users' connectors.

## Troubleshooting

**Connector created but agent doesn't use it**  
The agent discovers tools by their `description` field. Write clear, specific descriptions that match the tasks you want to automate. Include what the tool returns, not just what it accepts.

**401 errors when agent calls tool**  
Your `auth_token` may have expired. Update the connector config with a fresh token via `PUT /connectors/{id}`.

**500 errors from your API**  
Check the action history (`GET /action-history?session_id={id}`) to see the exact tool call parameters the agent sent. The agent may be sending unexpected parameter types — add explicit type and format constraints to your tool definitions.

**SSRF block on internal URL**  
Connectors cannot target internal IP ranges from Skygen's servers. For internal APIs, expose them via a public endpoint (behind authentication) or use a self-hosted Skygen deployment.

# Agent Prompt Files

The `Agent/prompts/` directory contains a library of task-description prompts used during local development and CUA exploration testing.

---

## `prompts/prod_apps/`

**Path:** `Agent/prompts/prod_apps/`

Each file in this directory is a Markdown task description for a specific web application. They are consumed by `run_local_wrapper.py` when running the agent locally with `--app <name>` or `--explore` mode.

### Purpose

These files serve as exploration tasks for the `CUA_OrchestratorAgent` in development and QA environments. When loaded, the file content is used as the agent's initial user prompt (instead of a live user message). The CUA orchestrator then navigates the application UI, exercises all documented workflows, and reports findings.

### Usage

```bash
# List available apps
python run_local_wrapper.py --list-apps

# Run exploration against an app
python run_local_wrapper.py --agent cua --app gmail

# Combine with --explore flag for full exploration mode
python run_local_wrapper.py --explore --app slack
```

The discovery mechanism (`_discover_apps()` in `run_local_wrapper.py`) scans `prompts/prod_apps/*.md` at startup and registers all found files as valid `--app` values. File names without the `.md` extension become the app key.

## File Format

Each prompt file follows a consistent three-section Markdown structure:

```markdown
## Task: <Application Name>

<One-line description of what the application is and its primary function.>

### Key Areas to Explore

- **Area name** — what to look for, sub-elements, notable UI patterns

### Expected Workflows

- Workflow description: steps → observable outcome

### Notes

- Edge cases, popups to dismiss, app-specific quirks
```

---

## Relationship to System Prompts

These files are **not** system prompts. They are the initial **user turn** injected into the agent's conversation history at task start. The actual agent persona and behavioral instructions come from the specialist's `SYSTEM_PROMPT` class attribute and the `PromptBuilder` pipeline.

The exploration task prompt replaces the `DEFAULT_PROMPTS` dict entry for the `"cua"` agent key in `run_local_wrapper.py` when `--app` is specified.

---

## Adding a New App Prompt

1. Create `Agent/prompts/prod_apps/<appname>.md` following the three-section format above.
2. Run `python run_local_wrapper.py --list-apps` to confirm the file is discovered.
3. Run `python run_local_wrapper.py --explore --app <appname>` to validate the task prompt with the CUA orchestrator locally.

No code changes are required — the file discovery is fully automatic.

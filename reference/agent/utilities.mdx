# Agent Utilities

The `Agent/Utilities/` directory contains all shared infrastructure used across agent classes. These modules are not agent-specific — they are composable building blocks that `BaseAgent` and specialist agents assemble at construction time.

---

## ModelRouter

**File:** `Agent/Utilities/ModelRouter.py`

See the full [LLM Routing Reference](/reference/agent/llm-routing) for the complete API.

`ModelRouter` is a thin policy layer over `ModelCatalog` and `CircuitBreakerRegistry`. The public entry point is the module-level function `choose_model_route()`, which calls the singleton `ModelRouter` instance's `Choose()` method.

```python
from Utilities.ModelRouter import choose_model_route, RoutingParams

decision = choose_model_route(
    payload={"messages": [...]},
    routing_params=RoutingParams(requires_heavy_reasoning=True),
    config=agent_config,
    agent_name="PlannerAgent",
)
# decision.model, decision.provider, decision.tier, decision.reason
```

## LLM_Providers

**Directory:** `Agent/Utilities/LLM_Providers/`

### ModelRegistry

**File:** `Agent/Utilities/LLM_Providers/ModelRegistry.py`

Typed registry of available LLM models. `BuildDefaultCatalog()` returns the catalog wired with all built-in models. Custom models can be added with `catalog.Register(ModelDescriptor(...))`.

See [LLM Routing — ModelCatalog](/reference/agent/llm-routing#modelcatalog) for the full API.

### ProviderFactory

**File:** `Agent/Utilities/LLM_Providers/ProviderFactory.py`

Registry-based factory that maps model names to provider implementations. The primary entry point is `llm_request()`:

```python
response = await llm_request(
    payload={"model": "google/gemini-2.5-flash", "messages": [...]},
    session=backend_session,
    session_id=session_id,
    agent_type="PlannerAgent",
    user_id=user_id,
)
```

`llm_request()` automatically:
- Applies Anthropic-style ephemeral cache control to the last 4 messages
- Selects the provider via the registry matcher chain
- Tracks API cost via `track_api_cost()`
- Supports streaming via `stream=True`

Provider registration uses `register(key, factory, matcher)`:

```python
register(
    key="my_provider",
    factory=lambda: MyProvider(),
    matcher=lambda model: model.startswith("my/"),
)
```

### CircuitBreakerRegistry

**File:** `Agent/Utilities/LLM_Providers/CircuitBreakerRegistry.py`

Tracks provider availability. The default registry is obtained via `GetDefaultRegistry()`. `ModelRouter` calls `IsOpen(provider_key)` to skip providers with known failures.

The Gemini circuit probe is registered automatically in `ProviderFactory`:

```python
_GetCircuitRegistry().Register("gemini", _probe_gemini_circuit)
```

### OpenAICompatibleProvider

**File:** `Agent/Utilities/LLM_Providers/OpenAICompatibleProvider.py`

Concrete `BaseProvider` subclass for any OpenAI-compatible API (OpenRouter, Groq, Cerebras). Configured at construction with `base_url_env` and `api_key_env` environment variable names.

### CerebrasProvider

**File:** `Agent/Utilities/LLM_Providers/CerebrasProvider.py`

Provider for Cerebras Inference API. Registered for `llama-3.3-70b` and `qwen-3-235b-a22b-instruct-2507`.

### GeminiProvider

**File:** `Agent/Utilities/LLM_Providers/GeminiProvider.py`

Provider file for Google Generative AI. Note: despite its presence, `ProviderFactory` currently routes Gemini models via OpenRouter (using `OpenAICompatibleProvider`), not directly via this provider. This file may be used for direct Gemini API access in non-default configurations.

Frozen (immutable) dataclass holding all per-session infrastructure references. Passed through every agent call.

```python
    task_id: str
```

#### `StateUpdate` ABC and subclasses

All state mutations flow through a `StateUpdate` subclass:

| Class | Description |
|---|---|

```python
@dataclass(kw_only=True)
class BaseAgentState:
    # Identity
    agent_id: Optional[str] = None

    # Evolutionary data
    messages: List[Dict[str, Any]] = field(default_factory=list)  # NOT 'history'
    action_results: List[Any] = field(default_factory=list)
    scratchpad_lines: List[str] = field(default_factory=list)

    # Orchestrator lifecycle
    lifecycle_mode: Literal["continuous", "batch", "time"] = "continuous"
    lifecycle_batch_size: int = 10
    lifecycle_timeout: int = 60
    step_num: int = 0         # NOT 'current_step'
    remaining_iter: int = 100
    is_terminated: bool = False
    is_persisted: bool = False

    # Sentinel & hibernation
    status: Literal["ACTIVE", "HIBERNATED", "PAUSED", "TERMINATED", "FAILED"] = "ACTIVE"
    # Note: there is NO 'is_paused' field — use status == "PAUSED"
    wake_reason: Optional[str] = None

    task_description: str = ""
    allowed_tools: Optional[List[str]] = None
    forbidden_tools: Optional[List[str]] = None
    forbidden_mcp_providers: Optional[List[str]] = None
```

## Hibernation

**Directory:** `Agent/Utilities/Hibernation/`

The hibernation system allows agents to sleep while waiting for external events (timers, user replies, visual conditions) without holding a worker thread.

### HibernationController

**File:** `Agent/Utilities/Hibernation/Controller.py`

High-level facade. See the [BrokerAgent documentation](/reference/agent/agents#hibernation-management) for integration details.

```python
controller = HibernationController(
    store=SentinelStore(redis_client),
    registry=HibernationStrategyRegistry(),
    waker=BrokerWaker(internal_queue),
)

# Put an agent to sleep
task = await controller.Hibernate(
    session_id=session_id,
    user_id=user_id,
    device_id=device_id,
    config={"strategy": "timer", "timeout_seconds": 60},
)

# Wake a specific task by ID
await controller.Wake(task_id, reason="User replied")

# Wake all tasks for a session (race-safe atomic HDEL)
woken_count = await controller.WakeSession(session_id)

# Wake all tasks for a user (used by MCP OAuth callback)
# Returns count of woken sessions
woken_count = await controller.WakeUserSessions(user_id)

# Clean up child sentinels when parent task completes
await controller.CancelChildrenOf(parent_session_id)
```

**Public methods:**

| Method | Signature | Description |
|--------|-----------|-------------|
| `Hibernate` | `(session_id, user_id, device_id, config) -> SentinelTask` | Put a session to sleep; registers strategy and stores task in Redis. |
| `Wake` | `(task_id, *, reason="Externally awoken", role="system") -> bool` | Force-wake a single hibernating task by ID. Returns `True` if the task existed. The `role` param controls whether the wake turn is injected as `system` (default) or `user` (for inbox-triggered wakes). |
| `WakeSession` | `(session_id, *, reason, role) -> int` | Wake all hibernating tasks for a session. Returns count of woken tasks. |
| `WakeUserSessions` | `(user_id, *, reason, role) -> int` | Wake all hibernating tasks belonging to a user. Used by push events like MCP OAuth callback. Returns count of woken sessions. |
| `CancelChildrenOf` | `(parent_session_id) -> int` | Cancel orphaned child-agent sentinels whose parent has completed; prevents phantom-wake hallucinations. |

### SentinelStore

**File:** `Agent/Utilities/Hibernation/Store.py`

Redis-backed storage for `SentinelTask` objects. Uses a Redis ZSET (`sentinel:tasks`, scored by `next_check_at`) and a HASH (`sentinel:task_data`).

#### Redis Keys

| Key | Type | Purpose |
|---|---|---|
| `sentinel:tasks` | ZSET | Step-timed task IDs, score = next wake check time |
| `sentinel:task_data` | HASH | JSON-serialized `SentinelTask` records |
| `sentinel:lock:{task_id}` | String | Distributed per-task lock (TTL 30s) |
| `sentinel:pending_user_wake:{user_id}` | String | Wake event stash for race window (TTL 300s) |

#### Write Methods

| Method | Description |
|---|---|
| `Schedule(task, next_check_at)` | HSET + ZADD — persist task and enqueue for polling |
| `Persist(task)` | HSET only — update task data without touching ZADD score |
| `Reschedule(task_id, next_check_at)` | ZADD only — reschedule without updating task data |
| `Remove(task_id)` | ZREM + HDEL — fully delete a task |
| `StashPendingUserWake(user_id, reason)` | SET with TTL 300s — store wake event that found no sleeping sessions |
| `ConsumePendingUserWake(user_id)` | Atomic GETDEL — read and delete pending wake reason |

#### Read Methods

| Method | Description |
|---|---|
| `ReadyTaskIds(before: float)` | ZRANGEBYSCORE 0..before — IDs of tasks ready to check |
| `Load(task_id)` | HGET — fetch and deserialize a single task |
| `FindBySession(session_id)` | HSCAN all — return tasks matching `session_id` |
| `FindByParentSession(parent_session_id)` | HSCAN all — return sub-agent tasks matching parent |
| `FindByUser(user_id)` | HSCAN all — return tasks matching `user_id` |

#### Locking Methods

| Method | Description |
|---|---|
| `TryClaim(task_id, ttl_seconds=30)` | SET NX EX — acquire distributed lock, returns bool |
| `Release(task_id)` | DEL — release the lock |
| `ClaimForWake(task_id)` | Atomic HDEL — prevents duplicate wakes across pods; returns True only for the winner |

The `ClaimForWake` pattern is the race prevention mechanism: all pods that receive a wake event call `ClaimForWake(task_id)` and only the one that gets `True` (deleted the hash field) proceeds to publish the wake message.

**Pending wake stash**: When a wake event fires (e.g., MCP OAuth complete) while the session is still executing an LLM turn — so `WakeUserSessions` returns `woken=0` — `StashPendingUserWake` caches the wake reason with 300s TTL. The next `Hibernate()` call for that user calls `ConsumePendingUserWake` and replays the wake immediately.

### Strategies

**Directory:** `Agent/Utilities/Hibernation/Strategies/`

#### `HibernationStrategy` (ABC)

Base class for all strategies. Subclasses implement `Check(task, store, waker)` called by the sentinel engine on each polling tick.

#### TimerStrategy

```
Name: "timer"
Config: {"timeout_seconds": float}  # Default: 300.0
```

Wakes the agent unconditionally after `timeout_seconds`. No polling between schedule and wake — fires once.

#### VisualConditionStrategy

```
Name: "visual_condition"
Config: {
    "condition": str,            # Natural language condition to check
    "interval_seconds": float,   # Default: 15.0
    "timeout_seconds": float,    # Default: 300.0
    "target_type": str,          # Default: "screen"
}
```

Re-checks a screenshot every `interval_seconds` using a vision LLM. Wakes when the condition is met or the timeout expires.

### HibernationStrategyRegistry

**File:** `Agent/Utilities/Hibernation/Registry.py`

Maps strategy names to concrete `HibernationStrategy` subclasses. `Build(config)` instantiates the matching strategy from a config dict.

### BrokerWaker

**File:** `Agent/Utilities/Hibernation/BrokerWaker.py`

Implements the `Waker` interface for `BrokerAgent`. On wake, re-enqueues the task's `original_payload` to `internal_queue` so the agent resumes via the normal `worker_loop()`.

## TerminateGuard

**Directory:** `Agent/Utilities/TerminateGuard/`

### `TerminateGuard`

**File:** `Agent/Utilities/TerminateGuard/Guard.py`

Pre-terminate validator for `communicate(message_type='result')`. Enabled in `PlannerAgent` (`TERMINATE_GUARD_ENABLED = True`). Ensures the agent has not completed prematurely before its task budget is exhausted.

```python
decision = await guard.Check(
    final_message=content,
    context=context,
    state=state,
)
# decision.accept → proceed with termination
# not decision.accept → inject rejection message into history via decision.AsReflectionMessage()
```

Constants:
- `EARLY_THRESHOLD = 0.50` — below this elapsed/TTL ratio triggers deterministic reject
- `LATE_THRESHOLD = 0.95` — at or above this ratio triggers deterministic accept

Decision zones based on elapsed time / TTL ratio:

| Zone | Condition | Action |
|---|---|---|
| `NO_TTL` | `time_limit_seconds == NO_TIME_LIMIT` or no active task | Accept unconditionally |
| `DETERMINISTIC_REJECT` | elapsed ratio < 0.50 | Reject without LLM |
| Mid-zone | 0.50 ≤ elapsed ratio < 0.95 | Ask LLM verifier |
| `DEADLINE_WINDOW` | elapsed ratio ≥ 0.95 | Accept without LLM |
| `LLM_VERIFIED_ACCEPT` | LLM verifier returned accept=true | Accept |
| `LLM_VERIFIED_REJECT` | LLM verifier returned accept=false | Reject with hint |
| `LLM_ERROR` | Verifier call failed | Fail-open (accept) |

The LLM verifier model defaults to `google/gemini-3.1-flash-lite-preview:nitro` and runs with a 30-second timeout.

### `GuardDecision`

**File:** `Agent/Utilities/TerminateGuard/Decision.py`

```python
class GuardZone(str, Enum):
    NO_TTL = "no_ttl"
    DETERMINISTIC_REJECT = "early"
    LLM_VERIFIED_ACCEPT = "llm_ok"
    LLM_VERIFIED_REJECT = "llm_no"
    DEADLINE_WINDOW = "near_end"
    LLM_ERROR = "llm_err"

@dataclass
class GuardDecision:
    accept: bool           # The ONLY field callers must inspect. NOT 'approved'.
    zone: GuardZone
    reason: str
    next_hint: str = ""    # Concrete pivot hint injected when accept=False

    def AsReflectionMessage(self) -> str:
        """Returns a plain-text system-role message to inject into agent history when rejected."""
```

**Important:** the acceptance field is `accept: bool`, not `approved: bool`.

### `PromptBlock`

**File:** `Agent/Utilities/PromptEngine/BlockBuilder.py`

Composable prompt block system using a Composite pattern. Each block renders as an XML-like tagged section.

## Communication Endpoints

**Directory:** `Agent/Utilities/CommunicationEndpoints/`

Defines the endpoint abstraction used by `CommunicateTool` and `WebEndpoint.AskText/AskChoice` for sending messages to the user and handling responses.

### `UserEndpoint` (ABC)

**File:** `Agent/Utilities/CommunicationEndpoints/UserEndpoint.py`

```python
class UserEndpoint(ABC):
    Name: ClassVar[str]           # Endpoint identifier key
    DisplayLabel: ClassVar[str]   # Human-readable label

    @abstractmethod
    async def SendInform(self, message: str, context: Any) -> None: ...
    @abstractmethod
    async def SendResult(self, message: str, context: Any) -> None: ...
    @abstractmethod
    async def AskText(self, prompt: str, *, context: Any) -> EndpointReply: ...
    @abstractmethod
    async def AskChoice(self, prompt: str, choices: list, *, context: Any) -> EndpointReply: ...

    def SupportsAttachments(self) -> bool: return False
```

### `EndpointReply`

```python
@dataclass
class EndpointReply:
    status: str                           # "received" | "cancelled" | "timeout" | "error" | "hibernate"
    value: Any = None                     # The user's reply (for "received")
    summary: str = ""
    hibernate_config: Optional[dict] = None   # Set when status="hibernate"
```

### `EndpointUnavailable`

Exception raised by endpoints when they cannot fulfill a request.

### `WebEndpoint`

**File:** `Agent/Utilities/CommunicationEndpoints/WebEndpoint.py`

```python
class WebEndpoint(UserEndpoint):
    Name = "web"
    DisplayLabel = "Web chat"
```

Constructor: `WebEndpoint(send_chat_message_callback=None, attachment_emitter=None, ux_manager=None)`

- `SendInform()` — tags the message with `communicate_kind=inform` to prevent the frontend treating it as terminal.
- `SendResult()` — degrades silently if callback is absent (no exception raised).
- `AskText(prompt, *, context)` → calls `_EmitAskAndHibernate()`
- `AskChoice(prompt, choices, *, context)` → normalizes choices to `{"id", "label"}` dicts → calls `_EmitAskAndHibernate()`

`_EmitAskAndHibernate()` behavior:
- If prompt is empty: returns `EndpointReply(status="hibernate", hibernate_config={...})` silently.
- Otherwise: emits `ask_human` UI frame to frontend, then returns `EndpointReply(status="hibernate", hibernate_config={"trigger_type": "timer", "timeout_seconds": USER_REPLY_TIMEOUT_SECONDS, "pending_question_type": "user_reply", ...})`.

This implements the ARCH-001 hibernate-on-ask pattern — the agent suspends and the Backend resumes it when the user replies, rather than blocking the worker thread.

### `CLIEndpoint`

**File:** `Agent/Utilities/CommunicationEndpoints/CLIEndpoint.py`

```python
class CLIEndpoint(UserEndpoint):
    Name = "cli"
    DisplayLabel = "Terminal (stdin/stdout)"
    _ASK_TIMEOUT_SECONDS = 300.0
```

Constructor: `CLIEndpoint(stream_in=None, stream_out=None, lock=None, ask_timeout=300.0)`

- `SendInform()` / `SendResult()` print to stdout prefixed with `[AGENT]` / `[AGENT FINAL]`.
- `AskText()` acquires a lock, calls `_ReadLine()` with timeout via `asyncio.wait_for(run_in_executor(...))`.
- `AskChoice()` loops printing choices until a valid index or `id`/`label` match is entered.

### `EndpointRegistry`

**File:** `Agent/Utilities/CommunicationEndpoints/EndpointRegistry.py`

Lazy-instantiating registry of named endpoints.

```python
registry = EndpointRegistry()
registry.Register("web", factory=lambda: WebEndpoint(...), default=True)
registry.Register("cli", factory=lambda: CLIEndpoint(...))

endpoint = registry.Get("web")          # Lazy instantiation from factory
default = registry.Default()            # Gets the default endpoint instance

names = registry.EnabledNames()         # List[str]
```

`Register(name, factory, *, default=False)`: warns on overwrite, invalidates cached instance. Setting `default=True` registers the endpoint under both its name and `_default`.

`__len__()`, `__contains__()`, `__iter__()` are all supported.

## Embedding Providers

**Directory:** `Agent/Utilities/Embedding_Providers/`

Embedding model integrations used by `DataFetcherAgent`'s RAG pipeline and by `ReasoningOverlapAlertRule` for loop detection.

The embedding model is configured via `AgentConfig.reasoning_overlap_embedding_model` (default: `perplexity/pplx-embed-v1-0.6b`).

### BaseEmbeddingProvider

**File:** `Agent/Utilities/Embedding_Providers/Base.py`

Abstract base class for all embedding providers.

```python
class BaseEmbeddingProvider(ABC):
    @property
    @abstractmethod
    def api_key_env_var(self) -> str: ...

    @abstractmethod
    def _create_client(self): ...

    @abstractmethod
    def prepare_payload(self, payload: dict) -> dict: ...

    @abstractmethod
    async def execute_request(self, payload: dict) -> dict: ...
```

**Lazy client creation:** the HTTP/SDK client is not created in `__init__`. The first call to `get_client()` calls `_create_client()` and caches the result.

**`close()`** — graceful shutdown. Tries `await client.aclose()` first (async); falls back to `asyncio.to_thread(client.close())` for sync clients; finally drops the reference.

### EmbeddingProviderFactory

**File:** `Agent/Utilities/Embedding_Providers/EmbeddingProviderFactory.py`

Matcher-registry that routes embedding requests to the correct provider based on the model name string. Providers are registered with three arguments:

```python
factory.register(key, factory_fn, matcher_fn)
```

- `key` — unique string identifier.
- `factory_fn` — callable that returns a `BaseEmbeddingProvider` instance (called once; result is cached).
- `matcher_fn(model_name: str) -> bool` — routing predicate.

**Routing rules:**

| Provider | Matcher condition | Env var |
|----------|------------------|---------|
| Gemini | `"gemini" in model and not model.startswith("google")` | `GEMINI_API_KEY` |
| OpenRouter (default) | fallback for everything else | `OPENROUTER_API_KEY` |

**Public API:**

```python
async def embedding_request(
    payload: dict,
    session: str,
    session_id: str,
    agent_type: str,
    user_id: str
) -> dict
```

Selects the provider for `payload["model"]`, calls `execute_request()`, and records cost via the cost tracker. Returns the raw provider response dict.

**`close_all_embedding_providers()`** — calls `close()` on every cached provider instance. Call during agent shutdown.

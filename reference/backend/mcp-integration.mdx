## Architecture Overview

The MCP integration subsystem lives in `app/integrations/mcp/`. All external code imports exclusively from `MCPFacade`; internal modules must never be imported directly from outside this package.

**Two distinct paths** coexist in the system:

| Path | Entry Point | Covers |
|---|---|---|
| First-party providers | `MCPFacade` â†’ `MCPToolsService` â†’ `BaseMCPProvider` subclass | Gmail, Calendar, Drive, Docs, Sheets, YouTube, Contacts, GitHub, Slack, Outlook |
| Custom HTTP connectors | `MCPFacade` â†’ `MCPToolsService` â†’ `_execute_custom_connector_tool` | User-defined REST endpoints registered in `CustomConnector` model |
| Composio (third-party) | `ComposioService` â†’ `ComposioAdapter.get_mcp_config()` | 500+ toolkits via Composio's own MCP server â€” agent connects directly, bypasses this package |

OAuth flows for first-party providers are handled by `MCPOAuthService`. Token storage is in `MCPTokenStore`. Code scaffolding for new providers uses `IntegrationCodeGenerator`.

## MCPFacade

**File:** `app/integrations/mcp/facade.py`

Single entry point for all MCP operations outside the package. Wraps `MCPToolsService` and `MCPOAuthService` behind a stable interface. When MCP is extracted to a microservice, only this facade needs to change.

**Singleton:** `get_mcp_facade()` â€” module-level `_facade_instance`, thread-safe lazy init.

**Lazy loading:** Both `MCPToolsService` and `MCPOAuthService` are imported on first use to prevent circular imports at module load time. `MCPOAuthService` is instantiated fresh per DB session via `_get_oauth_service(db)`.

### Data classes

```python
@dataclass
class ToolResult:
    success: bool
    data: str          # JSON string
    error: Optional[str] = None

@dataclass
class ToolInfo:
    name: str
    description: str
    input_schema: Dict[str, Any]

@dataclass
class IntegrationStatus:
    provider: str
    connected: bool
    scopes: Optional[List[str]] = None
```

### Methods

| Method | Signature | Description |
|---|---|---|
| `execute_tool` | `async (user_id, tool_name, parameters, custom_connectors=None) â†’ ToolResult` | Execute a named tool; returns `ToolResult(success=False)` on any exception instead of raising |
| `list_tools` | `(user_id=None, custom_connectors=None) â†’ tuple[List[ToolInfo], List[str]]` | List all available tools; second element is list of provider names not yet connected |
| `get_oauth_url` | `async (user_id, provider, db, redirect_uri=None) â†’ str` | Start OAuth flow; returns the authorization URL |
| `handle_oauth_callback` | `async (user_id, provider, code, state, db) â†’ bool` | Complete OAuth flow after redirect; returns `True` on success |
| `get_integrations_status` | `async (user_id) â†’ List[IntegrationStatus]` | Poll `is_authenticated()` on every registered provider for this user |

**Routing note:** `get_integrations_status` iterates `self.tools_service._providers` (the global provider registry). Each entry creates a per-user provider instance via `get_provider_for_user` to check authentication state.

## MCPToolsService

**File:** `app/integrations/mcp/mcp_tools_service.py`

Central provider registry. Coordinates tool execution across all registered providers and custom HTTP connectors.

**Global singleton:** `mcp_tools_service` â€” created at import time via `_create_global_service()` which calls `mcp_providers_registry.create_default_providers()`. Override for tests with `initialize_mcp_tools_service(providers)`.

**Constants:**
- `CUSTOM_TOOL_PREFIX = "custom__"` â€” prefix for custom connector tool names
- `_MAX_PROVIDER_CACHE_SIZE = 1000` â€” per-user provider cache ceiling

### Constructor

```python
MCPToolsService(providers: List[BaseMCPProvider])
```

Builds two internal dicts:
- `_providers: Dict[str, BaseMCPProvider]` â€” global provider instances keyed by `provider_name`
- `_provider_classes: Dict[str, type[BaseMCPProvider]]` â€” provider classes for per-user instantiation
- `_user_providers_cache: OrderedDict` â€” LRU cache of `(user_id, provider_name) â†’ provider_instance`; evicts oldest entry when size reaches `_MAX_PROVIDER_CACHE_SIZE`

### Methods

| Method | Signature | Description |
|---|---|---|
| `get_provider_for_user` | `(provider_name, user_id) â†’ Optional[BaseMCPProvider]` | Return or create a per-user provider instance; caches result; returns `None` on construction failure |
| `get_all_tools` | `(user_id=None, custom_connectors=None) â†’ tuple[List[dict], List[str]]` | Collect tools from all authenticated providers + custom connectors; second element is unauthenticated provider names |
| `execute_tool` | `async (tool_name, user_id=None, custom_connectors=None, **kwargs) â†’ str` | Route to custom connector or provider; return JSON string result |
| `_resolve_provider_name_from_tool` | `(tool_name) â†’ Optional[str]` | Longest-prefix-wins scan: `"google_chat_send"` â†’ `"google_chat"`, not `"google"` |
| `_get_custom_connector_tools` | `(custom_connectors) â†’ List[dict]` | Build tool schemas from `CustomConnector` objects with `status=connected` |
| `_slugify_tool_name` | `(value) â†’ str` | `"Send Email"` â†’ `"send_email"` â€” normalize to `[a-z0-9_]` |
| `_build_custom_tool_name` | `(connector_id, slug) â†’ str` | Produces `custom__<connector_id>__<slug>` |
| `_execute_custom_connector_tool` | `async (tool_name, custom_connectors, **kwargs) â†’ str` | Decrypt connector config, build URL, issue aiohttp request; GET â†’ query params, POST â†’ JSON body |

### Custom connector tool name format

```
custom__<connector_id>__<tool_slug>
```

Two double-underscore separators. `connector_id` is the UUID from the `CustomConnector` DB row. `tool_slug` is `_slugify_tool_name(tool.name)` or `_slugify_tool_name(connector.slug)` if `connector.tools` is empty.

### Provider resolution

```
execute_tool("gmail_send_email", ...)
  â†’ not custom__ prefix
  â†’ _resolve_provider_name_from_tool("gmail_send_email")
  â†’ scans _providers.keys(): ["gmail", "calendar", "google_chat", ...]
  â†’ "gmail" matches "gmail_send_email" (len=5)
  â†’ "google_chat" does not match (no "google_chat_" prefix)
  â†’ returns "gmail"
  â†’ get_provider_for_user("gmail", user_id) or _providers["gmail"]
  â†’ provider.execute_tool("gmail_send_email", **kwargs)
```

## BaseMCPProvider

**File:** `app/integrations/mcp/base_provider.py`

Abstract base for all first-party MCP providers. Implements tool dispatch via a name-based registry built at construction time.

### Auto-mapping convention

Tool names follow `{provider_name}_{method_name}`:
- `gmail_send_email` â†’ provider `gmail`, method `send_email`
- `calendar_list_events` â†’ provider `calendar`, method `list_events`
- `google_chat_send_message` â†’ provider `google_chat`, method `send_message`

`__init__` calls `_build_tool_registry()` which iterates `get_tools()` and, for each tool, strips the `provider_name + "_"` prefix to find the service method name. Entries where the method does not exist on `self.service` are skipped with a warning.

### Registry entry structure

```python
_tool_registry[tool_name] = {
    "service_method": str,    # name of method on self.service
    "defaults": dict,         # parameter defaults from JSON schema
    "required": list[str],    # required parameter names
}
```

### Abstract interface

| Property/Method | Type | Description |
|---|---|---|
| `provider_name` | `@property @abstractmethod str` | Provider identifier used in tool names (e.g. `"gmail"`) |
| `get_tools` | `@abstractmethod () â†’ List[dict]` | Return list of `{name, description, input_schema}` dicts â€” single source of truth for tool schemas |
| `is_authenticated` | `@abstractmethod () â†’ bool` | Return `True` if this provider has valid credentials for its user |

### Concrete methods

| Method | Signature | Description |
|---|---|---|
| `execute_tool` | `async (tool_name, **kwargs) â†’ str` | Registry lookup â†’ merge defaults â†’ validate required params â†’ call service method (sync or async) â†’ return JSON string |
| `_build_tool_registry` | `() â†’ None` | Populates `_tool_registry` from `get_tools()`; called in `__init__` |
| `_tool_name_to_method` | `(tool_name) â†’ Optional[str]` | Strip `provider_name + "_"` prefix; return `None` if not matching |

**Parameter filtering:** `execute_tool` inspects `inspect.signature(method)`. If the method does not declare `**kwargs`, only known parameters are passed â€” extra kwargs (e.g. an LLM passing `max_results` when the method expects `limit`) are silently dropped. If the method has `**kwargs`, all parameters are forwarded.

## MCPOAuthService

**File:** `app/integrations/mcp/mcp_oauth_service.py`

Handles the complete OAuth lifecycle for first-party providers. Instantiated fresh per request with a DB session â€” not a singleton.

**Factory:** `get_mcp_oauth_service(db: AsyncSession) â†’ MCPOAuthService`

**Supported providers:** `gmail`, `calendar`, `drive`, `docs`, `sheets`, `youtube`, `github`, `slack`, `outlook`, `contacts`

**Google providers constant:** `_GOOGLE_PROVIDERS = {"gmail", "calendar", "drive", "docs", "sheets", "contacts", "youtube"}` â€” used to decide whether to call Google's token revocation endpoint.

### Constructor

```python
MCPOAuthService(db: AsyncSession)
```

Builds `_adapter_factories: Dict[str, Callable]` â€” maps provider name to a lambda `user_id â†’ OAuthAdapter`. Each lambda creates the provider-specific config object and corresponding adapter.

### Methods

| Method | Signature | Description |
|---|---|---|
| `get_oauth_adapter` | `(provider, user_id) â†’ OAuthAdapter` | Instantiate adapter from factory; raises `ValueError` for unknown provider |
| `initiate_oauth` | `async (user_id, provider, redirect_uri=None) â†’ Dict` | Create/update `UserMCPIntegration(status=pending)`, generate auth URL, encode timestamp into state as `"{state}:{int(time.time())}"`, store in `extra_data["oauth_state"]`; returns `{auth_url, state, provider}` |
| `handle_oauth_callback` | `async (user_id, provider, code, state) â†’ Dict` | Validate CSRF state, exchange code via `oauth_server.exchange_code(code)`, save credentials via `mcp_token_store.asave_credentials()`, set `status=connected`, store GitHub username in `account_label` if present; returns `{success, provider, message, redirect_uri}` |
| `refresh_token` | `async (user_id, provider) â†’ bool` | Trigger `adapter.get_stored_credentials()` which auto-refreshes; returns `True` if valid credentials result |
| `revoke_integration` | `async (user_id, provider) â†’ bool` | For Google providers: call `revoke_google_token()` first; then `adelete_credentials`; set `status=revoked` in DB |

### Standalone helper

```python
async def revoke_google_token(user_id: str, provider: str) -> None
```

Reads stored credentials, prefers `refresh_token` for the revocation POST to `https://oauth2.googleapis.com/revoke`. Accepts `200` and `400` as success (Google returns 400 for already-revoked tokens).

### CSRF state encoding

```python
state = f"{oauth_server.state}:{int(time.time())}"
```

The timestamp suffix allows server-side TTL validation without a separate Redis key. The state is injected into the authorization URL's `state` query parameter (replacing any existing one) and stored in `extra_data["oauth_state"]`.

## OAuthMCPService

**File:** `app/integrations/mcp/common/base_oauth_service.py`

Base class for Google OAuth-backed MCP services. Extends `BaseMCPService`. All 7 Google provider services (Gmail, Calendar, Drive, Docs, Sheets, YouTube, Contacts) extend this class.

### Constructor

```python
OAuthMCPService(
    provider_name: str,   # e.g. "gmail"
    config: Any,          # GmailConfig, CalendarConfig, etc.
    user_id: str | None = None,
)
```

Creates `GoogleOAuthAdapter`, initializes a `threading.Lock` for the lazy client, and auto-registers tools by calling `get_tool_mappings()` and `register_tool()` for each entry.

### Methods

| Method | Signature | Description |
|---|---|---|
| `is_authenticated` | `() â†’ bool` | Delegates to `oauth_adapter.is_authenticated()`; catches all exceptions, returns `False` |
| `client` (property) | `â†’ Any` | Lazy API client with double-checked locking â€” authenticates on first access via `oauth_adapter.authenticate()` then `create_client(credentials)` |
| `create_client` | `@abstractmethod (credentials) â†’ Any` | Subclass creates the provider-specific API client from Google credentials |
| `get_tool_mappings` | `() â†’ Dict[str, Callable]` | Override to declare tool name â†’ bound method mappings; default returns `{}` for backward compat |
| `_tool` | `async (func, *args, error_code=None, **kwargs) â†’ str` | Unified execution wrapper â€” handles `PermissionError` (returns `{success:False, error_code:...}`) and all other exceptions (returns `{success:False, error:...}`) |

**Double-checked locking pattern:**

```python
if self._client is None:
    with self._client_lock:
        if self._client is None:          # re-check under lock
            credentials = self.oauth_adapter.authenticate()
            self._client = self.create_client(credentials)
```

## GoogleOAuthAdapter

**File:** `app/integrations/mcp/common/google_oauth.py`

Unified Google OAuth adapter replacing 7 near-identical per-provider adapters. Extends `BaseOAuthAdapter`.

### Constructor

```python
GoogleOAuthAdapter(
    provider_name: str,        # e.g. "gmail"
    config,                    # must have: client_id, client_secret, token_file,
                               #   redirect_uri, oauth_port, scopes
    user_id: str | None = None,
)
```

Sets `_include_granted_scopes = provider_name != "contacts"` (contacts uses `False` to avoid scope conflicts).

### Methods

| Method | Signature | Description |
|---|---|---|
| `create_oauth_server` | `() â†’ GoogleOAuthServer` | Build `GoogleOAuthServer` with `include_granted_scopes` flag |
| `authenticate` | `() â†’ Credentials` | Try `get_stored_credentials()` first; fall back to `authenticate_via_web()`; save to DB (if `user_id`) or file |
| `get_stored_credentials` | `() â†’ Optional[Credentials]` | DB lookup first (if `user_id`); auto-refresh expired tokens and save back; fall back to `token_file`; same auto-refresh for file-based creds |
| `validate_credentials` | `(credentials) â†’ bool` | Return `credentials is not None and credentials.valid` |

**Storage preference:** When `user_id` is set, all credential reads and writes go to DB via `mcp_token_store`. File-based storage is only used for serverless/local scenarios without a user context.

**Auto-refresh:** Both DB and file paths call `google.auth.transport.requests.Request()` for expired-but-refreshable tokens and persist the new token immediately.

## ComposioToolkitAdapter

**File:** `app/services/composio_adapter.py`

Handles toolkit discovery and the managed-toolkit slug registry. Composed into `ComposioAdapter`.

Inherits shared infrastructure from `_ComposioBase` (API key, environment prefix, lazy Composio SDK client).

**Redis cache keys:**
- `composio:toolkits:all` â€” full toolkit list, TTL 1 hour
- `composio:managed_toolkit_slugs:v1` â€” set of connectable slugs, TTL 1 hour

### Methods

| Method | Signature | Description |
|---|---|---|
| `get_toolkits` | `async (query=None, category=None) â†’ List[ToolkitInfo]` | Fetch all toolkits from Redis cache or SDK; apply query/category filters in-memory; enrich with `is_auth_managed` flag |
| `get_managed_toolkit_slugs` | `async () â†’ Set[str]` | Return lowercased set of connectable slugs; union of mechanism (1) â€” `composio_managed_auth_schemes` non-empty and mechanism (2) â€” explicit `auth_configs` registered in the Composio project |

**Two-mechanism managed-toolkit detection:**

Mechanism (1): Composio provides default OAuth credentials â€” `composio_managed_auth_schemes` is non-empty (~121 toolkits: Gmail, Slack, GitHub, Notion, HubSpot, Salesforce, Discord, etc.).

Mechanism (2): Skygen registered its own OAuth app via the Composio Dashboard â€” toolkit appears in `client.auth_configs.list()` (e.g. Telegram, ClickHouse, Perplexity, Reply.io, Cursor).

Both lists are paginated; pagination is capped at 20 pages (~10k toolkit ceiling). Union of both sets is returned. On Redis error, falls through to a live SDK call without writing back.

**Fail-open on `get_toolkits`:** If `get_managed_toolkit_slugs` raises `ComposioUnavailableError`, every toolkit is marked `is_auth_managed=True` to preserve pre-rollout behavior.

## ComposioConnectionAdapter

**File:** `app/services/composio_adapter.py`

Manages connected accounts, MCP server config generation, and semantic tool search. Composed into `ComposioAdapter`.

**User ID prefixing:** All Composio API calls use `prefixed_user_id(user_id)` = `"{environment}_{user_id}"` to isolate dev/staging/prod data within a single Composio account.

### Methods

| Method | Signature | Description |
|---|---|---|
| `get_user_connections` | `async (user_id) â†’ List[ConnectionInfo]` | List all connected accounts for user; runs `_extract_account_identity` on each |
| `initiate_auth` | `async (user_id, toolkit, redirect_url=None, allow_multiple=False, connection_data=None) â†’ AuthResult` | Paginate `auth_configs.list()` for toolkit match; if `connection_data` + `auth_config_id`: use V3 `/connected_accounts/link`; otherwise `session.authorize()` (Composio default credentials) |
| `disconnect` | `async (user_id, toolkit, account_id=None) â†’ bool` | Resolve `account_id` from first matching connection if omitted; call `connected_accounts.delete()` |
| `get_mcp_config` | `async (user_id, toolkits=None, disabled_toolkits=None) â†’ MCPConfig` | Create Composio session with `entity_id=prefixed_user_id`; return `MCPConfig(url, headers, entity_id, disabled_toolkits)` for direct agent â†’ Composio MCP connection |
| `get_tools_for_task` | `async (user_id, task_description, max_tools=20) â†’ List[ToolInfo]` | Composio semantic search for relevant tools |

**Account identity extraction:** `_extract_account_identity` uses a 6-tier fallback chain to extract email/display_name from Composio's inconsistent connection objects:
1. `data` dict fields
2. `state.val` extras
3. Deprecated `labels`
4. Top-level object attributes
5. Nested `member` object
6. `__pydantic_extra__` dict

For providers with masked API tokens (Gmail, Slack), `_resolve_email_via_action` calls a lightweight profile action (e.g. `GMAIL_GET_PROFILE`, `SLACK_AUTH_TEST`) to retrieve the actual email address.

## Overview

All business logic lives in `Backend/app/services/`. Services are consumed by routers via FastAPI `Depends()`, by Socket.IO handlers, and by the RabbitMQ worker. The architecture follows **Clean Architecture**: routers call services, services call repositories, repositories call ORM models.

## Core Infrastructure Services

### ActivityTracker (`activity_tracker.py`)

Throttled `User.last_active_at` updates via Redis SETNX (60-second TTL per user).

**Key functions:**

```python
async def touch_user_activity(user_id: str) -> None
def fire_and_forget_touch(user_id: str) -> None
```

`fire_and_forget_touch` wraps `touch_user_activity` in `asyncio.create_task()` for use in synchronous middleware. On Redis failure, falls through to DB write.

### SessionCleanupService (`session_cleanup.py`)

Unified finalization after agent terminal responses. Extracted to eliminate ~80 LOC duplication between the Socket.IO and Worker paths.

**Key functions:**

```python
async def finalize_agent_response(
    session_id, user_id, device_id, message_id, content, meta_data,
    is_scheduled=False, notify_via="socketio"
```

Steps: unlock session → unlock device → update scheduled task run → release Cloud PC control → notify device.

**Called by:** `socketio/agent_handlers/message_handlers.py`, `worker/handlers/agent_response_handler.py`

### DeviceLockManager (`device_lock_manager.py`)

Redis-based device and session locking via Lua atomic scripts.

**Key functions:**

```python
async def lock_device(redis_client, device_id, session_id, user_id) -> bool
async def unlock_device(redis_client, device_id, session_id) -> bool
async def lock_session(redis_client, session_id) -> bool
async def unlock_session(redis_client, session_id) -> None
async def bind_cloud_pc_session(redis_client, user_id, session_id) -> None
async def unbind_cloud_pc_session(redis_client, user_id) -> None
```

**Class:**

```python
class CloudPCAccessManager:
    async def acquire(session_id, cpc_id) -> bool
    async def release(session_id, force=False) -> bool
```

`CloudPCAccessManager` is the high-level facade used by the cloud-pc router for unified lock cleanup on session deletion.

**Lua scripts:**
- `_LOCK_OR_REACQUIRE_LUA` — returns 1 (new), 2 (same-session re-acquire), 0 (contested). Same-session re-acquire bumps TTL only, preserving `locked_at` to surface staleness.

### CloudPCControlService (`cloud_pc_control_service.py`)

Manages manual vs AI input control over the Cloud PC (take-control / resume-AI flow). Singleton via `get_cloud_pc_control_service()`.

```python
@dataclass
class TakeControlResult:
    status: str          # "paused", "no_active_agent"
    controller: str
    session_id: Optional[str] = None

@dataclass
class ResumeAIResult:
    status: str          # "resumed", "no_paused_agent", "device_conflict"
    controller: str
    queued_messages_count: int = 0

@dataclass
class RenewControlResult:
    status: str          # "renewed", "no_active_session"
    ttl: int = 120

class CloudPCControlService:
    async def take_control(db, user_id, session_id) -> TakeControlResult
    async def resume_ai(db, user_id, session_id, current_user, device_id=None) -> ResumeAIResult
    async def renew_control(db, user_id) -> RenewControlResult
```

**`take_control` steps:** validate session ownership → check agent is active (`is_session_active`) → pause session in Redis (`pause_session`) → send pause signal via Socket.IO (`send_agent_pause`) → save system "paused" message to DB → acquire user control via `CloudPCService.acquire_control`.

**`resume_ai` steps:** validate session → check session is paused → verify agent saved state (`agent_history:{session_id}` Redis key with `status=PAUSED`) → drain queued pause-period messages → acquire session lock (force=True) → unpause session → refresh device lock → build wake-reason prompt with queued messages → publish to RabbitMQ → release user control. **Rollback on failure:** re-pause, re-queue messages, release device lock via `CloudPCAccessManager.release`.

**`renew_control`:** calls `CloudPCService.renew_control(user_id, "user")` to bump the controller TTL.

### TriggerService (`trigger_service.py`)

Event-driven trigger orchestration: HMAC webhook verification, payload normalization, agent dispatch.

**Key classes:**

```python
class WebhookVerifier:
    def verify(payload: bytes, signature: Optional[str]) -> bool
    # Raises WebhookVerificationError on invalid/missing signature.
    # Returns True on success; False when Composio is unavailable.
    # Reads the HMAC-SHA256 secret from settings (not passed as arg).

class WebhookEventParser:
    def parse(raw: dict) -> WebhookEvent
    # Maps Composio webhook dict → WebhookEvent(event_type, user_id,
    # toolkit, trigger_id, payload, action_prompt, trigger_event_id).

class TriggerDispatcher:
    async def dispatch(
        event: WebhookEvent,
        action_prompt: Optional[str],
        trigger_event_id: Optional[str],
    ) -> bool
    # Publishes trigger event to active user sessions via Socket.IO
    # /agent namespace. Returns True if at least one session was notified.

class TriggerService:
    async def handle_webhook(payload: bytes, signature: Optional[str]) -> Dict[str, Any]
    # Returns {"status": "ok", "trigger_id": str} or {"status": "ignored", ...}.
    # Verifies HMAC, parses payload, dispatches event.

    # --- CRUD (all require ComposioService internally) ---
    async def create_trigger(db, user_id, toolkit, trigger_name, config) -> TriggerConfig
    async def list_triggers(db, user_id) -> List[TriggerConfig]
    async def delete_trigger(db, user_id, trigger_id) -> bool
    async def update_trigger_config(db, user_id, trigger_id, config) -> TriggerConfig

    # --- Event log ---
    async def get_trigger_events(db, user_id, trigger_id=None, limit=50) -> List[TriggerEvent]
    async def get_trigger_runs(db, user_id, trigger_id=None, limit=50) -> List[TriggerRun]
    async def update_trigger_execution(db, event_id, status, result=None) -> TriggerEvent

    # --- Discovery ---
    async def list_trigger_types(toolkit=None) -> List[TriggerTypeInfo]
    async def list_trigger_app_counts() -> List[Dict[str, Any]]
```

Composio sends three event types: `trigger_fired`, `connection_updated`, `connection_error`. Only `trigger_fired` routes through `TriggerDispatcher`; the other two update connection state.

## Knowledge Base Services

### KB Service (`kb_service.py`)

Module-level async functions (no class) for knowledge base operations. Backed by **Gemini Developer API** (`generativelanguage.googleapis.com/v1beta/.../batchEmbedContents`, keyed by `GEMINI_API_KEY`). Produces 768-dimensional vectors stored in pgvector with cosine-similarity search.

See [kb-service](/reference/backend/kb-service) for full reference.

- **3-level Copy-on-Write** — global insights (`user_id='global'`) → user-level overrides → emplogent-level overrides (`agent_type='character:<id>'`)

- **Embeddings** — `get_embeddings(texts)` calls Gemini Developer API directly. `get_embeddings_openrouter` exists as a separate function but is **not** called from any CRUD path


## Scheduling Services

### ScheduledTaskService (`scheduled_task_service.py`)

DST-aware cron scheduler with Redis distributed lock.

See [scheduled-task-service](/reference/backend/scheduled-task-service) for full reference.

```python
async def calculate_next_run(
    frequency: str,          # once | daily | weekly | monthly | hourly
    time_of_day: str,        # "HH:MM"
    tz: str,                 # IANA timezone string e.g. "Europe/Moscow"
    day_of_week: Optional[int] = None,    # 0=Mon..6=Sun
    day_of_month: Optional[int] = None,
    after: Optional[datetime] = None,
) -> Optional[datetime]
# Returns None for frequency='once' after the task has already run.

async def execute_scheduled_task(db, task, scheduled_for=None) -> ScheduledTaskRun
# 6 steps: per-task exec lock → billing check → Cloud PC auto-provision
# → variable substitution → create run row → publish to RabbitMQ.

async def scheduler_loop() -> None
# Runs as a background task. Acquires
# "scheduled_tasks:scheduler_lock:{env}" (SETNX, 120s TTL) so only
# one pod acts as scheduler. Renews lock every 30s.
# Two-phase pattern: execute first, advance next_run_at only on success.
# Fails stale runs older than MAX_LATE_SECONDS (3600) via fail_stale_runs.
```

**Redis locks used:**
- `scheduled_tasks:scheduler_lock:{env}` — singleton scheduler, 120s TTL, renewal every 30s
- `sched_task:exec:{task.id}` — per-task execution dedup, 300s TTL

## Email Services

### EmailService (`email_service.py`)

Amazon SESv2 async transport with exponential-backoff retry.

```python
async def send_email(
    *,                         # kwargs-only
    to: str,
    subject: str,
    html: str,
    text: Optional[str] = None,
    tags: Optional[List[dict]] = None,       # SES message tags [{Name, Value}]
    unsubscribe_url: Optional[str] = None,   # sets List-Unsubscribe header
    reply_to: Optional[str] = None,
) -> bool
```

Never raises. Returns `True` on success, `False` on failure. Permanent errors (MessageRejected, domain not verified, account suspended) are never retried. Transient errors retry up to 3× with exponential backoff + jitter (`base_delay=0.5s`). Short-circuited when `MAIL_ENABLED=False`.

## Monitoring Infrastructure

Enabled only when `ENABLE_METRICS=true` in the ConfigMap (controls `prometheus-fastapi-instrumentator` registration at startup in `app/main.py`). The `/metrics` endpoint is served by the instrumentator. All metrics carry an `environment` label.

### MetricsCollector (`app/monitoring/metrics.py`)

Periodic background collector that scrapes DB, Redis, and RabbitMQ to update Prometheus Gauge metrics. Wraps the module-level metric objects defined in the same file.

**Global instance management:**
```python
init_metrics_collector(environment: str = "development") → MetricsCollector
get_metrics_collector() → Optional[MetricsCollector]
```

**Constructor:**
```python
MetricsCollector(environment: str = "development")
```
Sets `APP_INFO.info({"environment": environment, "service": "skygen-backend"})` immediately.

**Lifecycle:**
| Method | Description |
|---|---|
| `start_collection(interval_seconds=30)` | Create asyncio Task for `_collection_loop`; attaches `log_task_exception` callback; no-op if already running |
| `stop_collection()` | Cancel the task; await `CancelledError` |
| `collect_all_metrics()` | Run all 5 sub-collectors via `asyncio.gather(return_exceptions=True)` |

**Sub-collectors:**
| Method | Source | Metrics updated |
|---|---|---|
| `_collect_user_metrics` | PostgreSQL | `skygen_users_total`, `skygen_users_active` |
| `_collect_websocket_metrics` | Local Socket.IO memory | `skygen_websocket_connections[device/agent]`, `skygen_devices_connected`, `skygen_users_online` |
| `_collect_session_metrics` | PostgreSQL | `skygen_chat_sessions_total`, `skygen_chat_messages_total[user/assistant/system]` |
| `_collect_queue_metrics` | RabbitMQ (`passive=True` declare) | `skygen_queue_messages`, `skygen_queue_consumers` — skipped if `QUEUE_METRICS_ENABLED=False` |
| `_collect_engagement_metrics` | PostgreSQL | `skygen_users_dau/wau/mau`, `skygen_users_new_today`, `skygen_messages_sent_today[user/assistant]` |

## Display & Transformation Services

### DisplayTransformer (`display_transformer.py`)

Pure transformation layer — no I/O, no async, no DB, no Redis. Maps the agent's internal wire protocol into `DisplayMessage` envelopes consumed by the frontend over WebSocket. 872 lines.

**DisplayMessage envelope** (`app/schemas/display_message.py`):

```python
class DisplayMessage(BaseModel):
    display_type: DisplayType      # see enum below
    session_id: str
    message_id: Optional[str]
    timestamp: str                 # ISO 8601
    render_data: Dict              # typed per display_type
    is_terminal: bool = False
    is_persistent: bool = True
    tokens_remaining: Optional[float]
    agent_mode: Optional[str]      # auto | cua | deep_research | mcp | swe | llm
    notification_id: Optional[str]
```

**`DisplayType` enum values:**

| Value | Persisted? | `render_data` model | When emitted |
|---|---|---|---|
| `user_message` | Yes | `UserMessageData` | User sends a message |
| `assistant_message` | Yes | `AssistantMessageData` | Agent terminal response |
| `ask_human_message` | Yes | `AskHumanMessageData` | Agent needs user input (ARCH-001) |
| `status_update` | No | `StatusUpdateData` | Transient progress (started, thinking, planning, executing, hibernated, paused) |
| `plan_update` | No | `PlanUpdateData` | Step list update |
| `mode_progress` | No | `ModeProgressData` | Per-mode progress (cua, deep_research, mcp, swe, ask_human) |
| `attachment_ready` | No | `AttachmentReadyData` | File published to S3 |
| `billing_notice` | No | `BillingNoticeData` | Limit reached / trial expiring |

**`_INTERNAL_FIELDS`** stripped from all outbound `render_data`:
`worker_id`, `sub_agent_count`, `sleep_seconds`, `error`, `reasoning`, `condition`, `type`

**Public transform methods:**

```python
class DisplayTransformer:
    def transform_agent_response(meta_data: dict, content: str, session_id: str,
                                  message_id: str) -> DisplayMessage
    # communicate_kind="inform" → non-terminal ASSISTANT_MESSAGE.
    # billing_limit detection → sets billing_limit=True on render_data.

    def transform_agent_status(status: str, session_id: str,
                                message_id: Optional[str] = None,
                                content: Optional[str] = None) -> DisplayMessage
    # → STATUS_UPDATE (is_persistent=False, is_terminal=False)

    def transform_agent_error(meta_data: dict, content: str, session_id: str,
                               message_id: str) -> DisplayMessage
    # → ASSISTANT_MESSAGE with status="error" (is_terminal=True)

    def transform_agent_step(meta_data: dict, session_id: str,
                              message_id: str) -> DisplayMessage
    # → MODE_PROGRESS (is_persistent=False)

    def transform_plan_update(meta_data: dict, session_id: str,
                               message_id: str) -> DisplayMessage
    # → PLAN_UPDATE (is_persistent=False)

    def transform_ui_meta(meta_data: dict, session_id: str,
                           message_id: str) -> DisplayMessage
    # Dispatch on meta_data["mode"]:
    #   "ask_human" + stage="input" → ASK_HUMAN_MESSAGE
    #     validates action_id as UUID; raises ValueError for non-UUID
    #   "mcp_connect_prompt" → ASK_HUMAN_MESSAGE with McpConnectBundle
    #   "start_cloud_pc_prompt" → ASK_HUMAN_MESSAGE with StartCloudPcPrompt
    #   anything else → MODE_PROGRESS

    def transform_attachments(attachments: list, session_id: str,
                               message_id: str) -> DisplayMessage
    # → ATTACHMENT_READY (is_persistent=False)

    def from_stored_message(db_message: dict, session_id: str) -> Optional[DisplayMessage]
    # Rehydrates persisted DB rows on page reload.
```

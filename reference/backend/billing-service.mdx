## Overview

`BillingService` (`app/services/billing_service.py`) manages the complete billing lifecycle: free plan token grants, paid subscription activation, monthly renewals, per-minute Cloud PC billing, and admin adjustments. It delegates tier-specific access checks to the `TIER_STRATEGIES` registry (`billing/tier_strategies.py`) and writes every monetary event as an immutable `TokenTransaction` record.

**Singleton:** `billing_service = BillingService()` — import from `app.services.billing_service`.

**Constructor dependencies:** None (stateless service, uses `billing_repo` and `BillingRepository`).

## BillingService class

### BillingService.can_user_access()

```python
async def can_user_access(
    self, db: AsyncSession, user_id: str
) -> Tuple[bool, str, Optional[Dict]]
```

Authoritative gate check before any agent execution. Performs lazy free-plan creation on first call.

<ResponseField name="db" type="AsyncSession" required>Database session.</ResponseField>
<ResponseField name="user_id" type="str" required>UUID string of the user.</ResponseField>

**Returns:** `(can_access: bool, reason: str, extra: dict | None)`

Reason codes:
- `"ok"` — access granted
- `"no_subscription"` — no plan found and no free plan configured
- `"free_limit_reached"` — free tokens exhausted; `extra["next_renewal"]` is ISO timestamp
- `"subscription_limit_reached"` — paid tokens exhausted; `extra["can_enable_overage"]` is `True`
- `"payment_required"` — subscription is `INCOMPLETE`
- `"payment_failed"` — subscription is `PAST_DUE`
- `"subscription_expired"` — subscription is `CANCELED` or `EXPIRED`

**Side effects:** May create a free `Subscription` + `TokenBalance` row for first-time users (lazy assignment). May call `stripe_service.renew_subscription_early()` for paid users who are low on tokens.

**Called by:**
- `routers/message/message_router.py` — before publishing to RabbitMQ
- `services/scheduled_task_service.py:execute_scheduled_task()`
- `services/cloud_pc/lifecycle.py:_ensure_cloud_pc_running()`
- `routers/billing/billing_router.py` — GET `/billing/status`

```python
can_access, reason, extra = await billing_service.can_user_access(db, str(user.id))
if not can_access:
    raise HTTPException(status_code=402, detail=reason)
```

### BillingService.activate_subscription()

```python
async def activate_subscription(
    self,
    db: AsyncSession,
    user_id: str,
    stripe_subscription_id: str,
    stripe_price_id: str,
    current_period_start: datetime,
    current_period_end: datetime,
) -> Subscription
```

Activates or upgrades a paid subscription, credits the plan's monthly token allotment to the user's balance (additive, not a reset), and records a `SUBSCRIPTION_TOPUP` transaction.

**Idempotent:** If called a second time with the same `stripe_subscription_id` and the subscription is already `STANDARD + ACTIVE`, returns early without double-crediting.

<ResponseField name="stripe_subscription_id" type="str" required>Stripe Subscription ID.</ResponseField>
<ResponseField name="stripe_price_id" type="str" required>Stripe Price ID used to look up the plan and token allotment.</ResponseField>
<ResponseField name="current_period_start" type="datetime" required>Naive UTC datetime from Stripe.</ResponseField>
<ResponseField name="current_period_end" type="datetime" required>Naive UTC datetime computed via `_period_end_from_plan()` (uses plan's `billing_period_days`, not raw Stripe value).</ResponseField>

**Returns:** `Subscription` ORM object.

**Side effects:** Updates `subscriptions` row, inserts `TokenTransaction(SUBSCRIPTION_TOPUP)`, commits.

**Called by:** `StripeService._handle_subscription_created()`, `StripeService._handle_setup_intent_succeeded()`, `StripeService._handle_checkout_session_completed()`.

### BillingService.switch_plan()

```python
async def switch_plan(
    self,
    db: AsyncSession,
    user_id: str,
    stripe_subscription_id: str,
    stripe_price_id: str,
    current_period_start: datetime,
    current_period_end: datetime,
) -> Subscription
```

Handles a plan change. Unlike `activate_subscription` which **adds** to the balance, this **resets** it to the new plan's allocation to prevent token farming via rapid plan cycling.

**Idempotent:** Skips if the subscription is already `ACTIVE` on the same `stripe_price_id`.

**Side effects:** Resets `token_balances.balance` to `plan.monthly_tokens`, records `SUBSCRIPTION_TOPUP` transaction.

**Called by:** `StripeService._handle_invoice_paid()` (billing_reason=`subscription_update`), `StripeService.change_plan()`.

### BillingService.calculate_token_cost()

```python
def calculate_token_cost(self, api_cost_usd: float) -> int
```

Converts a USD API cost to an integer token count, applying the margin multiplier from settings.

Formula: `ceil(api_cost_usd * TOKEN_MARGIN_MULTIPLIER * TOKENS_PER_DOLLAR)`, minimum 1.

**Returns:** Integer token count. Returns `0` if `api_cost_usd <= 0`.

**Called by:** `BillingService.check_sufficient_balance()`.

### BillingService.deduct_tokens_for_api_call()

```python
async def deduct_tokens_for_api_call(
    self,
    db: AsyncSession,
    user_id: str,
    api_cost_usd: float,
    reference_id: Optional[str] = None,
    description: Optional[str] = None,
) -> Tuple[bool, int, int, bool, Optional[str]]
```

Deducts tokens for an LLM API call. Uses fractional token accumulation — sub-token costs are accumulated in `balance.fractional_tokens` until they round up to at least 1 whole token, preventing per-call rounding losses.

For free users: hard cap, never goes negative.
For paid users: attempts early renewal via Stripe if balance is insufficient.

<ResponseField name="api_cost_usd" type="float" required>Actual USD cost of the API call.</ResponseField>
<ResponseField name="reference_id" type="str">Optional session or task ID for transaction record linkage.</ResponseField>

**Returns:** `(success, tokens_deducted, remaining_balance, is_overage, error_message)`

**Side effects:** Updates `token_balances`, inserts `TokenTransaction(API_USAGE)`, commits. May call `stripe_service.report_overage_usage()` for overage billing.

**Called by:** `routers/message/message_router.py` — after agent completes, records actual cost; `Agent/Services/cost_tracker.py`.

```python
success, deducted, balance, is_overage, err = await billing_service.deduct_tokens_for_api_call(
    db, user_id=str(user.id), api_cost_usd=0.0234, reference_id=str(session.id)
)
if not success:
    logger.warning("Token deduction failed: %s", err)
```

## Tier access strategies

### FreeTierStrategy.check_access()

Logic for `FREE` and `TRIAL` tiers. Checks balance >= 2 tokens. If low, attempts free token renewal (gated by `_can_renew_free_limit()`). Returns `free_limit_reached` with `next_renewal` timestamp if renewal is not yet due.

### PaidTierStrategy.check_access()

Logic for `STANDARD` tier. Checks subscription status, then balance. If low and `allow_overage=True`, grants access. Otherwise attempts `stripe_service.renew_subscription_early()`. Returns `subscription_limit_reached` with `can_enable_overage: True` if renewal fails.

## Webhook handler registry

`billing/webhook_handlers.py` defines `WEBHOOK_HANDLERS: Dict[str, WebhookEventHandler]` mapping Stripe event types to handler instances:

| Event type | Handler |
|---|---|
| `setup_intent.succeeded` | `SetupIntentSucceededHandler` |
| `checkout.session.completed` | `CheckoutSessionCompletedHandler` |
| `customer.subscription.created` | `SubscriptionCreatedHandler` |
| `customer.subscription.updated` | `SubscriptionUpdatedHandler` |
| `customer.subscription.deleted` | `SubscriptionDeletedHandler` |
| `invoice.paid` | `InvoicePaidHandler` |
| `invoice.payment_failed` | `InvoicePaymentFailedHandler` |
| `payment_method.attached` | `PaymentMethodAttachedHandler` |

Each handler delegates to `StripeService._handle_*()` methods documented in `stripe-service`.

## Overview

Repositories are the data access layer in Skygen's Clean Architecture stack:

```
Routers → Services → Repositories → Models (SQLAlchemy)
```

All repository functions:
- Accept `db: AsyncSession` as first parameter (injected via FastAPI `Depends(get_db)` or passed from services)
- Return ORM model instances or `None` / typed lists
- Use soft-delete (`deleted_at` timestamp) everywhere except where noted
- Never raise HTTP exceptions — callers in services/routers handle 404/403 mapping

Location: `Backend/app/repositories/`

---

## BaseRepository

**File:** `Backend/app/core/repository.py`

Generic async repository providing CRUD, pagination, and soft-delete. All domain repositories subclass it.

```python
class MyRepo(BaseRepository[MyModel]):
    model = MyModel

repo = MyRepo()
obj = await repo.get_by_id(db, some_uuid)
```

**Design rules (read the comments in the file):**
- Uses `flush()`, NOT `commit()`. The caller (route handler via `get_db` context manager) owns the transaction boundary. This enables atomic multi-entity operations.
- `db: AsyncSession` is passed to every method, never stored on the instance — keeps repositories stateless and safe for concurrent use.
- Soft-delete is automatic when the model has `SoftDeleteMixin` (column `deleted_at`).

### Methods

| Method | Signature | Notes |
|--------|-----------|-------|
| `get_by_id` | `(db, id, *, include_deleted=False) → Optional[T]` | Fetch by primary key. Excludes soft-deleted by default. |
| `get_by` | `(db, *, include_deleted=False, **filters) → Optional[T]` | Fetch first match for arbitrary field filters. |
| `list_all` | `(db, *, filters=None, active_only=True, limit=50, offset=0, order_by=None) → List[T]` | Paginated listing. `limit` capped at 200. |
| `count` | `(db, *, filters=None, active_only=True) → int` | Count matching records. |
| `create` | `(db, **kwargs) → T` | Insert + flush + refresh. Does not commit. |
| `update` | `(db, obj, **kwargs) → T` | Set fields + flush + refresh. Does not commit. |
| `soft_delete` | `(db, obj) → None` | Sets `obj.deleted_at = utcnow()`. Raises `AttributeError` if model lacks `SoftDeleteMixin`. |
| `hard_delete` | `(db, obj) → None` | `db.delete(obj)` + flush. Use only for non-soft-deletable models. |

## Messaging & Communication

### send_msg_to_agent_repo.py

**Not a standard CRUD repository.** Contains `get_payload_for_agent(db, redis, session_id, user_id, message_content, ...)` — the primary pre-flight check before dispatching a user message to the agent.

**Responsibilities:**
- Load session + device state
- Acquire device lock via `DeviceLockManager`
- Check Cloud PC availability
- Build the full payload dict for `BrokerAgent`

**Raises:**
- `CloudPCSessionConflict` — session conflicts with an active Cloud PC session
- `CloudPCNotReady` — Cloud PC is not in ACTIVE state

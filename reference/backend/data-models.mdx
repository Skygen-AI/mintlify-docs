# Data Models

All models are defined in `Backend/app/models/`. Primary keys are UUID strings (VARCHAR(36)) unless noted otherwise. Timestamps are `DATETIME` (UTC). Soft-delete patterns vary by model â€” see each entry.

**Common patterns:**
- `created_at` / `updated_at` â€” on every model; `updated_at` is auto-updated on write
- Soft-delete â€” three patterns in use: `deleted_at` timestamp, `is_active=False`, or `status=deleted`
- All foreign keys reference UUID string PKs

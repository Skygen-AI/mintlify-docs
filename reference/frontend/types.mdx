# Frontend Type Reference

All types are defined in `WebApp/ui-shared-tauri/src/types/`. The file `types/api.ts` is a re-export hub for backward compatibility — prefer importing from domain-specific files in new code.

---

## Authentication (`types/auth.ts`)

### `User`

```typescript
interface User {
  id: string;
  email: string;
  is_active?: boolean;
  is_admin?: boolean;
  is_email_verified?: boolean;
  otp_secret?: boolean;         // whether 2FA is enrolled
  is_2fa_verified?: boolean;
  display_name?: string;
  auth_provider?: string;       // "local" | "google"
  created_at?: string;
}
```

**Used by:** `authSlice`, `useAuth` hook, `GET /users/me`

<Warning>
`otp_secret?: boolean` is declared on the frontend `User` type but is **not sent** by the backend `UserRead` schema. `authSlice` uses `!user.otp_secret` to set `needsOTPSetup`, which means `needsOTPSetup` is always `true` for every user (since `undefined` is falsy). This is a confirmed bug — see type-mismatches for severity rating.
</Warning>

```typescript
interface Session {
```

**Used by:** `sessionSlice`, session list, session detail views

</Info>

### `MessageMetadata`

The metadata union. All variants extend `BaseMessageMetadata` and intersect with `Record<string, unknown>` for forward compatibility.

```typescript
// The base — every meta variant carries at least these
interface BaseMessageMetadata {
  status?: "processing" | "completed" | "error" | "cancelled" | "paused" | "stopped";
  action?: string;
  screenshot_url?: string;
  cancelled?: boolean;
  source?: "file_upload" | string;
  filename?: string;
  agent_mode?: string;
  mode?: string;
  is_follow_up?: boolean;
  rating?: number;
  rating_comment?: string | null;
  type?: string;              // discriminator: "billing_notification" | "agent_error" | "ui_meta_update" | "ask_human"
  ask_human_action?: string;
  published_files?: PublishedFile[];
  is_milestone?: boolean;
  sub_agent_count?: number;
  task_group_id?: string;
  plan_step_id?: string;
}

// Type-narrowed subtypes
type MessageMetadata = (
  | AgentResponseMetadata     // role=assistant, no type
  | AgentStepMetadata         // type="ui_meta_update" (optional)
  | UiMetaMessageMetadata     // type="ui_meta_update" (required), ui_meta required
  | BaseMessageMetadata
) & Record<string, unknown>;
```

Use the `isUiMetaUpdate()` type guard to narrow to `UiMetaMessageMetadata` before accessing `ui_meta`.

### Legacy UI Meta aliases (`messages.ts`)

These four interfaces are local copies of the corresponding `websocket.ts` payloads, kept for backward compatibility. They have slightly narrower `status` unions (no `"warning"` value) and no grouping fields (`step_id`, `phase_id`, etc.).

| Alias | Corresponds to |
|-------|---------------|
| `DeepResearchUiMeta` | `DeepResearchPayload` (no grouping fields) |
| `CuaUiMeta` | `CuaPayload` (no grouping fields) |
| `McpUiMeta` | `McpPayload` (no grouping fields) |
| `AskHumanUiMeta` | `AskHumanPayload` (inline choice shape, no grouping fields) |

`AnyUiMeta` is the four-member union of these legacy aliases. New code should use `UiMetaUpdatePayload` from `websocket.ts` instead.

Type guard helpers exported from `messages.ts`: `isUiMetaUpdate()`, `isDeepResearchMeta()`, `isCuaMeta()`, `isMcpMeta()`, `isAskHumanMeta()`.

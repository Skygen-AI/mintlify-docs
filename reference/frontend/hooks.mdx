## Auth

### useAuth

**File:** `hooks/useAuth.ts`  
**Purpose:** Convenience wrapper around `authSlice` in the Zustand store. Exposes all auth state and actions as a single flat object.

### Returns
<ResponseField name="user" type="User | null">Authenticated user object, or `null` if not logged in.</ResponseField>
<ResponseField name="isLoading" type="boolean">True during session restore or login.</ResponseField>
<ResponseField name="isAuthenticated" type="boolean">Whether a valid session exists. Optimistically true if `auth_token` is in localStorage.</ResponseField>
<ResponseField name="needsOTPSetup" type="boolean">True when 2FA setup is required after login.</ResponseField>
<ResponseField name="login" type="(email: string, password: string) => Promise<void>">Log in with email and password.</ResponseField>
<ResponseField name="signup" type="(email: string, password: string, displayName?: string, meta?: Record<string, unknown>) => Promise<void>">Register a new account. Optional `displayName` and `meta` for platform-specific metadata.</ResponseField>
<ResponseField name="logout" type="() => Promise<void>">Disconnect wsClient, POST /auth/logout, reset all user state across all slices.</ResponseField>
<ResponseField name="refreshUser" type="() => Promise<void>">Re-fetch user profile from `/users/me`.</ResponseField>
<ResponseField name="restoreSession" type="() => Promise<boolean>">Check stored tokens, upgrade legacy token to cookie, reconnect wsClient. Returns `true` if session was restored.</ResponseField>
<ResponseField name="setNeedsOTPSetup" type="(v: boolean) => void">Manually set the 2FA setup flag.</ResponseField>

### Example
```tsx
const { user, isAuthenticated, logout } = useAuth();
if (!isAuthenticated) return <Navigate to="/login" />;
```

**Used by:** `LoginPage`, `RegisterPage`, `AuthProvider`, `ProtectedRoute`

### useChatWebSocket

**File:** `hooks/useChatWebSocket.ts`  
**Purpose:** Manages all Socket.IO event subscriptions for a chat session.

**Subscribed events:**
- `display_message` â€” unified event protocol: handles `assistant_message`, `user_message`, `cancelled_message`, `paused_message`, `error_message`, `ask_human_message`, `status_update`, `step_progress`, `plan_update`, `mode_progress`, `attachment_ready`, `billing_notice`
- `new_message` â€” legacy event (deduped against `display_message` via `displayHandledIdsRef`)
- `task_progress` â€” drives `updateTaskProgress` in liveStateSlice
- `agent_typing` â€” drives `setTyping`; sealed sessions reject `is_typing: true`
- `device_status` â€” drives `updateDeviceStatus`
- `reconnected` â€” reloads messages for the current session (debounced to 5s)
- `ui_meta_update` â€” drives agent steps and plan updates; contains legacy MCP connect prompt fallback (disabled)
- `agent_error` â€” finalizes session with `error` status, inserts system error message
- `attachments_ready` â€” updates message attachments with S3 keys
- `message_deleted` â€” removes message from store (cross-device delete sync)

**Safety watchdog:** A `setInterval` at 15-second ticks. If a task is active for more than 120 seconds without any agent event (`lastAgentEventRef`), finalizes the session with `error`. Also REST-polls `GET /agent/task/{id}/status` every tick to catch stale `isActive=true` from page reloads before any events arrive.

### Parameters
<ParamField path="sessionId" type="string | null" required>The session to subscribe to.</ParamField>

### Returns
<ResponseField name="markAgentEvent" type="() => void">Resets the watchdog timer to now. Call when a user action proves the agent is alive (e.g., after `sendMessage`).</ResponseField>

**Used by:** `useChat`

### usePlanAccess

**File:** `hooks/usePlanAccess.ts`  
**Purpose:** Centralized feature access checker. Checks `cloud_pc` and `scheduled_tasks` via `status.cloud_pc_enabled` / `status.scheduled_tasks_enabled` flags and finds the cheapest upgrade plan when access is denied.

```typescript
export type FeatureKey = "cloud_pc" | "scheduled_tasks";

export interface PlanAccessResult {
  allowed: boolean;
  upgradePlanId?: string;
  planName?: string;
}
```

### Returns
<ResponseField name="checkFeatureAccess" type="(feature: FeatureKey) => PlanAccessResult">Check access and return `{ allowed, upgradePlanId?, planName? }`. When denied, finds the cheapest plan above the current `sort_order` that has the feature enabled in `features_config`.</ResponseField>
<ResponseField name="status" type="BillingStatus | null">Current billing status (direct store reference).</ResponseField>
<ResponseField name="modelDisplayName" type="string | null">Current plan's model display name from `status.model_display_name`.</ResponseField>
<ResponseField name="allowedModelProfiles" type="string[] | null">Model profiles available to this plan from `status.allowed_model_profiles`.</ResponseField>
<ResponseField name="activeModelProfile" type="string | null">Currently selected model profile from `status.active_model_profile`.</ResponseField>
<ResponseField name="modelProfilesMeta" type="ModelProfileMeta[] | null">Display metadata for model profiles from `status.model_profiles_meta`.</ResponseField>
<ResponseField name="setModelProfile" type="(profile: string) => Promise<void>">Switch the active model profile via `billingSlice.setModelProfile`.</ResponseField>

### Example
```tsx
const { checkFeatureAccess } = usePlanAccess();
const result = checkFeatureAccess("cloud_pc");
if (!result.allowed) {
  navigate(`/subscriptions?plan_id=${result.upgradePlanId}`);
}
```

**Used by:** `HomeUpdated`, `MainLayout`, `ChooseDeviceModal`

### useCloudPCEvents

**Returns:** `void` (no return value)

**Used by:** `MainLayout`

### useCloudPCPolling

## Layout / Init

### useMainLayoutInit

**File:** `hooks/useMainLayoutInit.ts`  
**Purpose:** Encapsulates all initialization effects that run when `MainLayout` mounts.

**Effects performed:**
1. Load sessions â†’ reconcile per-session task status (REST poll up to 8 most-recently-updated sessions, concurrency 4, 30-minute window) to restore sidebar spinner after hard refresh
2. Zombie-spinner watchdog: every 60s, REST-polls all sessions where `taskStatusBySession[id].isActive === true`; clears stale entries
3. Load devices
4. Restore Cloud PC state (REST reconcile)
5. Fetch Cloud PC list and bind active Cloud PC to `activeSessionId`
6. Load billing status, pricing, plans
7. Auto-refresh billing status every 30s (clears interval if 401, triggers logout via `logoutRef`)
8. Check onboarding status
9. Subscribe to `scheduled_session_created` WebSocket event

### Parameters
<ParamField path="isAuthenticated" type="boolean" required>Whether the user is logged in. Guards all effects.</ParamField>
<ParamField path="activeSessionId" type="string | null" required>The currently open session ID for Cloud PC binding reconcile.</ParamField>

### Returns
<ResponseField name="status" type="BillingStatus | null">Current billing status (from `useBilling`).</ResponseField>
<ResponseField name="logoutRef" type="MutableRefObject<(() => void) | null>">The layout must populate this ref with its logout callback so the billing auto-refresh can call logout on 401.</ResponseField>

**Used by:** `MainLayout`

### useOAuthPolling

**Purpose:** Polls `connectorsApi.getConnected()` with exponential backoff until the given provider shows `is_connected` or `status === 'ACTIVE'/'CONNECTED'`, or the max polling time elapses. Active only when `provider` is non-null; cleans up on unmount or provider change.

  intervals?: number[];   // Default: [2000, 4000, 8000, 16000, 30000]

}

<ParamField path="options.onTimeout" type="(provider: string) => void">Called if polling times out without connection.</ParamField>
<ParamField path="options.intervals" type="number[]">Custom backoff intervals in ms. Default: `[2000, 4000, 8000, 16000, 30000]`. Cycles through and caps at the last value.</ParamField>

# Frontend Components Reference

All components live in `ui-shared-tauri/src/components/`. This reference covers components used in two or more places (the most reusable subset). Page-specific leaf components (helpdesk sub-views, message card types, file preview types) are grouped at the end.

---

## Layout

### MainLayout

**File:** `components/layout/MainLayout.tsx`  
**Purpose:** Root authenticated layout. Renders the sidebar, header, global modals, Cloud Desktop panel, file preview panel, and all layout-level dialogs.

All protected routes are wrapped by `MainLayout` via React Router's `<Outlet />` pattern. It mounts `useMainLayoutInit`, `useCloudPCEvents`, and subscribes to global WebSocket events.

**Used in:** `App.tsx` (wraps all protected routes)

---

### AppHeader

**File:** `components/layout/AppHeader.tsx`  
**Purpose:** Top navigation bar. A thin wrapper that renders `HeaderAccountControls` with billing-aware token balance and notification count pulled from global hooks.

### Props
<ParamField path="onNotificationClick" type="() => void">Callback when the notification bell is clicked.</ParamField>
<ParamField path="onAvatarClick" type="() => void">Callback when the user avatar is clicked.</ParamField>
<ParamField path="onAvatarRef" type="(ref: HTMLElement | null) => void">Ref callback for dropdown positioning.</ParamField>
<ParamField path="isDropdownOpen" type="boolean">Whether the profile dropdown is open.</ParamField>
<ParamField path="className" type="string">Additional CSS classes.</ParamField>

**Used in:** `MainLayout`

---

### HeaderAccountControls

**File:** `components/layout/HeaderAccountControls.tsx`  
**Purpose:** Composite header widget combining `NotificationButton`, `TokensDisplay`, and `UserAvatar`. Platform-agnostic — all actions are handled via callbacks.

### Props
<ParamField path="tokens" type="number">Token balance to display.</ParamField>
<ParamField path="isTokensLoading" type="boolean">Shows a skeleton while balance is fetching.</ParamField>
<ParamField path="showTokens" type="boolean">Whether to render the token balance widget. Default: `true`.</ParamField>
<ParamField path="unreadNotifications" type="number">Unread notification count for the bell badge.</ParamField>
<ParamField path="onNotificationClick" type="() => void">Called when the notification bell is clicked.</ParamField>
<ParamField path="onAvatarClick" type="() => void">Called when the user avatar is clicked.</ParamField>
<ParamField path="onAvatarRef" type="(ref: HTMLElement | null) => void">Ref callback for dropdown anchor positioning.</ParamField>
<ParamField path="isDropdownOpen" type="boolean">Whether the profile dropdown is currently open.</ParamField>
<ParamField path="className" type="string">Additional CSS classes.</ParamField>

**Used in:** `AppHeader`

## UI Primitives

**File:** `components/ui/Button.tsx`  
**Purpose:** Base button component with variants and sizes. Extends `React.ButtonHTMLAttributes`.

### Props
<ParamField path="variant" type="'default' | 'destructive' | 'outline' | 'secondary' | 'ghost' | 'link'">Visual style. Default: `default`.</ParamField>

```tsx

</Button>
```

**Used in:** Throughout — 53+ files reference this component.

### Dialog / DialogContent / DialogHeader / DialogTitle / DialogFooter

**File:** `components/ui/Dialog.tsx`  
**Purpose:** Accessible modal dialog primitives. `Dialog` handles backdrop and close-on-click. `DialogContent` provides the centered card.

### Props (Dialog)
<ParamField path="open" type="boolean" required>Controls visibility.</ParamField>
<ParamField path="onOpenChange" type="(open: boolean) => void" required>Called when backdrop is clicked.</ParamField>
<ParamField path="children" type="ReactNode" required>Dialog content.</ParamField>

### Usage
```tsx
<Dialog open={isOpen} onOpenChange={setIsOpen}>
  <DialogContent>
    <DialogHeader><DialogTitle>Confirm</DialogTitle></DialogHeader>
    <p>Are you sure?</p>
    <DialogFooter><Button onClick={() => setIsOpen(false)}>OK</Button></DialogFooter>
  </DialogContent>
</Dialog>
```

**Used in:** 27+ files

This guide walks you from a fresh account to a running agent in under five minutes. You will create a user, obtain an access token, register a device, start a chat session, and send a message.

## Prerequisites

- An email address for account verification
- One of: the Skygen desktop app installed on your machine **or** access to create a Cloud PC

## 1. Create an account

Sign up through the web dashboard or via the API.

<CodeGroup>
```bash cURL
curl -X POST https://api.skygen.ai/security/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "your-strong-password"
  }'
```

```python Python
import httpx

response = httpx.post(
    "https://api.skygen.ai/security/signup",
    json={"email": "you@example.com", "password": "your-strong-password"},
)
response.raise_for_status()
```
</CodeGroup>

Check your inbox for a verification email and confirm the address before continuing.

## 2. Log in and get an access token

<CodeGroup>
```bash cURL
curl -X POST https://api.skygen.ai/security/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=you@example.com&password=your-strong-password"
```
</CodeGroup>

The response contains an `access_token` (JWT). Use it as `Authorization: Bearer <token>` in every authenticated request. See [Authentication](/guides/authentication) for the refresh flow.

## 3. Register a device

Devices are the machines your agent will run on. They can be real (your laptop via the desktop app) or virtual (a Cloud PC or E2B sandbox).

<CodeGroup>
```bash cURL
curl -X POST https://api.skygen.ai/devices \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My MacBook",
    "platform": "macos",
    "device_type": "real_device"
  }'
```
</CodeGroup>

The response includes a `device_token_kid` which the desktop app uses to authenticate its websocket connection. See [Devices](/concepts/devices) for the full lifecycle.

## 4. Start a chat session

<CodeGroup>
```bash cURL
curl -X POST https://api.skygen.ai/sessions \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "DEVICE_UUID_FROM_STEP_3",
    "title": "First run"
  }'
```
</CodeGroup>

Sessions scope agent memory and tool access. Learn more in [Chat sessions](/concepts/chat-sessions).

## 5. Send a message

<CodeGroup>
```bash cURL
curl -X POST https://api.skygen.ai/messages \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "SESSION_UUID_FROM_STEP_4",
    "role": "user",
    "content": "Take a screenshot of my desktop and describe what you see."
  }'
```
</CodeGroup>

The agent processes the message, plans actions, and executes them on the device. Replies stream back through the same session. For real-time updates subscribe to the `/user` Socket.IO namespace.

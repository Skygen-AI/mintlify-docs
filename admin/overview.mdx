## Overview

The Skygen admin panel is a set of protected API endpoints under `/admin/api/`. All admin endpoints require the `verify_admin` dependency which checks that the authenticated user has `is_admin = True`. The admin credentials are bootstrapped via `ADMIN_EMAIL` and `ADMIN_PASSWORD` environment variables during the init container migration step.

---

## Authentication

Admin endpoints use the same JWT authentication as user endpoints. Obtain a token by logging in with admin credentials:

```http
POST /auth/login
Content-Type: application/json

{"email": "admin@skygen.ai", "password": "..."}
```

Pass the returned `access_token` as `Authorization: Bearer {token}` on all admin requests.

## User management

### List users

```http
GET /admin/api/users?skip=0&limit=50&search=alice&is_active=true
Authorization: Bearer {admin_token}
```

<ParamField query="skip" type="integer">
  Pagination offset. Default: 0.
</ParamField>
<ParamField query="limit" type="integer">
  Page size. Max: 100. Default: 50.
</ParamField>
<ParamField query="search" type="string">
  Search by email or display name (case-insensitive).
</ParamField>
<ParamField query="is_active" type="boolean">
  Filter by active status.
</ParamField>
<ParamField query="is_admin" type="boolean">
  Filter admin users.
</ParamField>
<ParamField query="has_subscription" type="boolean">
  Filter users with active subscriptions.
</ParamField>
<ParamField query="sort_by" type="string">
  Column to sort by. Default: `created_at`.
</ParamField>
<ParamField query="sort_desc" type="boolean">
  Sort descending. Default: `true`.
</ParamField>

Response includes user data, active subscription plan name, and token balance for each user.

### Create a user

```http
POST /admin/api/users
Authorization: Bearer {admin_token}

{"email": "newuser@example.com", "password": "...", "is_admin": false}
```

### Get user details

```http
GET /admin/api/users/{user_id}
Authorization: Bearer {admin_token}
```

Returns user profile, subscription, token balance, session count, and last activity.

### Update a user

```http
PATCH /admin/api/users/{user_id}
Authorization: Bearer {admin_token}

{"is_active": false, "is_admin": false}
```

### Delete a user

```http
DELETE /admin/api/users/{user_id}
Authorization: Bearer {admin_token}
```

Soft-deletes the user and all associated sessions.

### Reset a user's password

```http
POST /admin/api/users/{user_id}/reset-password
Authorization: Bearer {admin_token}

{"new_password": "secure-password-123"}
```

## Billing management

### Grant tokens to a user

```http
POST /admin/api/billing/grant-tokens
Authorization: Bearer {admin_token}

{
  "user_id": "user-uuid",
  "amount": 1000,
  "description": "Compensation for service disruption"
}
```

Creates an `ADMIN_GRANT` transaction and increases the user's balance.

### Override subscription

```http
POST /admin/api/billing/override-subscription
Authorization: Bearer {admin_token}

{
  "user_id": "user-uuid",
  "plan_id": "pro-plan-uuid",
  "expires_at": "2026-06-01T00:00:00Z"
}
```

Grants a plan subscription outside of the normal Stripe payment flow.

### Manage subscription plans

```http
GET  /admin/api/plans
POST /admin/api/plans
PUT  /admin/api/plans/{plan_id}
DELETE /admin/api/plans/{plan_id}
```

Manage `SubscriptionPlan` records. Changes take effect for new subscriptions; existing subscribers are not automatically migrated.

## Model profile management

```http
GET  /admin/api/model-profiles
POST /admin/api/model-profiles
PUT  /admin/api/model-profiles/{key}
```

Controls display names, descriptions, and badges shown in the model profile selector. The `key` must match a `MODEL_PROFILES` entry in `Agent/config.py`.

---

## Invite codes

```http
GET  /admin/api/invites
POST /admin/api/invites
DELETE /admin/api/invites/{invite_id}
```

Generates one-time invite codes for beta access. Codes are linked to a plan on redemption.

---

## Session logs

```http
GET /admin/api/session-logs?user_id=...&date_from=...&limit=50
Authorization: Bearer {admin_token}
```

Returns all chat sessions with message count, token usage, agent type, and status. Useful for debugging user-reported issues.

---

## System settings

```http
GET  /admin/api/system-settings
PATCH /admin/api/system-settings

{"maintenance_mode": false, "registration_enabled": true}
```

Controls global feature flags without requiring a deployment.

---

## Analytics

```http
GET /admin/api/analytics/overview
GET /admin/api/analytics/users?period=7d
GET /admin/api/analytics/sessions?period=30d
```

Returns aggregate user activity, session counts, token consumption, and revenue metrics. Periods: `1d`, `7d`, `30d`, `90d`.

---

## Message ratings

```http
GET /admin/api/message-ratings
```

Returns user thumbs-up/thumbs-down ratings on agent messages. Used to track response quality over time.

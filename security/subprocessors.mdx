Skygen uses a small number of vetted third-party services to deliver its platform. This page lists every service that processes user data, categorizes it, and describes the data flow. It is the single source of truth for compliance reviews (SOC 2, data-processing agreements) and for users who want to know who sees what.

<Note>
We update this page whenever a subprocessor is added, removed, or changes role. Material changes are announced at least **30 days in advance** for existing customers unless the change is required to remediate a security issue.
</Note>

## Core infrastructure

### Amazon Web Services (AWS) {#aws}

- **Role:** Primary cloud provider. Hosts all application compute, databases, queues, caches, object storage, email relay, certificates, secrets, and logs.
- **Data seen:** Everything. Account details, chat sessions, messages, devices, Cloud PC snapshots, attachments, logs, metrics.
- **Regions used:**
  - `us-east-1` (N. Virginia) — EKS clusters, RDS, ElastiCache, S3 primary buckets, SES, Secrets Manager, most compute.
  - `eu-north-1` (Stockholm) — legacy ECS Fargate services (webapp, landing, admin panel), Lightsail support host. Migrating to `us-east-1`.
- **Encryption:** Data encrypted at rest (EBS, RDS, S3 default SSE-S3; Secrets Manager KMS). In transit via TLS 1.2+.
- **Authentication to Skygen workloads:** IAM Roles for Service Accounts (IRSA) — keyless auth from EKS pods, no static access keys in Kubernetes.
- **DPA:** [aws.amazon.com/compliance/data-privacy-faq](https://aws.amazon.com/compliance/data-privacy-faq/)
- **Compliance:** SOC 1/2/3, ISO 27001, PCI DSS, HIPAA, GDPR-ready.

For the full inventory of AWS services, see [AWS stack inventory](/deployment/aws-inventory).

## Payments

### Stripe {#stripe}

- **Role:** Subscription billing ($14.99/mo), metered overage (pay-per-token), webhook-driven payment confirmation.
- **Data seen:** Email address, billing country, card information. **Skygen never touches raw card data** — the frontend uses Stripe.js (via `@stripe/react-stripe-js`) to tokenize cards client-side.
- **Data written back to us:** Customer ID, subscription status, invoice metadata, payment method last-4. Stored in `users.stripe_customer_id` and related billing tables.
- **Webhooks:** Stripe calls `POST /billing/webhook` with HMAC-signed events (`invoice.paid`, `customer.subscription.updated`, etc.).
- **Regions:** Stripe processes data in the US and EU.
- **DPA:** [stripe.com/legal/dpa](https://stripe.com/legal/dpa)
- **Compliance:** PCI DSS Level 1, SOC 1/2, ISO 27001.

## AI / LLM providers

Skygen routes LLM calls through `Agent/Utilities/LLM_Providers/ProviderFactory.py`. Most models are accessed via **OpenRouter** (a reseller/aggregator); a few go direct for latency or feature reasons.

### OpenRouter {#openrouter}

- **Role:** Default LLM reseller for the broadest set of models (Anthropic Claude, OpenAI GPT, Meta Llama, Mistral, Google Gemini in some flows, etc.).
- **Endpoint:** `https://openrouter.ai/api/v1`
- **Data seen:** The full prompt, which may include:
  - The user's message
  - Recent chat history (for context)
  - File contents the user attached (images, documents parsed to text)
  - Screenshots from Cloud PC / desktop agents
  - Tool call parameters (e.g., MCP calls the model is directing)
- **Downstream models:** OpenRouter forwards traffic to the underlying model provider (Anthropic, OpenAI, Google, etc.). Each provider may have its own data-retention stance. Skygen's OpenRouter account is configured with provider-level privacy preferences where available.
- **DPA:** [openrouter.ai/privacy](https://openrouter.ai/privacy)
- **Compliance:** SOC 2 in progress (public roadmap).

### OpenAI {#openai}

- **Role:** Direct calls for specific models where OpenRouter adds latency or misses features. Used inside the Agent's reasoning and some tool-calling flows.
- **SDK:** `openai==2.8.1` (Agent only; not in Backend).
- **Data seen:** Prompt text, file contents passed into the request, model name.
- **Data retention:** Skygen's API access uses the OpenAI API terms (zero data retention available for paid tier when opted in).
- **DPA:** [openai.com/policies/business-terms](https://openai.com/policies/business-terms/)
- **Compliance:** SOC 2 Type 2, CCPA, GDPR.

### Google (Gemini) {#google-gemini}

- **Role:** Embedding generation for the Knowledge Base (`GEMINI_API_KEY`, `gemini-embedding` model family) plus direct Gemini inference for some agent flows.
- **SDK:** `google-generativeai` / `google-genai`.
- **Data seen:** Text to embed (knowledge base documents, user queries), or full prompts for inference.
- **DPA:** [cloud.google.com/terms/data-processing-addendum](https://cloud.google.com/terms/data-processing-addendum)
- **Compliance:** SOC 1/2/3, ISO 27001.

### Cerebras {#cerebras}

- **Role:** Low-latency inference for specific models (`llama-3.3-70b`, `qwen-3-235b-a22b-instruct-2507`). Used sparingly when latency matters.
- **Endpoint:** `https://api.cerebras.ai/v1`
- **Data seen:** Prompt text, model name.
- **DPA:** [cerebras.ai/policies](https://cerebras.ai/policies)

### Groq {#groq}

- **Role:** Low-latency inference for select open-weights models (matcher: model name starts with `groq/`).
- **Endpoint:** `https://api.groq.com/openai/v1`
- **Data seen:** Prompt text, model name.
- **DPA:** [groq.com/privacy-policy](https://groq.com/privacy-policy/)

### RunPod — Faster Whisper {#runpod}

- **Role:** Voice transcription. When a user sends audio, the agent forwards it to RunPod's `worker-faster_whisper` serverless endpoint.
- **Configuration:** `RUNPOD_API_KEY`, `RUNPOD_ENDPOINT_ID=worker-faster_whisper`, model `turbo`.
- **Data seen:** Raw audio bytes (time-bounded — deleted from RunPod after transcription per their default policy).
- **DPA:** [runpod.io/legal/privacy-policy](https://www.runpod.io/legal/privacy-policy)

## Sandboxed execution

### E2B {#e2b}

- **Role:** Sandboxed Cloud PC and code-interpreter environments. When a user starts Cloud PC or the agent runs code, it spins up an isolated VM on E2B.
- **SDK:** `e2b==2.15.1` and `e2b-desktop==2.3.0` (Backend), `e2b-code-interpreter==2.3.0` (Agent).
- **Data seen:**
  - Screenshots of the remote desktop (streamed to the user).
  - Files the user uploads into the sandbox.
  - Shell commands the agent runs inside the sandbox.
  - Browser session cookies if the user logs into websites inside Cloud PC.
- **Persistence:** Per-user Cloud PC backup snapshots are stored back in our own S3 bucket (`skygen-cloudpc-profiles-prod`). E2B holds the live sandbox only while it runs.
- **Region:** E2B's default region is the EU; confirm per workload.
- **DPA:** [e2b.dev/dpa](https://e2b.dev/dpa) (on request).

## Integration brokers

### Composio {#composio}

- **Role:** Managed MCP integrations — handles the OAuth flow, token refresh, and tool-call proxying for a wide catalogue of third-party apps (Gmail, Calendar, Drive, Docs, Sheets, Contacts, YouTube, Zoom, Slack, and more).
- **SDK:** `composio==0.11.x` in Backend (`app/services/composio_*.py`).
- **Data seen:**
  - OAuth access tokens for every app the user connects.
  - Request and response payloads for every tool call routed through Composio (e.g., "send an email," "read a calendar event").
- **Webhook:** Composio calls `POST /mcp/webhooks` with HMAC-signed events; secret `COMPOSIO_WEBHOOK_SECRET`.
- **DPA:** [composio.dev/legal/dpa](https://composio.dev/legal/dpa) (request from Composio).
- **Status:** `COMPOSIO_ENABLED` is a feature flag. When disabled, connector calls return a service-upgrading error.

## Email

### Amazon SES {#ses}

- **Role:** Primary transactional and lifecycle email sender — verification, password reset, helpdesk replies, inactive-user nudges, receipts.
- **Region:** `us-east-1` (production access, 50k/day quota, 14/s send rate).
- **Verified identity:** `skygen.ai` with DKIM + DMARC.
- **Data seen:** Recipient email, sender name, subject, body.
- **Covered under:** AWS DPA (see [AWS](#aws)).

### Resend (legacy, being retired) {#resend}

- **Role:** Historical email sender for lifecycle campaigns and contact audiences.
- **Current use:** Only `Backend/scripts/migrate_resend_contacts.py` references Resend — it's a one-shot migration tool to move contact lists from Resend audiences into our `user_email_segments` table. Resend is no longer in the runtime send path.
- **Timeline:** Resend API key and DNS records scheduled for removal once contact migration is complete.

## Observability

### Sentry {#sentry}

- **Role:** Error monitoring and performance tracing for the Agent service.
- **SDK:** `sentry-sdk==2.43.0` in Agent.
- **Data seen:** Python stack traces, breadcrumbs, request metadata (URL, method, status). User PII is scrubbed by Sentry-side filters.
- **Backend status:** Sentry is **not** initialized in Backend (as of April 2026). Backend errors go to Loki only.
- **DPA:** [sentry.io/legal/dpa](https://sentry.io/legal/dpa/)
- **Compliance:** SOC 2 Type 2, ISO 27001, GDPR.

## Analytics

Analytics subprocessors are **opt-in at the environment level** (`ANALYTICS_ENABLED=true` on Backend, `VITE_GTM_ID`/`VITE_GOOGLE_ADS_ID` on WebApp). When disabled, no events are sent.

### Google Analytics 4 (GA4) {#ga4}

- **Role:** Pageview and event tracking for marketing and product analytics.
- **Configuration:** `GA4_MEASUREMENT_ID` (Backend, for server-side Measurement Protocol), client-side via GTM on WebApp.
- **Data seen:** Anonymized device ID, pageview URL, event name, user agent. No email or user content.
- **DPA:** Google Ads Data Processing Terms.

### Google Tag Manager (GTM) {#gtm}

- **Role:** Client-side tag container — loads GA4, Google Ads pixel, etc. dynamically from the container you configure.
- **Loaded in:** `WebApp/src/main.tsx` (`https://www.googletagmanager.com/gtm.js?id=${GTM_ID}`) when `VITE_GTM_ID` is present.

### Google Ads {#google-ads}

- **Role:** Conversion tracking for advertising campaigns.
- **Loaded in:** `WebApp/src/main.tsx` (`https://www.googletagmanager.com/gtag/js?id=${GOOGLE_ADS_ID}`) when `VITE_GOOGLE_ADS_ID` is present.

### Yandex Metrica {#yandex-metrica}

- **Role:** Product analytics for the Russian-speaking audience.
- **Configuration:** `YANDEX_COUNTER_ID`.
- **Data seen:** Pageviews, anonymous device ID, session recording where enabled.
- **DPA:** [yandex.com/legal/confidential](https://yandex.com/legal/confidential/)

### Amplitude {#amplitude}

- **Role:** Server-side event analytics for product KPIs (activation, retention, funnel steps).
- **Configuration:** `AMPLITUDE_API_KEY`.
- **Data seen:** Event name + properties. Our event schemas exclude direct PII — user IDs are hashed where sent.
- **DPA:** [amplitude.com/legal/dpa](https://amplitude.com/legal/dpa)

## OAuth identity providers

Users sign in with one of these providers. Skygen receives only what the provider releases for the chosen scope (typically: email, profile name, provider-side user ID).

| Provider | Env vars | Scopes requested |
|----------|----------|------------------|
| Google | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, plus `GOOGLE_ANDROID_CLIENT_ID`, `GOOGLE_IOS_CLIENT_ID` for mobile | `openid email profile` |
| GitHub | `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` | `user:email` |
| Microsoft | `MICROSOFT_CLIENT_ID`, `MICROSOFT_CLIENT_SECRET` (MSAL) | `User.Read` |
| Apple | (via iOS app) | email, name (first-time only) |

Sign-in flows are implemented in `Backend/app/routers/security/` and are the **only** path through which identity-provider data enters our system. We do not ingest address books, social graphs, or extended profile data.

## MCP connected apps (user-authorized)

When a user connects a third-party app via MCP (either directly or through Composio), the app becomes a subprocessor **for that user**. The categories we support include:

- **Google Workspace**: Gmail, Calendar, Drive, Docs, Sheets, Contacts, YouTube — per-integration OAuth pairs (`GMAIL_CLIENT_ID`, `CALENDAR_CLIENT_ID`, etc.)
- **Telegram**: user-connected personal accounts via Telethon (`TELEGRAM_API_ID`, `TELEGRAM_API_HASH`), plus bot API
- **Slack**: via Composio (`SLACK_CLIENT_ID`, `SLACK_CLIENT_SECRET`, `SLACK_SIGNING_SECRET`)
- **Zoom**: via Composio
- **Plus the full Composio catalogue** (200+ apps) — see [MCP integrations](/concepts/mcp-integrations) for the current list.

**What they see:** Only the data scopes the user explicitly authorizes during OAuth consent. Revoking access in the user's connected-apps page revokes the token immediately.

**Storage:** OAuth tokens are encrypted at rest in Postgres using the `MCP_CRYPTO_KEY` AES key, rotated separately from application secrets.

## Data flow summary

What actually leaves the Skygen AWS account:

1. **Every agent run** → LLM provider (OpenRouter primary, OpenAI/Gemini/Cerebras/Groq for specific models)
2. **Every transcription request** → RunPod
3. **Every Cloud PC / code execution** → E2B sandbox
4. **Every MCP tool call** → Composio (or the app's own API if not brokered)
5. **Every payment event** → Stripe
6. **Every transactional email** → Amazon SES
7. **Every agent error** → Sentry
8. **Every pageview (if analytics enabled)** → GA4, GTM, Google Ads, Yandex Metrica
9. **Every KPI event** → Amplitude

Everything else (sessions, messages, file blobs, user accounts, billing state, vector embeddings) stays inside the Skygen AWS account.

## User controls

Users can limit data flow:

- **Turn off analytics:** Admin → Feature flags → `ANALYTICS_ENABLED=false`. No events sent to GA4/Amplitude/Yandex.
- **Disconnect an MCP app:** Settings → Integrations → Disconnect. Revokes the OAuth token, deletes cached credentials.
- **Pause Cloud PC:** Settings → Cloud PC → Pause. Releases the E2B sandbox and uploads a backup snapshot to our S3.
- **Delete account:** Settings → Account → Delete. Triggers soft-delete (30-day hold) then hard-delete of rows, files, and backup snapshots. Stripe customer is also cancelled; the subprocessor retains its own records per their retention policy.

## Subprocessor change notifications

When we add, remove, or change a subprocessor:

1. We update this page with the new line item and effective date.
2. We notify existing customers by email at least **30 days before** the change for non-security-critical additions.
3. Security-critical changes (removing a provider for compliance reasons) may happen faster, with prompt notification after.

Subscribe to subprocessor change notifications by emailing `security@skygen.ai`.

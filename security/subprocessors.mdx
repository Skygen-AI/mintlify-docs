## Overview

Skygen uses the following third-party subprocessors to provide its service. When you use Skygen, your data may be processed by these services. This list is maintained as part of our privacy and compliance commitments.

---

## Infrastructure subprocessors

| Subprocessor | Purpose | Data processed | Region |
|-------------|---------|----------------|--------|
| **Amazon Web Services (AWS)** | Cloud infrastructure (EKS, RDS, ElastiCache, S3, Secrets Manager, Amazon MQ) | All application data, session data, files, secrets | us-east-1 (primary), eu-north-1 (legacy) |
| **AWS ECR** | Container image registry | Container images (no user data) | us-east-1 |
| **AWS CloudWatch** | Infrastructure log collection | Kubernetes control plane logs | us-east-1 |

## AI / LLM providers

| Subprocessor | Purpose | Data processed | Notes |
|-------------|---------|----------------|-------|
| **Anthropic** | Claude LLM API (agent reasoning) | User prompts, task context, session history | Optional; used when `claude-*` models are selected |
| **OpenAI** | GPT LLM API (agent reasoning) | User prompts, task context, session history | Optional; used when `gpt-*` models are selected |
| **Google (Gemini)** | Gemini LLM API (agent reasoning, safety classifier) | User prompts, task context | Used as default and safety judge model |
| **OpenRouter** | LLM routing API (Qwen, Mistral, and other models) | User prompts, task context | Routes to various model providers |
| **Cerebras** | Fast inference LLM API | User prompts, task context | Optional; high-throughput inference |

<Note>
  LLM providers receive the content of your messages and task context to generate responses. They do not receive your authentication credentials, billing information, or device details.
</Note>

## Data residency

All primary data (PostgreSQL, Redis, S3, Secrets Manager) resides in **AWS us-east-1**. The legacy ECS frontend deployment in eu-north-1 does not store persistent data.

LLM provider data processing occurs in the respective provider's infrastructure. Composio processes data in their own cloud environment. E2B sandboxes are provisioned in E2B's cloud.

## User data not shared with subprocessors

The following data types are never transmitted to any subprocessor:

- Raw passwords (hashed before storage, never transmitted)
- Stripe payment card numbers (handled entirely by Stripe's SDK; Skygen never receives raw card data)
- AWS Secrets Manager secrets (used server-side only; never in agent prompts or LLM calls)
- Other users' session content (strict per-user isolation enforced at API level)

---

## Opting out of analytics

Google Analytics is used only for the marketing landing page (anonymous page view tracking). The Skygen application itself (`app.skygen.ai`) does not include Google Analytics. Authenticated user sessions are not tracked by any third-party analytics service.

## Data flows by feature

Understanding which subprocessors receive data for each Skygen feature:

### Agent chat

A user message in a chat session may flow through:

1. **AWS** — stored in PostgreSQL (RDS) and cached in ElastiCache (Redis)
2. **Anthropic / OpenAI / Gemini / OpenRouter** — the message and relevant session history is sent to the configured LLM provider for agent reasoning
3. **Composio** — if the agent calls a connected integration tool (e.g., reading Gmail), the tool call payload passes through Composio
4. **E2B** — if the agent opens a Cloud PC, the task prompt and subsequent GUI instructions go to E2B's sandbox

### File uploads and screenshots

Uploaded files and agent-captured screenshots are stored in **AWS S3**. Files are accessible only via presigned URLs (time-limited, user-scoped). No LLM provider receives raw file bytes — only text extracted from files as needed by the agent.

### Audio transcription

Audio submitted to `POST /audio/transcribe` is forwarded to **RunPod** (Faster Whisper). The audio file is not retained by RunPod after transcription. Transcription output is stored in Skygen's database (AWS RDS).

### Billing

When a user subscribes, payment details are entered directly into **Stripe**'s hosted payment form. Skygen receives only a Stripe customer ID and subscription status — never raw card data.

### OAuth integrations

When connecting a service like Gmail or GitHub, the OAuth flow redirects through **Composio**. Composio holds the OAuth access/refresh tokens and executes tool calls on the user's behalf. Skygen does not store OAuth tokens directly for Composio-managed integrations.

## Self-hosted deployment considerations

Organizations deploying Skygen on their own infrastructure can eliminate or replace several subprocessors:

| Subprocessor | Self-hosted alternative |
|---|---|
| AWS RDS | Any PostgreSQL instance |
| AWS ElastiCache | Self-hosted Redis |
| AWS S3 | MinIO (S3-compatible; configured via `S3_ENDPOINT_URL`) |
| RunPod (transcription) | Local Faster Whisper instance; update `RUNPOD_AUDIO_ENDPOINT` |
| Composio | Direct OAuth implementation per toolkit (significant engineering work) |

LLM providers (Anthropic, OpenAI, Gemini, OpenRouter) cannot be eliminated if you use AI capabilities, but can be restricted to a single provider by configuring `DEFAULT_LLM_PROVIDER` and removing other provider API keys.

<Note>
Every savings figure below is anchored to a line item in the **March 2026** bill ŌĆö no theoretical pricing fantasies. Where we can't verify the reduction percentage, we say so.
</Note>

## TL;DR

- Current run rate: **~$2,200/mo (~$26k/yr)** including tax.

## Tier 1: Verified waste {#tier-1}

Fix without new information. Low risk.

### 1. ECR lifecycle policies ŌĆö **$47/mo, $564/yr** {#ecr-lifecycle}

**Current:** 7 repos, `MUTABLE` tags, no lifecycle policies. Top repos hold hundreds of images each:

- `skygen/backend`: 1,281 images, ~450 GB
- `skygen/backend-worker`: 760 images, ~500 GB
- `skygen/agent`: 585 images, ~450 GB

ECR is billed at `$0.10/GB-mo`. March cost: `$58.61`. Cost has doubled every two months as images accumulate.

**Fix:** add a lifecycle policy per repo keeping the 15 newest `prod` / 5 newest `dev` / 10 newest untagged.

```json
{
  "rules": [
    { "rulePriority": 1, "description": "Keep last 15 prod", "selection": { "tagStatus": "tagged", "tagPrefixList": ["prod"], "countType": "imageCountMoreThan", "countNumber": 15 }, "action": { "type": "expire" } },
    { "rulePriority": 2, "description": "Keep last 10 dev", "selection": { "tagStatus": "tagged", "tagPrefixList": ["dev"], "countType": "imageCountMoreThan", "countNumber": 10 }, "action": { "type": "expire" } },
    { "rulePriority": 3, "description": "Expire untagged after 7 days", "selection": { "tagStatus": "untagged", "countType": "sinceImagePushed", "countUnit": "days", "countNumber": 7 }, "action": { "type": "expire" } }
  ]
}
```

Apply per repo:

```bash
aws ecr put-lifecycle-policy --repository-name skygen/backend --lifecycle-policy-text file://lifecycle.json --region us-east-1
# ...repeat for each repo
```

Expected reduction: ~80 % of storage. March: `$58.61 ŌåÆ ~$12/mo = $46/mo saved`. Grows larger over time since current rate is climbing.

### 2. CloudWatch Logs retention ŌĆö **$5ŌĆō10/mo, $60ŌĆō120/yr** {#cloudwatch-retention}

**Current:** 3 EKS log groups hold 54 GB combined with retention = `never`.

**Fix:** set a retention policy on each:

```bash
for lg in /aws/eks/skygen-us-dev/cluster /aws/eks/skygen-us-prod/cluster /aws/eks/skygen-us-dev/containers; do
  aws logs put-retention-policy --log-group-name "$lg" --retention-in-days 14 --region us-east-1
done
```

Storage savings are modest (`$0.03/GB-mo` ├Ś 54 GB = $1.60/mo). The bigger win is not eating CloudWatch queries (`$0.005/GB scanned`) on stale data.

<Note>
Ingestion (`$0.50/GB`) is the main CloudWatch cost and is unchanged by retention ŌĆö logs are still emitted. To reduce ingestion, filter noisy log lines at source (EKS control plane logging can be scoped to `api,audit` instead of `all`).
</Note>

### 3. Karpenter public IP misconfiguration ŌĆö **$7/mo, $88/yr + risk** {#karpenter-public-ip}

**Current:** 2 Karpenter-provisioned dev nodes (`i-0fd1f9d800614bca2`, `i-0b585fc7cf6b49199`) have public IPv4 addresses. Karpenter workloads should live in private subnets routed through NAT.

**Why it matters:**
- $3.65/mo ├Ś 2 IPs direct cost.
- Larger concern: exposing dev pod-hosting nodes to the public internet. Security posture > cost here.

**Fix:** Review the Karpenter `EC2NodeClass` `subnetSelector`. Likely pointing at public subnets, should point at private.

```yaml
apiVersion: karpenter.k8s.aws/v1
kind: EC2NodeClass
metadata:
  name: default
spec:
  subnetSelectorTerms:
    - tags:
        karpenter.sh/discovery: skygen-us-dev
        kubernetes.io/role/internal-elb: "1"   # private subnets only
```

### 7. Dev Valkey cluster 3 ŌåÆ 1 node ŌĆö **$24/mo, $288/yr** {#elasticache-dev}

**Current:** `skygen-dev-kuber` runs 3 ├Ś `cache.t4g.small` nodes (`$0.017/hr ├Ś 3 ├Ś 720h = $36.72/mo`) with **replication disabled**. Three copies of the same data, no failover benefit.

**Fix:** resize to 1 node. Data loss during resize is acceptable in dev.

Savings: `2 ├Ś $0.017/hr ├Ś 720 = $24.48/mo`.

### Tier 1 total: **~$84/mo direct + $7/mo Karpenter + cleanup = ~$91/mo, ~$1,100/yr**

## Tier 2: Architectural decisions {#tier-2}

Larger savings but require team alignment before execution.

### 1. Complete ECS ŌåÆ EKS migration ŌĆö **~$70/mo net, ~$840/yr** {#ecs-migration}

**Current:** ECS Fargate in `eu-north-1` runs 9 services for legacy webapp / landing / adminpanel while EKS prod has a new `landing-ssr` deployment behind `ssr.skygen.ai` (8 days old). Mid-migration.

**Finish it:**
1. Point `skygen.ai` marketing at `landing-ssr` in EKS; retire `skygen-landing` / `landing-backend` on ECS.
2. Add `webapp` deployment to EKS prod; retire `webapp` service on ECS.
3. Add `adminpanel` deployment to EKS prod; retire `skygen-adminpanel-service`.
4. Delete `skygen-production` ECS cluster (eu-north-1).
5. Delete `database-landing-prod` / `database-landing-dev` RDS ŌĆö or migrate their data into `skygen-prod-db` / `skygen-dev-db1-use1-eks`.

**Savings:**
- ECS Fargate compute: **$93/mo eliminated**.
- Needs ~1 extra Karpenter node to absorb load in EKS: **-$22/mo**.
- Plus `database-landing-*` RDS removed: **$24/mo** (`$10 dev + $14 prod`).
- Net: **~$95/mo = ~$1,140/yr**.

**Also gets rid of:**
- Cross-region traffic between ECS (eu-north-1) and RDS in us-east-1.
- A second deployment pipeline (ECS task definitions, eu-north-1 networking, separate observability).

<Warning>
Before deleting ECS services: verify DNS for `skygen.ai`, `www.skygen.ai`, `admin.skygen.ai` is actually pointing to the new EKS ingresses, not the ECS task ALBs. `dig` queries from this audit returned CGNAT-style `198.18.x.x` addresses (Cloudflare tunnel?), so resolution depends on the DNS provider, not the AWS side.
</Warning>

### 3. Dev RDS consolidation ŌĆö **~$30/mo, ~$360/yr** {#rds-consolidation}

**Current dev RDS footprint:**
- `skygen-dev-db1-use1` (`db.t4g.micro`, ~$10/mo) ŌĆö older instance
- `skygen-dev-db1-use1-eks` (`db.t4g.medium`, ~$24/mo) ŌĆö in-use by EKS dev
- `database-landing-dev` (`db.t4g.micro`, ~$10/mo) ŌĆö landing dev

Pick one dev DB and schema-isolate. The `db1-use1` instance (not `-eks`) may already be unused ŌĆö check connection metrics. Merging landing into the same instance under a different database name is fine for dev.

Savings: 2 ├Ś $10/mo = ~$20/mo, plus retire unused = ~$30/mo.

### 4. Fargate Spot for agent/worker ŌĆö **~$20/mo, ~$240/yr** {#fargate-spot}

Agent pods are batch-ish: a pod restart is survivable. Move `agent-pool` Karpenter nodepool to spot-preferred consolidation:

```yaml
apiVersion: karpenter.sh/v1
kind: NodePool
metadata:
  name: agent-pool
spec:
  template:
    spec:
      requirements:
        - key: karpenter.sh/capacity-type
          operator: In
          values: ["spot", "on-demand"]   # spot first
        - key: karpenter.k8s.aws/instance-category
          operator: In
          values: ["t", "c", "m"]
  disruption:
    consolidationPolicy: WhenEmptyOrUnderutilized
```

Spot nodes are ~70 % cheaper. Agent workloads on `t3a.medium` (~$22/mo) ├Ś 3 = $66/mo on-demand ŌåÆ ~$20/mo on spot = **~$45/mo savings**, conservatively halved to $20/mo accounting for interruption overhead.

### 5. Dev MQ ŌåÆ in-cluster ŌĆö **~$15/mo, ~$180/yr** {#dev-mq}

`Broker-kuber-dev-eks2` is a managed `mq.t3.micro` (~$15/mo). Dev tolerates broker restarts ŌĆö replace with a single `rabbitmq:3-management-alpine` pod in `skygen-agent` namespace:

```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata: { name: rabbitmq, namespace: skygen-agent }
# ...
```

Prod stays on managed MQ ŌĆö the reliability cost on prod is higher than $15/mo.

### 6. Secrets Manager audit ŌĆö **~$10ŌĆō15/mo, ~$120ŌĆō180/yr** {#secrets-audit}

127 secrets ├Ś $0.40 = $50.80. Some show `LastAccessedDate: None` ŌĆö never read since creation.

```bash
aws secretsmanager list-secrets --region us-east-1 \
  --query "SecretList[?LastAccessedDate==null].[Name,CreatedDate]" --output table
```

Candidates for review:
- Per-MCP-integration OAuth pairs for integrations that were never launched.
- Old `skygen/google-drive-*` if `skygen/google-restricted-*` replaced them.

For non-sensitive config, **SSM Parameter Store** (Standard tier) is free up to 10,000 parameters.

### 7. Prod MultiAZ RDS ŌĆö **negative savings, reliability decision** {#rds-multiaz}

`skygen-prod-db` is single-AZ. Enabling MultiAZ ~doubles RDS cost (~$24/mo extra). This is *added* spend, but pays for itself if an AZ outage stops revenue for 15 minutes. Flag for the team as a reliability decision, not a cost decision.

### Tier 2 total (excluding MultiAZ add): **~$260ŌĆō320/mo, ~$3,100ŌĆō3,800/yr**

## Tier 3: Needs investigation {#tier-3}

Savings depend on what the resource actually does. Investigate before acting.

### 1. OSWorld EC2 instances ŌĆö **up to $100/mo = $1,200/yr if idle** {#osworld-cost}

Three instances (`OSWorld Host` t3.large, 2├Ś `OSWorld-Client-1` t3.xlarge) have been running since FebŌĆōApril 2026. t3.xlarge on-demand = ~$121/mo each. Total: **~$315/mo of compute** for the OSWorld trio.

**Questions to answer:**
- Is OSWorld benchmarking a continuous workload, or is it run for periodic evaluations?
- If periodic: schedule start/stop via EventBridge (e.g., run 4 hrs/day = 83 % savings).
- If continuous: look at utilization. CloudWatch `CPUUtilization` average ŌĆö if <15 %, downsize.

### 2. Lightsail `Ubuntu-1` (eu-north-1) ŌĆö **$3.71ŌĆō$10/mo = $45ŌĆō120/yr** {#lightsail}

Running since 2026-01-11, 2 vCPU / 2 GB. IP `13.62.245.190`. Purpose not documented.

**Investigate:**
- SSH into it: what's running? (`systemctl list-units --type=service --state=running`)
- Does any DNS record point here? Any Skygen service reference this IP?
- Delete if unused.

### 3. Client VPN `eu-north-1` ŌĆö **$5/mo = $60/yr** {#client-vpn-cost}

March bill shows `EUN1-ClientVPN-EndpointHours = $5.05`. `describe-client-vpn-endpoints` returns empty. Either it's a stale association or there's a billing ghost.


### 4. `remote_desktop` EC2 (`i-0b2eb7657fb495db2`) ŌĆö **~$30/mo if idle** {#remote-desktop}

t3.medium, public IP `44.223.39.209`, running since April 3. Purpose not documented. Used for Cloud PC development? Orphaned demo host?

### 5. Dev cluster monitoring scale-down ŌĆö **~$15ŌĆō25/mo, ~$180ŌĆō300/yr** {#dev-monitoring}

Dev Prometheus has 20 Gi PVC, Loki 10 Gi, Grafana 5 Gi ŌĆö same as prod. Dev retention is rarely worth 15 days. Possible tune:

```yaml
prometheus:
  retention: 3d          # from 15d
  retentionSize: 8GB     # from 20GB
loki:
  table_manager:
    retention_period: 72h  # from 168h
```

Needs confirmation that nobody depends on 15-day dev retention for incident postmortems.

## Overview

Skygen's infrastructure runs on EKS with Karpenter for dynamic node provisioning. The main cost levers are: Spot instances for stateless workloads, right-sizing persistent resources (RDS, ElastiCache, Amazon MQ), eliminating unused services, and reducing data transfer costs. This guide covers concrete changes that reduce monthly spend.

---

## Karpenter Spot instances

Agent and worker pods are stateless and can be safely interrupted. Using EC2 Spot instances for these workloads typically saves 60–80% over On-Demand pricing.

### Node pool configuration

```yaml
# Backend/k8s/karpenter/nodepool-agent.yaml
spec:
  template:
    spec:
      nodeClassRef:
        name: default
      requirements:
        - key: karpenter.sh/capacity-type
          operator: In
          values: ["spot", "on-demand"]   # prefer spot
        - key: node.kubernetes.io/instance-type
          operator: In
          values: ["t3.small", "t3.medium", "t3a.small", "t3a.medium"]
  disruption:
    consolidationPolicy: WhenUnderutilized
    consolidateAfter: 30s
```

The `consolidateAfter: 30s` setting aggressively consolidates underutilized nodes. Backend pods should use `on-demand` only if they require guaranteed availability; agent and worker pods are good Spot candidates.

### Disruption handling

Backend pods handle graceful shutdown via `preStop` hooks and the default SIGTERM handler in uvicorn. Agents checkpoint task state to Redis before termination, so Spot interruption does not lose user work.

## Right-sizing compute

### Backend pods

Current resource requests are conservative. Monitor actual usage via Prometheus `container_cpu_usage_seconds_total` and `container_memory_working_set_bytes`:

```promql
# P95 CPU usage for backend pods over 7 days
histogram_quantile(0.95, 
  sum(rate(container_cpu_usage_seconds_total{pod=~"backend.*"}[5m])) by (le)
)
```

If P95 CPU stays below 200m, reduce requests from `300m` to `150m` and limits from `1000m` to `500m`. This allows the HPA to pack more pods per node.

### RDS

Switch `db.t3.medium` (dev) to `db.t3.small` if the database is not CPU-bound. Enable RDS auto-scaling storage to avoid over-provisioning. In dev, consider stopping the RDS instance during off-hours using EventBridge Scheduler:

```bash
aws rds stop-db-instance --db-instance-identifier skygen-dev
```

### ElastiCache

`cache.t3.micro` is adequate for dev. In prod, monitor `CurrConnections` and `CacheHits` to right-size. Enable cluster mode only if you need horizontal scaling beyond a single node.

## Node pool consolidation

Karpenter's `WhenUnderutilized` consolidation policy deletes nodes that are below a utilization threshold. Key settings:

```yaml
disruption:
  consolidationPolicy: WhenUnderutilized
  consolidateAfter: 60s    # wait 60s before consolidating
  budgets:
    - nodes: "20%"         # never terminate more than 20% at once
```

During business hours when load is high, consolidation is automatically inhibited by Karpenter because pods would not fit on remaining nodes. At night with low load, nodes consolidate to as few as the minimum.

## Storage tiering

### S3 Intelligent-Tiering

Enable Intelligent-Tiering on the screenshots bucket to automatically move infrequently-accessed objects to cheaper storage tiers:

```bash
aws s3api put-bucket-intelligent-tiering-configuration \
  --bucket skygen-files \
  --id tiering \
  --intelligent-tiering-configuration '{
    "Id": "tiering",
    "Status": "Enabled",
    "Tierings": [
      {"Days": 90, "AccessTier": "ARCHIVE_ACCESS"},
      {"Days": 180, "AccessTier": "DEEP_ARCHIVE_ACCESS"}
    ]
  }'
```

Screenshots older than 90 days can move to Archive Access (68% cheaper than S3 Standard). Agent trajectory files are write-once read-rarely — good candidates.

### S3 lifecycle rules

Delete temporary presigned-upload objects that were never completed:

```json
{
  "Rules": [{
    "Status": "Enabled",
    "Filter": {"Prefix": "uploads/tmp/"},
    "AbortIncompleteMultipartUpload": {"DaysAfterInitiation": 1}
  }]
}
```

## CloudWatch cost reduction

CloudWatch is the largest hidden cost for EKS clusters. The control plane log types enabled are `api`, `audit`, `authenticator`, `controllerManager`, `scheduler`. The `audit` log stream is the most expensive.

**Option 1:** Disable `audit` logs if not required for compliance:

```bash
aws eks update-cluster-config \
  --name skygen-us-dev \
  --logging '{"clusterLogging":[{"types":["api","authenticator","controllerManager","scheduler"],"enabled":true},{"types":["audit"],"enabled":false}]}'
```

**Option 2:** Route application logs to Loki instead of CloudWatch. Skygen already does this — ensure no Fluent Bit sidecar is accidentally shipping application logs to CloudWatch in addition to Promtail.

## Amazon MQ vs self-hosted RabbitMQ

Amazon MQ charges per broker-hour plus data transfer. For dev, consider replacing Amazon MQ with a self-hosted RabbitMQ pod in the same cluster:

```yaml
# Single-node RabbitMQ for dev
apiVersion: apps/v1
kind: Deployment
metadata:
  name: rabbitmq
  namespace: skygen-agent
spec:
  replicas: 1
  template:
    spec:
      containers:
        - name: rabbitmq
          image: rabbitmq:3-management
          resources:
            requests:
              memory: "256Mi"
              cpu: "100m"
```

This eliminates the Amazon MQ hourly charge (~$50/month for a `mq.t3.micro` broker) at the cost of no managed HA.

## Data transfer optimization

EC2 data transfer within the same AZ is free. Within a region, cross-AZ traffic costs $0.01/GB each way. Pin pods and their data dependencies to the same AZ using topology spread constraints:

```yaml
topologySpreadConstraints:
  - maxSkew: 1
    topologyKey: topology.kubernetes.io/zone
    whenUnsatisfiable: DoNotSchedule
    labelSelector:
      matchLabels:
        app: backend
```

## Summary of potential savings

| Action | Estimated monthly saving |
|--------|------------------------|
| Spot instances for agent/worker nodes | $80–$150 |
| Right-size backend pod requests | $20–$40 (fewer nodes needed) |
| S3 Intelligent-Tiering on screenshots | $10–$30 (after 3 months) |
| Disable CloudWatch audit logs (dev) | $15–$30 |
| Self-hosted RabbitMQ in dev | $50 |
| RDS dev instance scheduling (nights/weekends) | $40–$60 |
| **Total** | **~$215–$360/month** |

Apply changes incrementally and monitor Prometheus metrics after each change to detect regressions.

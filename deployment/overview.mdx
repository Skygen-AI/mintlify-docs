Skygen runs on AWS with Kubernetes (EKS) as the primary compute platform. This page describes the architecture, prerequisites, and the fast path to getting all services running.

## Infrastructure Components

| Component | Service | Notes |
|-----------|---------|-------|
| Container orchestration | AWS EKS | us-east-1; dev namespace `skygen-agent`, prod `skygen-prod` |
| Container registry | AWS ECR | `118183257315.dkr.ecr.us-east-1.amazonaws.com/skygen/` |
| Database | AWS RDS (PostgreSQL 15) | Behind PgBouncer (transaction pool mode) |
| Cache / pub-sub | AWS ElastiCache (Redis) | Used for Socket.IO state and session locks |
| Message queue | Amazon MQ (RabbitMQ) | Agent task dispatch |
| File storage | AWS S3 | Attachments, Cloud PC profiles, screenshots |
| Secrets | AWS Secrets Manager + ExternalSecrets | Auto-synced into K8s secrets |
| TLS | AWS ACM wildcard cert | `*.skygen.ai` |
| Node provisioning | Karpenter | Backend, agent, and worker node pools |
| Monitoring | Prometheus + Grafana + Loki | `monitoring` namespace via Helm |

## EKS Cluster Setup

Two clusters:

| Cluster | Namespace | Use |
|---------|-----------|-----|
| `skygen-us-dev` | `skygen-agent` | Development / staging |
| `skygen-us-prod` | `skygen-prod` | Production |

### Prerequisites

- AWS CLI configured with sufficient IAM permissions
- `kubectl` pointing at the target cluster
- `helm` 3.x
- `gh` CLI (GitHub Actions triggers)
- Docker / ECR login

```bash
# Authenticate to ECR
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin \
  118183257315.dkr.ecr.us-east-1.amazonaws.com
```

## Services Deployed

### Backend (FastAPI)

- **Deployment**: `backend` in `skygen-agent` namespace
- **Replicas**: 2 in dev, HPA-managed in prod (target CPU 70%)
- **Entry point**: `uvicorn app.main:socket_app --host 0.0.0.0 --port 8000`
- **Init container**: runs `alembic upgrade heads` and `python -m scripts.create_admin` on each deploy
- **Node pool**: `role=backend` Karpenter nodes
- **Metrics**: Prometheus scrape on `:8000/metrics`

### Worker (RabbitMQ consumer)

- **Deployment**: `worker` — separate image from `Dockerfile.worker`
- **Purpose**: processes asynchronous tasks consumed from RabbitMQ
- **Node pool**: `role=worker`

### Agent (Python 3.12+)

- **Deployment**: `skygen-agent` pod
- **Entry point**: `python run_agent.py`
- **HTTP port**: 8080 (health + `/metrics` + internal API)
- **Node pool**: `role=agent`

### WebApp

- **Deployed to ECS Fargate** in eu-north-1 (legacy) — migrating to EKS
- **CI/CD trigger**: `push: develop` on `ui-shared-tauri` auto-triggers `deploy-dev.yml`
- **Static assets**: Nginx container

## Database Migrations

Migrations run automatically in the Backend init container on deploy. To run manually:

```bash
kubectl exec -n skygen-agent deploy/backend -- alembic upgrade heads
```

To create a new migration:

```bash
cd Backend
alembic revision --autogenerate -m "describe the change"
```

<Warning>
  The database is behind PgBouncer in **transaction pool mode**. `CREATE INDEX CONCURRENTLY`, `REINDEX CONCURRENTLY`, and `VACUUM` will fail when routed through PgBouncer. Use plain `CREATE INDEX` in migrations, or apply these commands directly against the PostgreSQL endpoint (bypass PgBouncer).
</Warning>

## CI/CD Deployments

Deployment is triggered via **GitHub Actions `workflow_dispatch`** — merging to `develop` does NOT auto-deploy.

```bash
# Deploy Backend to dev
gh workflow run k8s-dev.yml -f branch=develop

# Deploy Agent to dev
gh workflow run k8s-dev.yml -f branch=develop  # in Agent repo

# Coordinated deploy (all services)
gh workflow run coordinated-deploy-dev.yml

# Deploy to prod
gh workflow run k8s-prod.yml -f branch=main
```

<Note>
  The `main` and `develop` branches are protected. You cannot push directly; create a PR and have it reviewed.
</Note>

## Environment Variables

All environment variables are managed through:
1. **AWS Secrets Manager** — sensitive values (API keys, passwords)
2. **K8s ConfigMap** — non-sensitive config per environment
3. **ExternalSecrets** controller — syncs AWS Secrets Manager → K8s secrets

See the complete reference at [Environment Variables](/deployment/environment).

## Quick Deploy Steps

<Steps>
  <Step title="Build and push the image">
    ```bash
    docker build -t skygen/backend:dev .
    docker tag skygen/backend:dev \
      118183257315.dkr.ecr.us-east-1.amazonaws.com/skygen/backend:dev
    ```

  </Step>
  <Step title="Trigger the deployment workflow">

  </Step>
  <Step title="Verify health">
    ```bash
    curl https://dev.skygen.ai/auth/ping
    # → {"msg":"pong"}
    ```
  </Step>
</Steps>

## Local Development

Use Docker Compose for a local stack (without RabbitMQ — disable queue metrics):

```bash
# Start PostgreSQL + Redis + MinIO
docker compose -f Backend/docker-compose-local.yml up -d

# Install dependencies
cd Backend && poetry install

# Run migrations
alembic upgrade head

# Start backend
uvicorn app.main:socket_app --reload --port 8000
```

Set `QUEUE_METRICS_ENABLED=false` in your local `.env` to suppress RabbitMQ connection noise.

For the WebApp:

```bash
cd WebApp
npm install
npm run dev  # → http://localhost:5173
```

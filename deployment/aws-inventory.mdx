| Instance | Class | Version | Storage | MultiAZ | Purpose |

| `skygen-dev-db1-use1` | `db.t4g.micro` | 17.6 | 20 GB gp2 | No | dev — older instance, role unclear |

| `database-landing-dev` | `db.t4g.micro` | 17.6 | 20 GB gp2 | No | landing dev |

Backups: automated (default 7-day retention; `backing-up` status observed on prod).

<Warning>
**No MultiAZ on prod.** A single-AZ `db.t4g.medium` is a SPOF for Skygen users — an AZ outage or DB crash stops all API traffic. Enabling MultiAZ doubles RDS cost but is the standard production posture.
</Warning>

<Note>
Two dev instances (`skygen-dev-db1-use1` and `-eks`) plus two landing instances suggests historical drift. Consolidation candidate. See [Cost optimization](/deployment/cost-optimization#rds-consolidation).
</Note>

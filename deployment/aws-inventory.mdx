## Overview

Skygen runs primarily on AWS us-east-1 with a legacy ECS deployment in eu-north-1 (being migrated). The primary cluster is EKS-based with Karpenter for node autoscaling. All secrets are managed in AWS Secrets Manager and synced to Kubernetes via External Secrets Operator.

---

## EKS clusters

| Cluster | Region | Namespace | Purpose |
|---------|--------|-----------|---------|
| `skygen-us-dev` | us-east-1 | `skygen-agent` | Development environment |
| `skygen-us-prod` | us-east-1 | `skygen-prod` | Production environment |

### Cluster configuration

Both clusters are configured via eksctl (`Backend/k8s/eks/eks-us-cluster.yaml`):

- **Kubernetes version:** 1.30
- **VPC CIDR:** 10.0.0.0/16
- **NAT gateway:** Single (cost-optimized)
- **OIDC:** Enabled (required for IRSA)

### Node groups

| Node group | Instance type | Role | Min/Max |
|-----------|--------------|------|---------|
| `agent-nodes` | t3.small | `role=agent` | 1/10 |
| `worker-nodes` | t3.small | `role=worker` | 1/10 |
| `backend-nodes` | Karpenter-managed | `role=backend` | Dynamic |

Karpenter manages backend nodes using `NodePool` and `EC2NodeClass` resources (`Backend/k8s/karpenter/`). Node pools support Spot instances for cost optimization.

## Kubernetes workloads

### Backend deployment

- **Image:** `118183257315.dkr.ecr.us-east-1.amazonaws.com/skygen/backend:dev`
- **Replicas:** 2 (dev) / HPA-managed (prod)
- **Resources:** 768Mi–1536Mi RAM, 300m–1000m CPU
- **Command:** `uvicorn app.main:socket_app --host 0.0.0.0 --port 8000 --workers 1`
- **Single worker** required: Socket.IO uses in-memory SID maps; scale via K8s replicas

### HPA (Horizontal Pod Autoscaler)

```yaml
minReplicas: 2
maxReplicas: 10
metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        averageUtilization: 70
```

### PodDisruptionBudget

```yaml
minAvailable: 1
```

## Amazon ECR

Container images are stored in ECR:

| Repository | Description |
|-----------|-------------|
| `skygen/backend` | FastAPI backend image |
| `skygen/agent` | Python agent image |
| `skygen/worker` | RabbitMQ consumer worker image |

Images are tagged with git SHA and environment (`dev`, `prod`). CI/CD pushes via GitHub Actions (`aws-actions/amazon-ecr-login`).

---

## Amazon RDS (PostgreSQL)

- **Engine:** PostgreSQL 15
- **Instance:** `db.t3.medium` (dev), `db.r6g.large` (prod)
- **Multi-AZ:** Yes (prod)
- **Connection pooling:** PgBouncer in transaction mode (note: `CONCURRENTLY` operations fail through PgBouncer — apply migrations directly to Postgres)
- **DATABASE_URL** secret synced via ExternalSecrets

---

## Amazon ElastiCache (Redis)

Redis is used for:
- Session caching (device status, user online state)
- Pub/sub for horizontal scaling of Socket.IO
- Hibernation sentinel task store
- Rate limiting (token buckets)

- **Engine:** Redis 7.x
- **Instance:** `cache.t3.micro` (dev), `cache.r6g.large` (prod)
- **REDIS_URL** secret synced via ExternalSecrets

---

## Amazon MQ (RabbitMQ)

RabbitMQ handles async message processing between the backend and worker pods.

- **Engine:** RabbitMQ 3.x (Amazon MQ managed)
- **RABBITMQ_URL** secret synced via ExternalSecrets
- Worker pods (`Dockerfile.worker`) consume from queues

## Amazon S3

All file storage uses S3 via the MinIO-compatible SDK:

| Bucket path | Content |
|-------------|---------|
| `screenshots/agent/{session_id}/` | Agent-captured screenshots |
| `screenshots/{user_id}/` | User-uploaded screenshots |
| `connector-icons/{user_id}/` | Custom connector icons |
| `helpdesk/{ticket_id}/attachments/` | Helpdesk file attachments |
| `knowledge/` | Knowledge base files |

- **Region:** us-east-1
- **Configuration:** `MINIO_ENDPOINT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`, `MINIO_BUCKET`
- Screenshots and file attachments use presigned POST URLs (5-minute expiry)

## AWS Secrets Manager

All application secrets are stored in Secrets Manager under `skygen/{env}/`:

| Key prefix | Contents |
|-----------|---------|
| `skygen/dev/backend-secrets-us` | DATABASE_URL, RABBITMQ_URL, REDIS_URL, Gmail OAuth, Calendar OAuth |
| `skygen/dev/backend-secrets-eu` | SECRET_KEY, AGENT_API_TOKEN, MCP_CRYPTO_KEY, STRIPE keys, COMPOSIO keys |
| `skygen/dev/monitoring-*` | Grafana credentials, Alertmanager secrets |

Secrets are synced to K8s Secrets via External Secrets Operator with IRSA-based authentication.

---

## IAM

- **OIDC provider** attached to each EKS cluster
- **IRSA** (IAM Roles for Service Accounts) used by:
  - Backend service account → S3, Secrets Manager
  - ExternalSecrets service account → Secrets Manager read access
  - Agent service account → S3 (trajectory uploads)

---

## CloudWatch

Cluster control plane logs shipped to CloudWatch:
- API server logs
- Audit logs
- Authenticator logs
- Controller manager logs
- Scheduler logs

Application logs are shipped to **Loki** (not CloudWatch) to reduce cost.

## Legacy ECS (eu-north-1)

The `ui-shared-tauri` frontend is deployed to ECS Fargate in eu-north-1 (`deploy-dev.yml`). This is a legacy deployment being migrated to the primary EKS cluster. It deploys automatically on push to `develop`.

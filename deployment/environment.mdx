title: "Environment variables"
description: "Every setting the backend, worker, and agent read at startup — grouped by domain."

All configuration lives in `Backend/app/settings.py` as Pydantic `BaseSettings`, loaded from `.env` or real env vars in production. Prod secrets are synced from AWS Secrets Manager via ExternalSecrets.

## Required in production

These MUST be set or the service will not start correctly.

| Var | Purpose |
|-----|---------|
| `SECRET_KEY` | JWT signing key, minimum 32 chars |
| `AGENT_API_TOKEN` | Backend ↔ Agent shared bearer token |
| `MCP_CRYPTO_KEY` | AES key for encrypting custom-connector secrets |
| `DATABASE_URL` | PostgreSQL async connection string |
| `RABBITMQ_URL` | RabbitMQ broker URL |
| `REDIS_URL` | Redis for Socket.IO + caching |
| `API_BASE_URL` | Public backend URL (for OAuth redirects) |
| `BACKEND_URL` | Internal backend URL (worker → backend) |
| `ALLOWED_ORIGINS` | CORS allowlist |
| `ENVIRONMENT` | `development` or `production` |
| `INTERNAL_SERVICE_TOKEN` | Inter-service auth (backend ↔ worker) |

## Secrets management

Production: AWS Secrets Manager via ExternalSecrets operator.

- `skygen/{env}/backend-secrets-us` — DB/Redis/RabbitMQ URLs, OAuth client pairs, billing keys
- `skygen/{env}/backend-secrets-eu` — `SECRET_KEY`, `AGENT_API_TOKEN`, `MCP_CRYPTO_KEY`, `GEMINI_API_KEY`, `E2B_API_KEY`, `COMPOSIO_API_KEY`, MinIO, Stripe, Resend, Telegram
- `skygen/{env}/monitoring-*` — Grafana admin, Alertmanager SMTP

ExternalSecret CRDs sync changes into Kubernetes Secrets; pods pick them up on restart.

All Backend configuration is driven by environment variables loaded via Pydantic `BaseSettings` from a `.env` file or the process environment. This page documents every variable in `Backend/app/settings.py`.

<Note>
  In production, sensitive values are stored in AWS Secrets Manager and synced to Kubernetes via ExternalSecrets. Non-sensitive values live in the environment-specific ConfigMap.
</Note>

## Core / Required

These variables have no safe defaults and must be set in every environment.

<ParamField body="DATABASE_URL" type="string" required>
  PostgreSQL connection string including credentials. Format: `postgresql+asyncpg://user:password@host:5432/dbname`. In production, this points to PgBouncer, not Postgres directly.
</ParamField>

<ParamField body="RABBITMQ_URL" type="string" required>
  RabbitMQ AMQP URL. Example: `amqp://user:pass@rabbitmq:5672/vhost`.
</ParamField>

<ParamField body="SEND_MESSAGE_TO_AGENT_QUEUE" type="string" required>
  RabbitMQ queue name for sending messages to the agent. Supports `{device_id}` interpolation for per-device routing. Example: `content` or `agent-{device_id}`.
</ParamField>

<ParamField body="ANSWER_AGENT_QUEUE" type="string" required>
  RabbitMQ queue name for agent responses back to the backend worker.
</ParamField>

## Security

<ParamField body="SECRET_KEY" type="string" required>
  HMAC secret for JWT signing. Must be at least 32 characters. The server will refuse to start in production if this is not set or too short.
</ParamField>

<ParamField body="ALGORITHM" type="string" default="HS256">
  JWT signing algorithm. Should remain `HS256` unless you have a specific reason to change it.
</ParamField>

<ParamField body="ACCESS_TOKEN_EXPIRE_MINUTES" type="integer" default="43200">
  JWT token lifetime in minutes. Default is 30 days (43,200 minutes).
</ParamField>

<ParamField body="AGENT_API_TOKEN" type="string" required>
  Bearer token used by the Agent service to call protected backend endpoints (e.g. Knowledge Base, billing). Must match the token configured in the Agent service.
</ParamField>

<ParamField body="MCP_CRYPTO_KEY" type="string" required>
  Encryption key for MCP connector credentials. Must be set to a strong random key. The server will error on startup if this is the default value `qwerty1234567890`.
</ParamField>

## Redis

<ParamField body="REDIS_URL" type="string" default="">
  Redis connection URL for distributed Socket.IO state and session locks. Example: `redis://redis:6379/0`. Required for multi-replica deployments.
</ParamField>

## Agent Internal Communication

<ParamField body="AGENT_INTERNAL_URL" type="string" default="http://agent-service:8080">
  Internal HTTP URL for Backend-to-Agent calls (e.g. playbook extraction). In EKS, use the K8s service name.
</ParamField>

<ParamField body="BACKEND_URL" type="string" default="http://backend:8000">
  Internal URL for Worker-to-Backend communication. In ECS with Service Discovery: `http://backend:8000`. For local dev: `http://localhost:8000`.
</ParamField>

<ParamField body="API_BASE_URL" type="string" default="https://kdev.skygen.ai">
  Public API URL used for OAuth redirect URIs. Production: `https://api.skygen.ai`. Dev: `https://dev.skygen.ai`.
</ParamField>

## Frontend URLs

<ParamField body="FRONTEND_URL" type="string" default="http://localhost:3000">
  WebApp origin (no path). Production: `https://skygen.ai`. Used for constructing email verification links, OAuth callbacks, Stripe redirects, etc.
</ParamField>

<ParamField body="MAIN_SITE_URL" type="string" default="https://skygen.ai">
  Next.js marketing site URL. Used for public email links (e.g. homepage download anchors). Defaults to production so dev builds produce clickable links.
</ParamField>

<ParamField body="WEBAPP_BASE_PATH" type="string" default="">
  Path prefix for WebApp routing. Production: `/app`. Empty for local dev (WebApp at root).
</ParamField>

## Authentication Cookies

<ParamField body="AUTH_COOKIE_NAME" type="string" default="skygen_session">
  Name of the `HttpOnly` JWT session cookie. Dev override: `skygen_session_dev` to prevent collision with prod on shared `.skygen.ai` domain.
</ParamField>

<ParamField body="SKYGEN_LOGGED_IN_COOKIE_NAME" type="string" default="skygen_logged_in">
  Name of the public UI-marker cookie (boolean flag, not a JWT). Dev override: `skygen_logged_in_dev`.
</ParamField>

<ParamField body="AUTH_COOKIE_DOMAIN" type="string" default="">
  Cookie `Domain` attribute. Empty = host-only (correct for localhost). Production: `.skygen.ai` (leading dot covers all subdomains).
</ParamField>

<ParamField body="AUTH_COOKIE_SECURE" type="boolean" default="false">
  Set to `true` in production to require HTTPS for the session cookie.
</ParamField>

<ParamField body="AUTH_COOKIE_SAMESITE" type="string" default="lax">
  Cookie `SameSite` policy. `lax` allows top-level navigations (email links) while blocking CSRF on cross-site POST.
</ParamField>

<ParamField body="AUTH_COOKIE_MAX_AGE_SECONDS" type="integer" default="2592000">
  Cookie expiry in seconds. Default: 30 days (matches `ACCESS_TOKEN_EXPIRE_MINUTES`).
</ParamField>

<ParamField body="AUTH_LEGACY_UPGRADE_DISABLED" type="boolean" default="false">
  When `true`, `POST /auth/upgrade-session` returns HTTP 410. Set after the localStorage-to-cookie migration is complete.
</ParamField>

## Storage (S3 / MinIO)

<ParamField body="MINIO_ENDPOINT" type="string" default="">
  S3-compatible endpoint URL. For AWS S3: leave empty and use AWS credential chain. For MinIO: `http://minio:9000`.
</ParamField>

<ParamField body="MINIO_ACCESS_KEY" type="string" default="">
  S3 access key ID.
</ParamField>

<ParamField body="MINIO_SECRET_KEY" type="string" default="">
  S3 secret access key.
</ParamField>

<ParamField body="MINIO_BUCKET" type="string" default="">
  Default S3 bucket for screenshots and general file uploads.
</ParamField>

<ParamField body="ATTACHMENTS_BUCKET" type="string" default="">
  S3 bucket for message attachments.
</ParamField>

<ParamField body="MINIO_PUBLIC_URL" type="string" default="">
  Public base URL for presigned URLs. Leave empty to use the endpoint URL.
</ParamField>

<ParamField body="AWS_S3_REGION" type="string" default="">
  AWS region for S3 operations. Example: `us-east-1`.
</ParamField>

## E2B (Cloud PC)

<ParamField body="E2B_API_KEY" type="string" default="">
  API key for the E2B sandbox service. Required when `CLOUD_PC_ENABLED=true`.
</ParamField>

<ParamField body="E2B_DEFAULT_TIMEOUT" type="integer" default="3600">
  Default sandbox session timeout in seconds (1 hour).
</ParamField>

<ParamField body="CLOUD_PC_ENABLED" type="boolean" default="false">
  Feature flag for Cloud PC (persistent E2B desktop). Set to `true` to enable the Cloud PC feature.
</ParamField>

<ParamField body="CLOUD_PC_S3_BUCKET" type="string" default="skygen-cloudpc-profiles">
  S3 bucket for Cloud PC filesystem snapshots (backup/restore).
</ParamField>

<ParamField body="CLOUD_PC_TEMPLATE_ID" type="string" default="desktop">
  E2B sandbox template to use when creating Cloud PCs.
</ParamField>

<ParamField body="CLOUD_PC_AUTO_PAUSE_MINUTES" type="integer" default="5">
  Minutes of inactivity before a Cloud PC is auto-paused.
</ParamField>

<ParamField body="CLOUD_PC_MAX_IDLE_MINUTES" type="integer" default="30">
  Maximum idle time before auto-pause even if heartbeats are still arriving.
</ParamField>

<ParamField body="CLOUD_PC_MAX_BACKUP_SIZE_MB" type="integer" default="500">
  Maximum allowed Cloud PC filesystem snapshot size in MB.
</ParamField>

<ParamField body="CLOUD_PC_COST_PER_HOUR_USD" type="float" default="0.1656">
  Raw E2B cost per hour (2 vCPU, 4 GiB RAM).
</ParamField>

<ParamField body="CLOUD_PC_MARGIN_MULTIPLIER" type="float" default="2.5">
  Markup multiplier applied to `CLOUD_PC_COST_PER_HOUR_USD` for user billing.
</ParamField>

<ParamField body="MULTI_PARALLEL_CLOUD_PC_ENABLED" type="boolean" default="false">
  Enables multi-parallel Cloud PC mode. When `false`, legacy single-CPC-per-user behavior applies.
</ParamField>

## Scheduler

<ParamField body="STALE_RUN_TIMEOUT_MINUTES" type="integer" default="30">
  Number of minutes before a scheduled task run that has not updated its status is marked as stale and timed out.
</ParamField>

## Billing (Stripe)

<ParamField body="BILLING_ENABLED" type="boolean" default="true">
  Master billing switch. Set to `false` to bypass all billing checks (useful for internal deployments).
</ParamField>

<ParamField body="STRIPE_SECRET_KEY" type="string" default="">
  Stripe secret key (`sk_test_...` or `sk_live_...`).
</ParamField>

<ParamField body="STRIPE_PUBLISHABLE_KEY" type="string" default="">
  Stripe publishable key (`pk_test_...` or `pk_live_...`).
</ParamField>

<ParamField body="STRIPE_WEBHOOK_SECRET" type="string" default="">
  Stripe webhook signing secret (`whsec_...`). Used to verify webhook authenticity.
</ParamField>

<ParamField body="STRIPE_PRICE_ID_STANDARD" type="string" default="">
  Stripe Price ID for the $14.99/month subscription.
</ParamField>

<ParamField body="STRIPE_PRICE_ID_OVERAGE" type="string" default="">
  Stripe Price ID for metered overage (pay-per-token billing).
</ParamField>

<ParamField body="STRIPE_METER_EVENT_NAME" type="string" default="token_overage">
  Stripe Billing meter event name for tracking overage. Must match the meter created in your Stripe dashboard.
</ParamField>

<ParamField body="SUBSCRIPTION_PRICE_USD" type="float" default="14.99">
  Monthly subscription price in USD.
</ParamField>

<ParamField body="MONTHLY_TOKENS_INCLUDED" type="integer" default="2000">
  Tokens included in the base subscription per month.
</ParamField>

<ParamField body="TOKEN_MARGIN_MULTIPLIER" type="float" default="1.5">
  Multiplier applied to raw LLM API cost when converting to Skygen tokens.
</ParamField>

<ParamField body="TOKENS_PER_DOLLAR" type="float" default="131.58">
  Normalization constant used in token pricing calculations.
</ParamField>

<ParamField body="OVERAGE_PRICE_PER_TOKEN" type="float" default="0.0075">
  Price per token beyond the included monthly allowance ($14.99 / 2000 tokens).
</ParamField>

## Registration

<ParamField body="REGISTRATION_MODE" type="string" default="open">
  Controls who can register. `open` = anyone can register. `invite_only` = requires a valid invite code.
</ParamField>

## Email (Amazon SES)

<ParamField body="MAIL_ENABLED" type="boolean" default="true">
  Set to `false` to disable all outbound email (useful in dev/test environments).
</ParamField>

<ParamField body="MAIL_FROM" type="string" default="noreply@skygen.ai">
  From address for outbound emails. Must be a verified identity in Amazon SES.
</ParamField>

<ParamField body="MAIL_FROM_NAME" type="string" default="Skygen AI">
  Display name for the from address.
</ParamField>

<ParamField body="MAIL_REPLY_TO_SUPPORT" type="string" default="support@skygen.ai">
  Reply-To address for helpdesk/support emails.
</ParamField>

<ParamField body="AWS_SES_REGION" type="string" default="us-east-1">
  AWS region for SES API calls.
</ParamField>

<ParamField body="AWS_SES_CONFIGURATION_SET" type="string" default="">
  Optional SES configuration set name for tracking email bounces and complaints.
</ParamField>

<ParamField body="EMAIL_UNSUBSCRIBE_BASE_URL" type="string" default="https://api.skygen.ai/email/unsubscribe">
  Base URL for email unsubscribe links. A `?token=...` parameter is appended.
</ParamField>

<ParamField body="EMAIL_SEND_CONCURRENCY" type="integer" default="10">
  Maximum concurrent SES send calls during the inactive-users batch email job.
</ParamField>

<ParamField body="EMAIL_TRIGGERS_ENABLED" type="boolean" default="true">
  Controls lifecycle emails (inactivity, uninstall, welcome). Transactional emails (verification, password reset, helpdesk) are not affected.
</ParamField>

<ParamField body="FRONTEND_URL" type="string" default="http://localhost:3000">
  Also used to construct links in emails. See Frontend URLs section above.
</ParamField>

## MCP / Composio

<ParamField body="COMPOSIO_API_KEY" type="string" default="">
  Composio API key (`comp_...`). Different per environment (dev vs prod).
</ParamField>

<ParamField body="COMPOSIO_WEBHOOK_SECRET" type="string" default="">

<ParamField body="COMPOSIO_ENABLED" type="boolean" default="true">

<ParamField body="LOKI_URL" type="string" default="">

</ParamField>

<ParamField body="RATING_ALERT_TELEGRAM_BOT_TOKEN" type="string" default="">

</ParamField>

<ParamField body="RATING_ALERT_TELEGRAM_CHAT_ID" type="string" default="">

<ParamField body="HELPDESK_TELEGRAM_CHAT_ID" type="string" default="">

<ParamField body="QUEUE_METRICS_ENABLED" type="boolean" default="true">

## Client Version Check

<ParamField body="CLIENT_VERSION_CHECK_ENABLED" type="boolean" default="true">
  When `true`, clients must send `client_version` with every message. Missing or outdated versions receive a prompt to update.
</ParamField>

<ParamField body="MIN_CLIENT_VERSION" type="string" default="1.0.0">
  Minimum required client version (semver). Clients below this version are asked to update.
</ParamField>

## Environment

<ParamField body="ENVIRONMENT" type="string" default="development">
  Current environment name. Used for Composio user isolation (prefixes `user_id`) and security validation behavior. Values: `development`, `production`.
</ParamField>

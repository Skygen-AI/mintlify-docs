All Skygen API errors follow a consistent JSON format:

```json
{
  "detail": "Human-readable error message"
}
```

For validation errors (HTTP 422), FastAPI returns a structured array:

```json
{
  "detail": [
    {
      "loc": ["body", "field_name"],
      "msg": "field required",
      "type": "value_error.missing"
    }
  ]
}
```

## HTTP Status Code Reference

### 400 Bad Request

The request is malformed or violates a business rule.

| Detail message | Cause | Resolution |
|----------------|-------|------------|
| `Email already registered` | Registration attempt with an email that already has an account | Use `POST /auth/token` to log in instead |
| `This email is already registered using Google. Please sign in with Google.` | Email exists but was created via Google OAuth | Use the Google sign-in flow |
| `Invalid invite code` | Invite code not found in the database | Check the code for typos; request a new code |
| `This invite code has been deactivated` | Invite code was manually deactivated by an admin | Request a new code |
| `This invite code has already been used` | Invite code was already redeemed | Each code is single-use |
| `This invite code has expired` | Invite code past its `expires_at` date | Request a new code |
| `Invite code is required for registration` | Server is in `invite_only` mode | You must have a valid invite code |
| `Invalid token` | JWT has no `sub` claim or is structurally invalid | Re-authenticate to get a fresh token |
| `Invalid or expired token` | JWT failed signature verification or is past expiry | Re-authenticate |
| `meta dict required` | `PATCH /auth/update-registration-meta` missing `meta` field | Include a non-empty `meta` object |
| `Session is not deleted` | Attempted to restore a session that is still active | Only deleted sessions can be restored |
| `Bearer header required` | `POST /auth/upgrade-session` missing `Authorization` header | Add `Authorization: Bearer <token>` |

### 401 Unauthorized

Authentication failed or token is invalid.

| Detail message | Cause | Resolution |
|----------------|-------|------------|
| `Incorrect email or password` | Login credentials do not match | Check credentials; use forgot-password if needed |
| `Not authenticated` | No JWT token or cookie provided | Add `Authorization: Bearer <token>` header |
| `Invalid token` | Token was issued before the user's last password change | Re-authenticate to get a new token |
| `Inactive user` | User account has been deactivated by an admin | Contact support |
| `Could not validate credentials` | Generic JWT validation failure | Re-authenticate |
| `Invalid Google token` | Google ID token verification failed | Ensure the token is fresh and from an allowed client ID |
| `Invalid Google token claims` | Google token missing `email` or `sub` | Ensure the Google SDK returns verified tokens |
| `Google email not verified` | Google account's email is not verified | Verify the Google account's email first |

### 403 Forbidden

Authentication succeeded but the user lacks permission.

| Detail message | Cause | Resolution |
|----------------|-------|------------|
| `Forbidden` | Accessing another user's session or resource | Use only resources owned by your user |
| `Not authorized` | Accessing another user's device | Use only your own devices |
| `Cannot create session with a device that doesn't belong to you` | `device_id` belongs to a different user | Use a device owned by your account |
| `Admin access required` | Non-admin accessing an admin-only endpoint | Requires an admin user token |

### 404 Not Found

The requested resource does not exist or was deleted.

| Detail message | Cause | Resource |
|----------------|-------|---------|
| `Session not found` | Session ID invalid, deleted, or not yours | Chat sessions |
| `Device not found` | Device ID invalid, deleted, or not yours | Devices |
| `Scheduled task not found` | Task ID invalid or belongs to another user | Scheduled tasks |
| `User not found` | User ID not in database | User management (admin) |
| `Emplogent not found` | Emplogent ID invalid or not yours | Emplogents |
| `Message not found` | Message ID invalid | Chat messages |

### 409 Conflict

The request conflicts with the current state of the resource.

| Detail message | Cause | Resolution |
|----------------|-------|------------|
| `Device with token kid '...' already exists` | KID collision on device registration | Use a unique KID or omit it to let the server generate one |
| `Previous run still in progress` | Triggered a scheduled task run when `max_concurrent_runs` is reached | Wait for the current run to finish, or increase `max_concurrent_runs` |

### 410 Gone

The endpoint has been permanently disabled.

| Detail message | Cause |
|----------------|-------|
| `legacy upgrade disabled` | `POST /auth/upgrade-session` has been disabled via `AUTH_LEGACY_UPGRADE_DISABLED=true` |

### 422 Unprocessable Entity

The request body is syntactically valid but fails business-rule validation.

| Detail message | Cause |
|----------------|-------|
| `scheduled_at must be in the future` | One-time task with a `scheduled_at` in the past |
| `Invalid time_of_day format, expected HH:MM` | Scheduled task `time_of_day` not in `HH:MM` format |
| `Could not extract playbook from this session` | Agent could not parse a playbook from the session trajectory |
| `Session has no messages` | Tried to extract playbook from an empty session |
| FastAPI validation array | Missing required fields, wrong types, out-of-range values |

### 429 Too Many Requests

Rate limit exceeded.

| Endpoint | Limit |
|----------|-------|
| `POST /auth/token` | Per-IP rate limit (exact threshold is environment-specific) |
| `POST /auth/forgot-password` | Per-IP rate limit |
| `POST /auth/reset-password` | Per-IP rate limit |

Back off and retry after a brief delay (start with 5 seconds).

### 500 Internal Server Error

Unexpected server-side error. These should be rare. If you encounter one consistently:
1. Check the `detail` message for context.
2. Retry once — transient database or network errors may resolve.
3. If persistent, contact support with the request ID from the response headers (if present) and the timestamp.

### 501 Not Implemented

A feature is not configured on the server.

| Detail message | Cause |
|----------------|-------|
| `Google OAuth is not configured on this server` | `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` are not set |

### 503 Service Unavailable

A downstream service is unreachable.

| Detail message | Cause |
|----------------|-------|
| `Agent service unavailable` | The Agent service did not respond to a playbook extraction request |

## Billing Errors

Billing errors are returned as structured payloads when sending a message while in a billing-blocked state. These are returned with HTTP 402:

```json
{
  "billing_error": {
    "error_type": "subscription_required",
    "title": "Subscription Required",
    "message": "Subscribe to continue using Skygen.",
    "action": "subscribe"
  }
}
```

Common billing error types:

| `error_type` | Cause |
|-------------|-------|
| `subscription_required` | User has no active subscription |
| `payment_failed` | Stripe payment failed |
| `overage_disabled` | Token allowance exhausted and overage billing is disabled |
| `trial_expired` | Free trial period has ended |

## Knowledge Base Errors

KB endpoints return 500 on internal embedding or retrieval failures. The error detail includes the underlying exception message for debugging.

## Client-Side Handling Patterns

```python
import httpx
import time

def api_call_with_retry(method, url, **kwargs):
    """Simple retry logic for transient errors."""
    headers = kwargs.pop("headers", {})
    headers["Authorization"] = f"Bearer {TOKEN}"

    for attempt in range(3):
        try:
            resp = httpx.request(method, url, headers=headers, **kwargs)

            if resp.status_code == 401:
                # Token expired — re-authenticate
                raise Exception("Token expired, please re-authenticate")

            if resp.status_code == 429:
                # Rate limited — back off
                time.sleep(5 * (attempt + 1))
                continue

            if resp.status_code >= 500:
                # Server error — retry with backoff
                if attempt < 2:
                    time.sleep(2 ** attempt)
                    continue

            resp.raise_for_status()
            return resp

        except httpx.NetworkError:
            if attempt < 2:
                time.sleep(2 ** attempt)
                continue
            raise

    raise Exception("Max retries exceeded")
```

```javascript
async function apiCall(method, path, body) {
  const resp = await fetch(`https://api.skygen.ai${path}`, {
    method,
    headers: {
      "Authorization": `Bearer ${token}`,
      "Content-Type": "application/json",
    },
    body: body ? JSON.stringify(body) : undefined,
  });

  if (resp.status === 401) {
    // Redirect to login
    window.location.href = "/login";
    return;
  }

  if (!resp.ok) {
    const error = await resp.json();
    throw new Error(error.detail || `HTTP ${resp.status}`);
  }

  return resp.json();
}
```

## 3. RabbitMQ â€” Messages Published to Backend

**Send path:** `BrokerAgent.publish_agent_message()` â†’ `_BuildAgentMessageTransport()` â†’ `Transport.Send(AgentMessage)`

## 4. In-Process Inbox Messages (Agent â†” Agent)

These are **NOT Socket.IO events**. They are Python dict messages pushed directly into `BaseAgent.inbox` (an `asyncio.Queue`). Processed by `_handle_inbox_messages()` at the start of each ReAct iteration.

They do **not** cross process boundaries â€” strictly for orchestrator â†” sub-agent communication within the same Python process.

### `STOP`
```python
{"type": "STOP", "reason": "string"}
```
**Effect:** Sets `should_cancel = True`. Next LLM call is skipped â†’ `_terminal_return(status=cancelled)`.

### `INFO`
```python
{"type": "INFO", "content": "string"}
```
**Effect:** Injects a `system` message into agent's conversation as background context. Does not interrupt iteration.

### `INFORM`
```python
{"type": "INFORM", "content": "string", "from_agent_id": "string"}
```
**Effect:** Same as `INFO` but semantically indicates a parent-to-child update during execution.

### `CHILD_DONE`
```python
{
  "type": "CHILD_DONE",
  "agent_id": "string",
  "summary": "string",
  "failed": false,
  "plan_step_id": "string | null"
}
```
**Effect:** Orchestrator receives sub-agent result. Triggers `on_subagent_finish()` hook.

### `CHILD_HIBERNATED`
```python
{
  "type": "CHILD_HIBERNATED",
  "agent_id": "string",
  "trigger_type": "user_reply | timer | visual_condition | mcp_connection_ready | cloud_pc_connected",
  "plan_step_id": "string | null"
}
```
**Effect:** Orchestrator notified that child suspended itself. `PlannerAgent.pre_action_guard()` uses this to block finalization while child awaits user reply.

### `USER_FOLLOWUP`
```python
{
  "type": "USER_FOLLOWUP",
  "content": "string",
  "attachments": [],
  "session_id": "string"
}
```
**Effect:** Delivers user's follow-up text. If agent is in `ask_human` hibernate, wakes it via BrokerWaker.

### `PAUSE`
```python
{"type": "PAUSE"}
```
**Effect:** Agent exits ReAct loop after current iteration, persists state, returns `status=paused`.

### `FLUSH`
```python
{"type": "FLUSH"}
```
**Effect:** Triggers immediate scratchpad flush and plan snapshot publication.

### `USER_CANCEL`
```python
{"type": "USER_CANCEL", "session_id": "string"}
```
**Effect:** Soft cancel â€” injects gentle cancellation hint before next LLM call. Hard cancel goes through Redis flag.

## 5. Session Lifecycle

### States

```
[CREATED] â†’ [RUNNING] â†’ [TERMINATED]          (status=completed)
                      â†˜ [HIBERNATED] â†’ [RUNNING] (wake)
                      â†˜ [PAUSED]    â†’ [RUNNING] (resume)
                      â†˜ [CANCELLED]            (status=cancelled)
                      â†˜ [ERROR]                (status=error)
```

### Lifecycle flow

**Start:**
1. Backend publishes message to RabbitMQ worker queue
2. `BrokerAgent.worker_loop()` picks up message
3. `BrokerAgent._route_session()` routes by `agent_mode` â†’ `_process_planner_mode()`, `_process_llm_mode()`, etc.
4. `publish_agent_message(message_type="agent_status", meta.status="started")` sent to RabbitMQ
5. `BaseAgent.chat()` enters the ReAct loop

**ReAct loop (each iteration):**
1. **Inbox check** â€” `_handle_inbox_messages()` processes STOP/PAUSE/USER_FOLLOWUP/etc.
2. **Cancellation check** â€” Redis cancel flag + Inbox USER_CANCEL
3. **Budget alerts** â€” `BudgetAlertManager` evaluates all alert rules
4. **Think** â€” `_think_phase()`: build prompt â†’ LLM call via `ModelRouter.choose_model_route()`
5. **Guardrails** â€” `SafetyManager` checks response for unsafe actions
6. **Parse + Memory** â€” extract tool calls, update `MemoryManager` scratchpad
7. **Termination check** â€” `TerminateGuard` evaluates B3 false-completion guard (PlannerAgent only)
8. **Act** â€” `ActionExecutor` executes tool calls
9. **Observe** â€” inject tool results; CUA injects screenshots here

**Hibernate:**
1. Agent executes `HibernateTool` (or `RequestConfirmationTool` for ARCH-001)
2. `_handle_termination()` detects `HIBERNATE` in tool results
3. `SentinelStore` creates `SentinelTask` keyed by `session_id`
4. `publish_agent_message(message_type="agent_status", meta.status="hibernated")` sent
5. Agent state persisted to Redis: `agent_history:{session_id}`

**Wake:**
1. `Sentinel` polling loop evaluates all `SentinelTask` entries
2. Strategy (`TimerStrategy`, `VisualConditionStrategy`, or user_reply) signals ready
3. `BrokerWaker.Wake()` re-enqueues original RabbitMQ payload with wake metadata
4. For sub-agent hibernate: `BrokerWaker._WakeSubAgent()` pushes `USER_FOLLOWUP` into parent's Inbox
5. `BrokerAgent._process_planner_mode()` detects restored session (non-empty Redis history) and resumes

**Terminate:**
1. `CommunicateTool(message_type="result")` or max iterations reached
2. `_terminal_return()` emits `agent.terminate` to `SharedEventLog`
3. `AgentStateRepository.persist_state()` saves to Redis
4. `_publish_terminal_outcome()`: sends `agent_status` + terminal `agent_response`
5. `_finalize_orphan_plan_steps()` forces remaining plan steps to `skipped`
6. `DeviceLockManager` releases device lock

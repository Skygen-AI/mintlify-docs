## ChatMessage

**Table:** `chat_messages`
**Soft-delete:** No (hard delete supported)
**Services:** `chat_message_repo`, `message_router`, `display_transformer`

| Column | Type | Constraints | Notes |
|--------|------|------------|-------|
| id | VARCHAR(36) | PK, index | UUID string |
| session_id | VARCHAR(36) | FK→chat_sessions(id) ON DELETE CASCADE, NOT NULL, index | |
| task_id | VARCHAR | FK→tasks(id), nullable, index | |
| role | VARCHAR | NOT NULL | "user"\|"assistant"\|"system" |
| content | TEXT | NOT NULL | |
| meta_data | JSON | default={} | `type`, `status`, `ui_meta`, `step_number`, `attachment_id`, etc. |
| created_at | DATETIME | NOT NULL | |

**meta_data types:**
- `type="ui_meta_update"` + `ui_meta.type="plan"` → agent plan steps
- `type="agent_step"` → filtered from GET /messages response
- `type="version_update_required"` → client update notice
- `status="completed"|"error"` → final assistant message

**Note:** `attachments` relationship is `lazy="selectin"` — always eager-loaded (performance concern).

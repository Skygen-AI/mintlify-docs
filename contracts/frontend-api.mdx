## Auth Storage Model

### Token persistence

| Location | Key | Purpose |
|----------|-----|---------|
| `platform.storage` (localStorage on web) | `auth_token` | Bearer header for HTTP + WS fallback |
| `platform.storage` | `refresh_token` | Token refresh flow (not yet wired to auto-refresh) |
| Cookie `skygen_session` | — | HttpOnly cookie set by backend on login/register |

**Cookie migration (active):** `authApi.upgradeSession()` is called at the start of every `restoreSession()`. POSTs to `/auth/upgrade-session` with legacy Bearer token; backend re-emits as HttpOnly cookie. Allows transition from pure-Bearer to cookie-primary auth without forcing re-login.

### Token refresh

`wsClient.setTokenRefresher()` is called after every successful login/signup/session restore. The refresher reads `platform.storage.get('auth_token')` and is called by Socket.IO on every reconnect attempt. **No automatic HTTP token refresh is wired** — only the WS layer refreshes the token dynamically.

## HTTP API Calls

### Authentication

#### POST /auth/token
**Called from:** `authApi.login()` — `api/http/authApi.ts:49`
**Auth:** None
**Request:** `application/x-www-form-urlencoded` — `username`, `password`
**Response:** `TokenResponse { access_token, refresh_token?, token_type }`
**Side effects:** Saves `auth_token` and `refresh_token` to platform.storage; backend sets `skygen_session` cookie
**Used by:** `authSlice.login()`

#### POST /auth/register
**Called from:** `authApi.register()` — `api/http/authApi.ts:94`
**Response:** `RegisterResponse { user: User, access_token, token_type }`
**Side effects:** Saves `auth_token` to platform.storage
**Used by:** `authSlice.signup()`

#### GET /users/me
**Called from:** `authApi.getCurrentUser()` — `api/http/authApi.ts:143`
**Auth:** Bearer token OR `skygen_session` cookie
**Response:** `User { id, email, is_active, is_admin, is_email_verified, is_2fa_verified, display_name, auth_provider, created_at }`
**Used by:** `authSlice.restoreSession()`, `authSlice.refreshUser()`, `authSlice.login()`

#### POST /auth/logout
**Called from:** `authApi.logout()` — `api/http/authApi.ts:455`
**Side effects:** Backend invalidates cookie; client removes `auth_token`, `refresh_token`, `user_data` from storage
**Note:** Best-effort — failure is non-fatal; local state cleared regardless

#### POST /auth/upgrade-session
**Called from:** `authApi.upgradeSession()` — `api/http/authApi.ts:412`
**Auth:** Bearer token (legacy localStorage token)
**Side effects:** Backend converts to cookie session; 410 response clears local storage

#### POST /auth/forgot-password
**Request:** `{ email: string }`
**Response:** `{ msg: string }`

#### POST /auth/reset-password
**Request:** `{ token: string, new_password: string }`
**Response:** `{ msg: string }`

## Socket.IO Connection

**Namespace:** `/user`
**Transport:** WebSocket only (`transports: ['websocket']` — polling disabled)
**Authentication:**
- `auth.token` — JWT from platform.storage (may be null for cookie-only sessions)
- `auth.user_id` — authenticated user ID
- `withCredentials: true` — sends `skygen_session` cookie on cross-origin handshake

**Connection lifecycle:**
1. `authSlice.login()` / `signup()` / `restoreSession()` calls `wsClient.connect(userId, token)`
2. `wsClient.setTokenRefresher()` — socket re-fetches token on every reconnect
3. `authSlice.logout()` calls `wsClient.disconnect()`

**Reconnection config:**
- `reconnectionDelay: 1000ms` / `reconnectionDelayMax: 30000ms`
- `reconnectionAttempts: Infinity`
- `timeout: 20000ms`
- Tab visibility handler: forces reconnect if `visibilityState === 'visible'` and socket is inactive

**Deduplication:** `SocketIOClient` maintains a LRU set of 200 `notification_id` values. On reconnect, backend replays durable events; duplicate IDs are ACK'd and silently dropped.

| `display_type` | Purpose |
|----------------|---------|
| `assistant_message` | Final agent response |
| `user_message` | Confirmed user message echo |
| `cancelled_message` | Task cancelled |
| `paused_message` | Task paused |
| `error_message` | Agent error |
| `ask_human_message` | Agent asking user for input |
| `status_update` | Transient task status (started/thinking/planning/executing/completed) |
| `step_progress` | CUA action step |

| `mode_progress` | Deep research / MCP / SWE tool execution progress |

**Handler:** `wsClient.handleAgentResponse()`

**Payload:** `{ session_id, is_typing }`

### `device_status_update`

**Action:** `deviceSlice.updateDeviceStatus()` — if device unknown, re-fetches device list

### `agent_error`

**Payload:** `{ session_id, content?, error? }`

### `confirmation_request`

**Payload:** `{ notification_id?, confirmation: ConfirmationRequest }`

**Payload:** `{ status, timestamp, cloud_pc_id?, stream_url?, stream_auth_key?, sandbox_id?, reason? }`

**Durability:** ACK'd

**Action:** Triggers `billingSlice.loadStatus()` — re-fetches billing state after Stripe webhooks

**Durability:** ACK'd. Triggers MCP integration UI update.

## HTTP Client Architecture

**Base client:** `httpClient` (`api/http/httpClient.ts`):
- Auth header injection via `getPlatform().storage.get('auth_token')`
- `credentials: 'include'` on all requests (cookie support)
- `HttpClientError` thrown on non-2xx with `{ status, statusText, data }`
- `extractErrorMessage()` handles string, array (Pydantic validation), nested object forms

**Raw fetch usage:** `authApi`, `sessionApi.sendMessage`, and `sessionApi.getMessages/getTaskStatus/getPlan` bypass `httpClient` for:
1. Non-JSON content type (form auth)
2. Status-branching on 402/403/409/426 (sendMessage)
3. Abort signal / timeout (fetchWithTimeout wrapper, 30s default)

**Mock mode:** `USE_MOCKS = false` in `httpClient.ts`, `authApi.ts`, `billingApi.ts` — flip to `true` for offline development.

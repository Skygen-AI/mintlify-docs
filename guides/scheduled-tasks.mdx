---
title: "Scheduled Tasks"
description: "Create recurring agent tasks that run automatically on any schedule — hourly, daily, weekly, or one-time."
---

Scheduled tasks let you run an agent prompt on a recurring schedule without manual intervention. The scheduler evaluates all enabled tasks every minute and dispatches any that are due to the agent via RabbitMQ, exactly like a manual message.

## Frequency Options

| Frequency | Description | Required fields |
|-----------|-------------|-----------------|
| `once` | Runs once at a specific datetime | `scheduled_at` |
| `hourly` | Runs every hour at the specified minute | `time_of_day` (HH:MM — minute used) |
| `daily` | Runs every day at a specific time | `time_of_day` |
| `weekly` | Runs once a week | `time_of_day`, `day_of_week` (0=Mon, 6=Sun) |
| `monthly` | Runs once a month | `time_of_day`, `day_of_month` (1–31) |

All times are interpreted in the `timezone` you specify (default: UTC).

## Creating a Scheduled Task

<CodeGroup>
```bash curl (daily task)
curl -s -X POST https://api.skygen.ai/scheduled-tasks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Morning briefing",
    "prompt": "Check my Gmail inbox and Slack messages. Summarize anything important from the last 24 hours.",
    "frequency": "daily",
    "time_of_day": "08:00",
    "timezone": "America/New_York"
  }'
```

```bash curl (weekly task)
curl -s -X POST https://api.skygen.ai/scheduled-tasks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Weekly report",
    "prompt": "Pull this week'\''s GitHub PRs merged to main. Write a brief changelog.",
    "frequency": "weekly",
    "time_of_day": "09:00",
    "day_of_week": 0,
    "timezone": "UTC"
  }'
```

```bash curl (one-time task)
curl -s -X POST https://api.skygen.ai/scheduled-tasks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Conference prep",
    "prompt": "Research speakers at PyCon 2026 and prepare a schedule.",
    "frequency": "once",
    "time_of_day": "10:00",
    "timezone": "UTC",
    "scheduled_at": "2026-06-01T10:00:00Z"
  }'
```

```python Python
import httpx

BASE_URL = "https://api.skygen.ai"
headers = {"Authorization": f"Bearer {TOKEN}"}

resp = httpx.post(f"{BASE_URL}/scheduled-tasks", headers=headers, json={
    "name": "Morning briefing",
    "prompt": "Check my Gmail and Slack. Summarize anything important from last 24 hours.",
    "frequency": "daily",
    "time_of_day": "08:00",
    "timezone": "America/New_York",
})
task = resp.json()
print("Task ID:", task["id"])
print("Next run:", task["next_run_at"])
```
</CodeGroup>

**Full request schema:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Human-readable name |
| `prompt` | string | Yes | The instruction sent to the agent |
| `frequency` | string | Yes | `once`, `hourly`, `daily`, `weekly`, `monthly` |
| `time_of_day` | string | Yes | `HH:MM` in 24-hour format |
| `timezone` | string | Yes | IANA timezone name (e.g. `America/New_York`) |
| `day_of_week` | int | For weekly | 0=Monday, 6=Sunday |
| `day_of_month` | int | For monthly | 1–31 |
| `scheduled_at` | datetime | For once | ISO 8601 UTC datetime |
| `device_id` | UUID | No | Attach to a specific device for GUI tasks |
| `computer_use` | bool | No | Enable GUI/screenshot capabilities |
| `connectors` | array | No | MCP connector links (see below) |
| `expires_at` | datetime | No | Stop running after this date |
| `max_concurrent_runs` | int | No | Default 1 — prevent overlapping runs |

**Response:**
```json
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "name": "Morning briefing",
  "frequency": "daily",
  "time_of_day": "08:00",
  "timezone": "America/New_York",
  "is_enabled": true,
  "next_run_at": "2026-05-09T12:00:00Z",
  "last_run_at": null,
  "run_count": 0,
  "created_at": "2026-05-08T10:00:00Z"
}
```

## Attaching MCP Connectors

Link connectors that the agent should use during execution:

```json
{
  "name": "Slack digest",
  "prompt": "Summarize the #general channel from today.",
  "frequency": "daily",
  "time_of_day": "18:00",
  "timezone": "UTC",
  "connectors": [
    {
      "connector_type": "slack",
      "connector_id": "<your-slack-connector-id>"
    }
  ]
}
```

## Listing Tasks

```bash
# All tasks
curl https://api.skygen.ai/scheduled-tasks \
  -H "Authorization: Bearer $TOKEN"

# Only enabled tasks
curl "https://api.skygen.ai/scheduled-tasks?is_enabled=true" \
  -H "Authorization: Bearer $TOKEN"
```

Supports `limit` (max 100) and `offset` for pagination.

## Enabling / Disabling a Task

Toggle a task without deleting it:

```bash
curl -X PATCH "https://api.skygen.ai/scheduled-tasks/$TASK_ID/toggle" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "is_enabled": false }'
```

When disabled, `next_run_at` is preserved for display purposes. The scheduler skips disabled tasks.

## Updating a Task

Update any field including schedule and prompt:

```bash
curl -X PATCH "https://api.skygen.ai/scheduled-tasks/$TASK_ID" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "time_of_day": "09:30",
    "timezone": "Europe/London"
  }'
```

When schedule fields change, `next_run_at` is automatically recalculated.

## Running a Task Manually

Trigger an immediate run outside the normal schedule:

```bash
curl -s -X POST "https://api.skygen.ai/scheduled-tasks/$TASK_ID/run" \
  -H "Authorization: Bearer $TOKEN"
```

Returns a `ScheduledTaskRun` object immediately. The agent executes asynchronously.

<Note>
  Manual runs respect `max_concurrent_runs`. If a previous run is still in progress, you'll receive HTTP 409 `Previous run still in progress`.
</Note>

## Viewing Execution History

### All runs for a specific task

```bash
curl "https://api.skygen.ai/scheduled-tasks/$TASK_ID/runs?limit=20" \
  -H "Authorization: Bearer $TOKEN"
```

### All completed runs across all tasks

```bash
curl "https://api.skygen.ai/scheduled-tasks/runs/completed?limit=50" \
  -H "Authorization: Bearer $TOKEN"
```

**Run object fields:**

| Field | Description |
|-------|-------------|
| `id` | Run UUID |
| `task_id` | Parent task UUID |
| `status` | `pending`, `running`, `completed`, `error` |
| `started_at` | When the agent began processing |
| `completed_at` | When the agent finished |
| `error_message` | Error details if status is `error` |
| `session_id` | The chat session created for this run |

## Extracting a Playbook from a Session

After completing a successful manual session, you can extract a reusable automation playbook:

```bash
curl -s -X POST https://api.skygen.ai/scheduled-tasks/extract-playbook \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "session_id": "<completed-session-id>" }'
```

The Agent analyzes the session trajectory and returns:

```json
{
  "command_name": "morning_briefing",
  "description": "Checks Gmail and Slack for important messages",
  "variables": ["days_back", "channels"],
  "tasks": [...]
}
```

Use this as the basis for a new scheduled task prompt.

## Deleting a Task

Soft-deletes the task (and its future runs). Historical run data is preserved.

```bash
curl -X DELETE "https://api.skygen.ai/scheduled-tasks/$TASK_ID" \
  -H "Authorization: Bearer $TOKEN"
```

Returns HTTP 204 on success.

## Real-Time Events

Scheduled task lifecycle events are broadcast over Socket.IO to all connected clients of the user:

| Event | Trigger |
|-------|---------|
| `scheduled_task_created` | Task created via API |
| `scheduled_task_updated` | Task updated or toggled |
| `scheduled_task_deleted` | Task deleted |

Listen on the `/user` namespace after authenticating with your user ID.

## Error Reference

| HTTP Status | Detail | Cause |
|------------|--------|-------|
| 404 | `Scheduled task not found` | Task ID does not exist or belongs to another user |
| 404 | `Device not found` | `device_id` not found or belongs to another user |
| 409 | `Previous run still in progress` | `max_concurrent_runs` limit reached |
| 422 | `scheduled_at must be in the future` | One-time task with past datetime |
| 422 | `Invalid time_of_day format` | Expected `HH:MM` |
| 422 | `Task could not be executed` | Agent dispatch failed |
| 503 | `Agent service unavailable` | Playbook extraction: agent unreachable |

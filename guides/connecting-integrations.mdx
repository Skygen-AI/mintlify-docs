## Overview

Skygen integrates with third-party services through MCP (Model Context Protocol) toolkits. Most integrations are backed by Composio, which handles OAuth flows, credential storage, and tool discovery. Telegram is a custom provider using MTProto (not supported by Composio). Once connected, the agent can use the toolkit's tools automatically during task execution.

## Connecting a toolkit (OAuth)

<Steps>
  <Step title="Open the Connectors panel">
    In the chat interface, click the grid icon in the toolbar to open the Connectors panel. This lists all available toolkits and their connection status.
  </Step>
  <Step title="Click Connect">
    Find the toolkit you want to connect (e.g., Gmail) and click its Connect button. A signed OAuth callback URL is generated and you are redirected to the provider's authorization page.
  </Step>
  <Step title="Authorize access">
    Log in to your account at the provider and approve the requested permissions. Skygen requests minimal scopes necessary for the toolkit's tools to work.
  </Step>
  <Step title="Return to Skygen">
    After authorization, the provider redirects to `https://app.skygen.ai/mcp/oauth/callback` with a signed state token. The backend validates the state, stores the connection, and emits a `mcp_connection_completed` Socket.IO event to your browser.
  </Step>
  <Step title="Verify connection">
    The toolkit badge in the Connectors panel turns green. The agent can now use this toolkit in future tasks.
  </Step>
</Steps>

## OAuth flow internals

The `GET /mcp/integrations/connect` endpoint generates the provider authorization URL:

```http
GET /mcp/integrations/connect?toolkit=gmail&platform=web&return_to=/chat
Authorization: Bearer {token}
```

The `return_to` parameter (max 512 chars, same-origin paths only) controls where the user lands after authorization. It is validated server-side to prevent open redirect attacks.

A signed state token is embedded in the callback URL using HMAC to prevent CSRF. The callback endpoint `GET /mcp/oauth/callback` verifies the signature before processing.

After the connection is stored, the backend calls `WakeUserSessions()` to resume any agents that were hibernated waiting for this connection.

## Managing connections

### List connected integrations

```http
GET /mcp/integrations/available
Authorization: Bearer {token}
```

Returns all available toolkits with their connection status, number of accounts connected, and whether each connection is active.

### Disconnect a toolkit

```http
DELETE /mcp/integrations/{toolkit}
Authorization: Bearer {token}
```

Removes the connection from Composio and marks it inactive in Skygen.

### Toggle a connection active/inactive

```http
PATCH /mcp/integrations/{connection_id}/active
Authorization: Bearer {token}

{"is_active": false}
```

Inactive connections are not offered to the agent during task execution.

### Label a connection

```http
PATCH /mcp/integrations/{connection_id}/label
Authorization: Bearer {token}

{"label": "Work Gmail"}
```

Labels help distinguish multiple accounts for the same toolkit (e.g., personal vs. work Gmail).

## Multi-account support

Some toolkits (Gmail, Google Calendar) support multiple connected accounts. Each account is a separate connection record. When the agent needs to use a toolkit with multiple accounts, it will ask the user which account to use via an `ask_human` prompt.

## Connecting Telegram

Telegram uses MTProto (not OAuth) because Composio does not support it. The connection flow:

<Steps>
  <Step title="Open Telegram settings">
    In the Connectors panel, click Connect next to Telegram.
  </Step>
  <Step title="Enter your phone number">
    The backend creates a Telegram client session via `app/integrations/mcp/telegram/` and sends an OTP to your phone.
  </Step>
  <Step title="Enter the OTP">
    Input the code received from Telegram in the prompt. The session is encrypted and stored using the `MCP_CRYPTO_KEY` AES key.
  </Step>
  <Step title="Verify">
    The toolkit becomes available. The agent can now read and send Telegram messages on your behalf.
  </Step>
</Steps>

## Connection state reconciliation

Sometimes the Composio OAuth redirect lands on the app without the toolkit parameter (e.g., Composio redirects to `app.skygen.ai/` instead of the callback URL). The backend reconciles this automatically:

When `GET /mcp/integrations/available` or `GET /mcp/triggers` is called by the frontend after an OAuth return, the backend calls `reconcile_pending_connections()` which polls Composio for any recently-completed connections for the current user and triggers the wake flow for hibernating sessions.

Redis `SETNX` deduplication prevents multiple reconcile calls from waking the same session multiple times.

## Troubleshooting

**Agent says "connect Gmail" but I already connected it**  
The connection may have been revoked on the provider side. Open the Connectors panel, disconnect Gmail, and reconnect.

**OAuth redirect loop**  
Clear cookies and try again. The signed state token expires after the OAuth flow completes — reusing an old link will fail.

**Telegram OTP not received**  
Ensure your phone number includes the country code (e.g., `+14155552671`). OTPs expire after 2 minutes.

**Connection shows as inactive**  
A Composio webhook notified Skygen that the connection was revoked. Reconnect to refresh the token.

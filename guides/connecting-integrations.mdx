title: "Connecting integrations"
description: "Give agents access to Gmail, Slack, GitHub, Notion, and 500+ other services via OAuth."
---

Integrations let the agent act on your behalf in third-party apps. Connection is a one-time OAuth flow per account, orchestrated by Composio. Skygen stores only your intent (`is_active`, `account_label`); the tokens and refresh cycle live with Composio.

## Browse the catalog

```bash
curl "https://api.skygen.ai/mcp/toolkits?category=email&per_page=20" \
  -H "Authorization: Bearer $TOKEN"
```

Filter by `q` (search), `category` (`email`, `crm`, `productivity`, `developer-tools`, ...), paginated with `page` + `per_page` (max 500). Response includes per-toolkit `connected_accounts` array so you can see what the current user already has.

## Connect a new account

<Steps>
  <Step title="Request OAuth URL">
    ```bash
    curl https://api.skygen.ai/mcp/integrations/connect-url/gmail \
      -H "Authorization: Bearer $TOKEN"
    # → { "redirect_url": "https://composio.dev/authorize?...", "toolkit": "gmail" }
    ```
    No database row is created yet. The URL expires in about 10 minutes.
  </Step>
  <Step title="Open the URL in a popup">
    Your frontend opens `redirect_url` in a new window. The user consents in Google's (or GitHub's, Slack's, etc.) screen.
  </Step>
  <Step title="Composio posts the webhook">
    Behind the scenes Composio calls `POST /mcp/webhooks/composio`. Skygen verifies the HMAC signature (`COMPOSIO_WEBHOOK_SECRET`), creates a `ComposioConnection` with `status=connected`, `is_active=true`, and emits a Socket.IO event so your UI can update.
  </Step>
  <Step title="Verify">
    ```bash
    curl https://api.skygen.ai/mcp/integrations/available \
      -H "Authorization: Bearer $TOKEN"
    ```
    The new account appears under `composio[].connected_accounts[]`.
  </Step>
</Steps>

## Multiple accounts for one toolkit

Connect a second Gmail (personal and work):

```bash
curl "https://api.skygen.ai/mcp/integrations/connect-url/gmail?add_another=true" \
  -H "Authorization: Bearer $TOKEN"
```

Without `add_another=true`, Composio's SDK rejects a second OAuth request for an already-connected toolkit. Both accounts appear in `connected_accounts[]` with distinct `account_id`s.

## Rename an account

```bash
curl -X PATCH https://api.skygen.ai/mcp/integrations/connections/$ACCOUNT_ID/label \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"label": "Work Gmail"}'
```

`account_label` takes priority over Composio's `email`/`display_name` in list responses, so the agent sees "Work Gmail" instead of `alice@company.com`.

<Note>
**Lazy sync.** If the account was connected via `/connect-url` and does not yet have a DB row (rare race), the first `label` or `active` call creates one from Composio's live data. No manual sync needed.
</Note>

## Toggle temporarily off

Hide a connected account from the agent without revoking OAuth:

```bash
# One account
curl -X PATCH https://api.skygen.ai/mcp/integrations/gmail/active \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"is_active": false, "account_id": "ca_abc"}'

# All accounts for a toolkit
curl -X PATCH https://api.skygen.ai/mcp/integrations/gmail/active \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"is_active": false}'
```

Flip back to `true` to re-enable. The OAuth grant is preserved — no re-auth needed.

## Disconnect

```bash
curl -X POST https://api.skygen.ai/mcp/integrations/disconnect/gmail \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"account_id": "ca_abc"}'
```

Revokes the OAuth grant with the third-party service. Permanent. To reconnect, run the OAuth flow again.

## Telegram is different

Telegram uses MTProto, not OAuth. It has its own endpoints under `/mcp/telegram/*` — bot token + phone number + verification code flow instead of redirect. The rest (agent treats it as a tool, same session model) is identical.

## Gotchas

<Warning>
**Webhook is asynchronous.** After the user consents, your UI will not see the connection immediately. Listen for the Socket.IO event on `/user`, or poll `/mcp/integrations/available`. Composio usually posts within 1–3 seconds.
</Warning>

<Note>
**Environment prefix in entityId.** Composio's webhook may include an environment prefix in `entityId` (e.g. `prod_user_123`). The backend parser strips it.
</Note>

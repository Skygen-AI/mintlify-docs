## Overview

Skygen includes a built-in helpdesk system that allows users to submit support tickets directly from the app. Tickets support threaded messages, file attachments stored in S3, satisfaction ratings, and real-time notifications via Socket.IO. New tickets and replies trigger Telegram alerts to the support team.

---

## Creating a ticket

```http
POST /helpdesk/tickets
Authorization: Bearer {token}
Content-Type: application/json
```

```json
{
  "title": "Agent not responding after OAuth",
  "content": "After connecting Gmail, the agent shows 'waiting' indefinitely.",
  "priority": "high",
  "category": "bug"
}
```

<ParamField body="title" type="string" required>
  Short description of the issue.
</ParamField>

<ParamField body="content" type="string" required>
  Full description of the problem.
</ParamField>

<ParamField body="priority" type="string">
  `low`, `normal`, or `high`. Default: `normal`.
</ParamField>

<ParamField body="category" type="string">
  Issue category (e.g. `bug`, `feature_request`, `billing`, `general`).
</ParamField>

**Response — 201 Created:**

```json
{
  "id": "ticket-uuid",
  "ticket_number": 142,
  "title": "Agent not responding after OAuth",
  "status": "open",
  "priority": "high",
  "category": "bug",
  "created_at": "2026-05-08T10:00:00Z"
}
```

Rate limit: 10 ticket creations per user per hour.

## Listing tickets

```http
GET /helpdesk/tickets
Authorization: Bearer {token}
```

<ParamField query="status" type="string">
  Filter by status: `open`, `in_progress`, `resolved`, `closed`.
</ParamField>

<ParamField query="limit" type="integer">
  Default: 20. Max: 100.
</ParamField>

<ParamField query="offset" type="integer">
  Pagination offset. Default: 0.
</ParamField>

## Getting ticket details

```http
GET /helpdesk/tickets/{ticket_id}
Authorization: Bearer {token}
```

Returns the ticket with its full message thread and attachments.

---

## Replying to a ticket

```http
POST /helpdesk/tickets/{ticket_id}/messages
Authorization: Bearer {token}

{"content": "I've tried reconnecting but the issue persists."}
```

Rate limit: 30 replies per user per hour.

---

## Ticket status values

| Status | Description |
|--------|-------------|
| `open` | Newly submitted, awaiting response |
| `in_progress` | Support team is working on it |
| `resolved` | Issue addressed; awaiting user confirmation |
| `closed` | Ticket closed (auto or manual) |

---

## Updating a ticket

```http
PATCH /helpdesk/tickets/{ticket_id}
Authorization: Bearer {token}

{"status": "closed"}
```

Users can close their own tickets. Support staff can change any status via the admin API.

---

## Satisfaction rating

After a ticket is resolved, users can rate the support experience:

```http
POST /helpdesk/tickets/{ticket_id}/rate
Authorization: Bearer {token}

{"rating": 5, "comment": "Fast and helpful!"}
```

Ratings are stored and used in aggregate reporting visible to admins.

## File attachments

Attachments use a presigned S3 upload flow:

**Step 1: Get presigned URL**

```http
POST /helpdesk/tickets/{ticket_id}/attachments/presigned
Authorization: Bearer {token}

{"filename": "screenshot.png", "content_type": "image/png"}
```

**Step 2: Upload to S3 directly**

```python
import requests

resp = requests.post(
    presigned["upload_url"],
    data=presigned["fields"],
    files={"file": open("screenshot.png", "rb")},
)
```

**Step 3: Confirm the attachment**

```http
POST /helpdesk/tickets/{ticket_id}/attachments/confirm
Authorization: Bearer {token}

{"attachment_key": "helpdesk/{ticket_id}/attachments/{uuid}.png"}
```

Rate limit: 20 attachment uploads per user per hour.

## Knowledge base

The helpdesk includes a searchable knowledge base with articles that can resolve common issues without opening a ticket.

### List articles

```http
GET /helpdesk/kb
Authorization: Bearer {token}
```

### Search articles

```http
GET /helpdesk/kb?q=oauth+callback
Authorization: Bearer {token}
```

### Submit article feedback

```http
POST /helpdesk/kb/{article_id}/feedback
Authorization: Bearer {token}

{"helpful": true}
```

## Team notifications

New tickets and replies send Telegram alerts to the configured support chat. The alert format:

```
📩 New Ticket #142
Agent not responding after OAuth
[high] [bug]
After connecting Gmail, the agent shows...
```

Configure via env vars:
- `RATING_ALERT_TELEGRAM_BOT_TOKEN` — Telegram bot token
- `HELPDESK_TELEGRAM_CHAT_ID` — Target chat ID for alerts

If these are not set, alerts are silently skipped.

## Admin endpoints

Admins can manage all tickets at `GET /admin/api/helpdesk/tickets` with full filtering by user, status, and date range. See [Admin Panel](/admin/overview) for details.

title: "Knowledge Base"
description: "Give your agent persistent memory. Store facts, preferences, and context that the agent retrieves and applies across sessions."
---

The Skygen Knowledge Base (KB) is a vector-indexed store of insights that the agent reads from and writes to during task execution. Unlike conversation history (which is per-session), KB entries persist across all sessions for a user and can be shared globally across all users.

<Note>
  The Knowledge Base requires `KB_ENABLED=true` and `GEMINI_API_KEY` to be set in the backend configuration. Gemini is used for embedding generation.
</Note>

## How It Works

1. **During execution** — the agent calls `POST /knowledge/retrieve` with a query string to fetch the top-N relevant insights using vector similarity.
2. **After learning something new** — the agent calls `POST /knowledge/save` to persist a fact or preference.
3. **Updating stale information** — the agent calls `POST /knowledge/update/{insight_id}`. If the entry is a global (system) insight, a user-scoped copy-on-write is created rather than modifying the original.

The agent handles KB read/write automatically as part of its reasoning loop. The endpoints described here are primarily used by the agent service (authenticated with `AGENT_API_TOKEN`) rather than by end users directly.

## Insight Structure

Each KB entry is an **insight** with the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `insight_id` | string | Unique identifier |
| `agent_type` | string | Agent that owns this insight (e.g. `broker`, `cua`) |
| `insight_type` | string | Category within the agent (e.g. `user_preference`, `task_result`) |
| `global_insight_type` | string | Cross-agent category (e.g. `user_data`, `domain_knowledge`) |
| `user_id` | string | Owning user UUID, or `global` for system-wide entries |
| `content` | string | The text of the insight |
| `metadata` | object | Optional structured metadata |
| `timestamp` | string | ISO 8601 timestamp |
| `is_user_visible` | bool | Whether to show this entry in user-facing KB views |

## API Reference

### Save an Insight

<Warning>
  This endpoint requires `AGENT_API_TOKEN` authentication (Bearer token). It is not accessible with a regular user JWT.
</Warning>

```bash
POST /knowledge/save
Authorization: Bearer <AGENT_API_TOKEN>
Content-Type: application/json

{
  "agent_type": "broker",
  "insight_type": "user_preference",
  "global_insight_type": "user_data",
  "user_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "content": "User prefers concise summaries in bullet-point format",
  "metadata": { "source": "session_123" },
  "is_user_visible": true
}
```

**Response:**
```json
{
  "insight_id": "kb_abc123",
  "success": true,
  "message": "Knowledge saved successfully"
}
```

### Retrieve Relevant Insights

```bash
POST /knowledge/retrieve
Authorization: Bearer <AGENT_API_TOKEN>
Content-Type: application/json

{
  "query": "user communication preferences",
  "agent_type": "broker",
  "insight_type": "user_preference",
  "user_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "num_insights": 5
}
```

**Response:**
```json
{
  "insights": [
    {
      "insight_id": "kb_abc123",
      "content": "User prefers concise summaries in bullet-point format",
      "agent_type": "broker",
      "insight_type": "user_preference",
      "global_insight_type": "user_data",
      "user_id": "...",
      "timestamp": "2026-05-08T10:00:00Z",
      "is_user_visible": true
    }
  ],
  "error": null
}
```

### Update an Insight

```bash
POST /knowledge/update/{insight_id}
Authorization: Bearer <AGENT_API_TOKEN>
Content-Type: application/json

{
  "content": "User prefers concise summaries in numbered lists",
  "user_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6"
}
```

If the insight is a global entry, this operation creates a user-scoped copy (copy-on-write) and returns the new `insight_id`:

```json
{
  "success": true,
  "message": "Copy-on-Write: user override created",
  "insight_id": "kb_new456"
}
```

### List User Insights

```bash
GET /knowledge/insights/{user_id}
Authorization: Bearer $TOKEN
```

Returns all `is_user_visible=true` insights for the user. This endpoint uses regular user JWT auth and is suitable for displaying the KB contents in the UI.

### Save a Session Summary

After a session ends, the agent can save a compressed summary to avoid context bloat in future sessions:

```bash
POST /knowledge/summary
Authorization: Bearer <AGENT_API_TOKEN>
Content-Type: application/json

{
  "user_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "session_id": "session-uuid",
  "summary_data": {
    "summary": "User asked agent to reorganize their Google Drive. Completed successfully.",
    "keywords": ["google drive", "file organization"],
    "original_range": [1, 47]
  }
}
```

## Global vs. User Insights

| Scope | `user_id` value | Writeable by | Description |
|-------|----------------|-------------|-------------|
| Global | `global` | Agent service only | Platform-wide knowledge (e.g. app documentation, common tasks) |
| User-scoped | UUID | Agent service | Per-user facts, preferences, task history |
| User override | UUID (copy) | Agent service | Copy-on-write override of a global entry for a specific user |

## Disabling the Knowledge Base

Set `KB_ENABLED=false` in the backend environment to disable all KB endpoints and skip KB reads during agent execution. Useful for privacy-sensitive deployments where persistent memory is not desired.

## Cost Considerations

Each KB save triggers a Gemini embedding call to index the content. Retrieve operations use cosine similarity against stored embeddings and do not incur LLM API costs. Embeddings consume your `GEMINI_API_KEY` quota, not Skygen tokens.

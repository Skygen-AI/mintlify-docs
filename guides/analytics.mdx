Skygen has a dual-layer analytics system: **server-side automatic tracking** for business-critical events (registrations, tasks, billing) and a **client-side event API** for app lifecycle events (installs, launches, crashes).

Events are forwarded to your configured analytics destinations: Google Analytics 4, Yandex Metrica, and/or Amplitude.

<Note>
  Analytics requires `ANALYTICS_ENABLED=true` in the backend configuration. Individual destinations are configured with their respective API keys (`GA4_MEASUREMENT_ID`, `AMPLITUDE_API_KEY`, etc.).
</Note>

## Automatically Tracked Events

These events are emitted by the backend automatically — you do not need to send them from your client:

| Event | Trigger |
|-------|---------|
| `registration_completed` | User registers (email/password or Google OAuth) |
| `first_task_completed` | User's first agent task finishes |
| `device_installed` | `POST /devices` called — includes platform and device type |
| `daily_active_user` | First API activity after 20+ hours since last DAU event |
| `monthly_active_user` | First API activity after 25+ days since last MAU event |

## Client-Side Event API

Send analytics events from your app client:

```bash
POST /analytics/events
Authorization: Bearer $TOKEN
Content-Type: application/json

{
  "event_name": "app_launched",
  "client_id": "<ga4-client-id>",
  "ga_session_id": "<ga4-session-id>",
  "properties": {
    "platform": "macos",
    "app_version": "2.1.0"
  }
}
```

**Allowed `event_name` values:**

| Event | Description |
|-------|-------------|
| `app_installed` | User downloaded and installed the application |
| `app_launched` | User launched the application |
| `app_load_time` | App startup performance measurement |
| `app_crash` | Application crash |
| `app_uninstalled` | User uninstalled the application |
| `session_started` | User started a new chat session |
| `session_ended` | User ended a chat session |
| `daily_active_user` | DAU (tracked once per 20-hour window) |
| `monthly_active_user` | MAU (tracked once per 25-day window) |
| `device_installed` | Device registered (also auto-tracked) |

Sending an unlisted event name returns HTTP 400.

**Response:**
```json
{
  "success": true,
  "message": "Event tracked successfully"
}
```

## Specialized Event Endpoints

### App Crash

```bash
POST /analytics/app-crash
Authorization: Bearer $TOKEN
Content-Type: application/json

{
  "error_message": "Segmentation fault in renderer",
  "error_type": "SIGSEGV",
  "stack_trace": "...",
  "app_version": "2.1.0",
  "platform": "macos"
}
```

### App Load Time

```bash
POST /analytics/app-load-time
Authorization: Bearer $TOKEN
Content-Type: application/json

{
  "load_time_ms": 1240,
  "startup_type": "cold"
}
```

`startup_type`: `cold` (full restart), `warm` (from sleep), or `hot` (from background).

### Session Events

```bash
POST /analytics/session-started
Authorization: Bearer $TOKEN
Content-Type: application/json

{
  "session_id": "3fa85f64-...",
  "device_id": "optional-device-uuid"
}
```

```bash
POST /analytics/session-ended
Authorization: Bearer $TOKEN
Content-Type: application/json

{
  "session_id": "3fa85f64-...",
  "duration_seconds": 145,
  "messages_count": 12
}
```

### App Uninstalled

```bash
POST /analytics/app-uninstalled
Authorization: Bearer $TOKEN
Content-Type: application/json

{
  "reason": "Switching to a competitor"
}
```

`reason` is optional and forwarded as a property to your analytics backends.

## Session Activity Tracking

Every time an authenticated user calls the events endpoint, their `last_active_at` timestamp is updated. DAU and MAU events are de-duplicated automatically:

- **DAU**: Tracked at most once per 20 hours per user.
- **MAU**: Tracked at most once per 25 days per user.

These events are used to generate Daily Active Users and Monthly Active Users metrics in your analytics dashboards.

## GA4 Session Stitching

If you use Google Analytics 4, pass the `client_id` (from the `_ga` cookie) and `ga_session_id` (from the `_ga_<measurement_id>` cookie) with your events. This stitches server-side events to the user's browser session in GA4, giving you accurate attribution across web and app surfaces.

```javascript
// Read GA4 identifiers from cookies
function getGA4Ids() {
  const gaCookie = document.cookie.split(";").find(c => c.trim().startsWith("_ga="));
  if (!gaCookie) return {};
  const clientId = gaCookie.split("=")[1].split(".").slice(2).join(".");
  return { client_id: clientId };
}

await fetch("https://api.skygen.ai/analytics/events", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    event_name: "app_launched",
    ...getGA4Ids(),
    properties: { platform: "web" },
  }),
});
```

## Admin Analytics Endpoints

These endpoints are restricted to admin users and provide platform-level aggregates:

```bash
# User statistics
GET /users/stats?days=30
Authorization: Bearer $ADMIN_TOKEN

# Returns:
{
  "total_users": 1240,
  "active_users": 987,
  "verified_users": 1100,
  "new_users_last_days": 43,
  "days": 30
}
```

## Configuration Reference

| Environment variable | Description |
|---------------------|-------------|
| `ANALYTICS_ENABLED` | Master switch. Set to `false` to disable all tracking. |
| `GA4_MEASUREMENT_ID` | Google Analytics 4 measurement ID (e.g. `G-XXXXXXXXXX`) |
| `GA4_API_SECRET` | GA4 Measurement Protocol API secret |
| `GA4_DEBUG_MODE` | Set to `true` to use GA4 debug endpoint and log all events |
| `AMPLITUDE_API_KEY` | Amplitude project API key |
| `YANDEX_COUNTER_ID` | Yandex Metrica counter ID |

All keys are optional. Events are sent only to destinations with a configured key.

---
title: "Custom connectors"
description: "Turn any HTTP API or MCP server into a first-class agent tool with encrypted secrets."
---

Custom connectors let you plug your internal services into Skygen without waiting for a Composio integration. Point at an HTTP endpoint or an MCP server, define a few tools, and the agent picks them up alongside the 500+ Composio toolkits.

## Types

Three connector types, picked at creation:

| Type | What it is | Tool definition |
|------|------------|-----------------|
| `custom_api` | Generic HTTP tool. Agent specifies method/url/headers/body at call time. Secrets substituted via `{{NAME}}` templates. | Auto-generated from the config |
| `custom_mcp` | MCP server (HTTP or stdio). Tools discovered at runtime by probing the server on create. | Discovered, not declared |
| `app` | Legacy structured tools with fixed schema | Hand-written per tool |

## Schema

```python
class ConnectorBase(BaseModel):
    name: str
    slug: str                          # Normalized to lowercase-dash
    type: ConnectorTypeEnum            # "app" | "custom_api" | "custom_mcp"
    description: Optional[str]
    icon_url: Optional[str]            # S3 URL from presigned upload
    note: Optional[str]                # User instructions for the agent
    config: Dict[str, Any]             # Type-specific, encrypted at rest
    tools: List[ConnectorTool]
    status: ConnectorStatusEnum        # "draft" | "connected" | "error"
```

Each tool:

```python
class ConnectorTool(BaseModel):
    name: str
    description: Optional[str]
    input_schema: Dict                 # JSON Schema for parameters
    action: Dict                       # Execution config (HTTP endpoint, etc.)
```

## Encrypted secrets

Anything under `config.secrets` is encrypted with `MCP_CRYPTO_KEY` (Fernet, 32-byte base64) before persistence. On read the backend decrypts and redacts secrets to `***` before returning responses. Plaintext secrets never leave backend memory.

At tool execution, `{{SECRET_NAME}}` placeholders in headers, URLs, and bodies are replaced with the decrypted values.

## Create a `custom_api` connector

```bash
curl -X POST https://api.skygen.ai/connectors/custom \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Internal Orders",
    "slug": "internal-orders",
    "type": "custom_api",
    "description": "Orders microservice",
    "config": {
      "server_url": "https://orders.internal",
      "timeout": 30,
      "secrets": {
        "API_KEY": "sk-abc123"
      }
    },
    "tools": []
  }'
```

The agent now sees one tool, `custom_api__<connector_id>`, with this schema:

```json
{
  "type": "object",
  "properties": {
    "method": { "type": "string", "enum": ["GET","POST","PUT","DELETE","PATCH"] },
    "url":    { "type": "string", "description": "Full URL to call" },
    "headers":{ "type": "object", "description": "Use {{SECRET_NAME}} for secrets" },
    "body":   { "type": "object", "description": "For POST/PUT/PATCH" }
  },
  "required": ["method", "url"]
}
```

At call time the agent supplies something like:

```json
{
  "method": "GET",
  "url": "https://orders.internal/orders/order_001",
  "headers": {"Authorization": "Bearer {{API_KEY}}"}
}
```

Skygen decrypts `API_KEY`, substitutes into the header, and makes the request.

## Create a `custom_mcp` connector

```bash
curl -X POST https://api.skygen.ai/connectors/custom \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "name": "My MCP server",
    "slug": "my-mcp",
    "type": "custom_mcp",
    "config": {
      "server_url": "http://localhost:3000/mcp",
      "transport_type": "http",
      "headers": {"Authorization": "Bearer TOKEN"}
    },
    "tools": []
  }'
```

On create, Skygen probes the server (10-second synchronous timeout). If reachable, it discovers tools and sets `status=connected`. If the probe fails, `status=error` and `config.probe_error` holds the message.

## Icon upload

<Steps>
  <Step title="Request presigned POST">
    ```bash
    curl -X POST https://api.skygen.ai/connectors/icon/presigned \
      -H "Authorization: Bearer $TOKEN" \
      -d '{"content_type": "image/png", "file_extension": "png"}'
    # → { "upload_url": "...", "fields": {...}, "icon_key": "connector-icons/user_123/abc.png" }
    ```
    Max size 2 MB. Allowed: png, jpeg, svg+xml, webp. URL valid 300s.
  </Step>
  <Step title="POST multipart to S3">
    Client-side upload.
  </Step>
  <Step title="Attach to connector">
    ```bash
    curl -X PATCH https://api.skygen.ai/connectors/custom/$CONNECTOR_ID \
      -H "Authorization: Bearer $TOKEN" \
      -d '{"icon_url": "https://.../connector-icons/user_123/abc.png"}'
    ```
  </Step>
</Steps>

## Sensitive tools

Mark a tool with `requires_confirmation: true` in its `action` to force a [Confirmation](/concepts/confirmations) before the agent can call it. Useful for destructive endpoints like `cancel_order` or anything that writes to production.

## CRUD endpoints

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/connectors/custom` | Create. Probes MCP servers synchronously. |
| `GET` | `/connectors/custom` | List user's connectors. Config decrypted and redacted. |
| `PATCH` | `/connectors/custom/{id}` | Partial update. Config re-encrypted if modified. |
| `DELETE` | `/connectors/custom/{id}` | Delete. |
| `POST` | `/connectors/icon/presigned` | Presigned S3 URL for icon upload. |

## Gotchas

<Warning>
**MCP probe is synchronous.** Servers that take longer than 10 seconds to respond to the initial capability exchange fail to create. Either make your server faster to warm up, or retry the create after the server is ready.
</Warning>

<Note>
**Slug normalization.** Non-alphanumeric characters become `-`. `"My API"` becomes `"my-api"`. Provide your own slug if you care about the value.
</Note>

<Note>
**Never log secrets.** The redaction step applies to responses. If you log `config` from your own code, ensure you call `redact_config()` first.
</Note>

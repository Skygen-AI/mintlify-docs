## Overview

The Agent Library is a curated collection of prompt templates organized into categories (Finance, Research, Productivity, Marketing, etc.). Each card contains a title, subtitle, description, badge text, and a ready-to-use prompt. Users click a card to instantly launch an agent task with that prompt — no typing required.

The library is admin-managed (content curated by the Skygen team) and displayed prominently on the home page and the dedicated Library page.

---

## Data model

### LibraryCategory

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Category identifier |
| `slug` | string | URL-friendly slug (e.g. `productivity`) |
| `name` | string | Display name (e.g. `Productivity`) |
| `description` | string | Optional description |
| `sort_order` | integer | Display ordering |
| `is_active` | boolean | Visibility toggle |

### LibraryCard

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Card identifier |
| `category_id` | UUID | Parent category |
| `title` | string | Card title (max 200 chars) |
| `subtitle` | string | Subtitle for landing bullets |
| `description` | string | Short description shown on the card |
| `prompt` | string | Full agent prompt template |
| `image_base64` | string | Card image as data URI (`data:image/png;base64,...`) |
| `badge_text` | string | Optional badge (e.g. "Saved $3k/year on average") |
| `send_to_landing_main_screen` | boolean | Whether to feature on the home page |
| `sort_order` | integer | Display ordering within category |

## API reference

### List categories

```http
GET /library/types
```

No authentication required. Returns all active categories ordered by `sort_order`.

```json
{
  "items": [
    {"id": "...", "slug": "productivity", "name": "Productivity", "description": null},
    {"id": "...", "slug": "finance", "name": "Finance", "description": null}
  ],
  "total": 6
}
```

### Get cards by category

```http
GET /library/cards?type_slug=productivity
```

No authentication required.

<ParamField query="type_slug" type="string" required>
  Category slug returned from `/library/types`.
</ParamField>

```json
{
  "items": [
    {
      "id": "card-uuid",
      "title": "Daily Email Digest",
      "subtitle": "Summarize your inbox every morning",
      "description": "Reads unread emails and creates a structured digest",
      "prompt": "Read my unread emails from the last 24 hours and create a structured daily digest...",
      "image_base64": "data:image/png;base64,...",
      "badge_text": "Saves 30 min/day"
    }
  ],
  "total": 12
}
```

### Get all library cards (paginated)

```http
GET /library/cards/all?limit=50&offset=0
```

Returns all active cards regardless of category. Useful for search or bulk display.

## Using a library card

In the frontend, clicking a library card:

1. Pre-fills the chat input with the card's `prompt` value.
2. Optionally auto-submits the prompt to start the agent task immediately.
3. Creates a new chat session (or reuses the active one).

From the API:

```python
import httpx

# 1. Get card prompt
cards = httpx.get("https://api.skygen.ai/library/cards?type_slug=productivity").json()
card = cards["items"][0]

# 2. Create a session and send the prompt
session = httpx.post(
    "https://api.skygen.ai/chat/sessions",
    headers={"Authorization": f"Bearer {token}"},
    json={"title": card["title"]},
).json()

httpx.post(
    f"https://api.skygen.ai/chat/sessions/{session['id']}/messages",
    headers={"Authorization": f"Bearer {token}"},
    json={"content": card["prompt"]},
)
```

## Admin management

Admins manage library content at:

- `GET /admin/api/library/categories` — list all categories
- `POST /admin/api/library/categories` — create a category
- `PUT /admin/api/library/categories/{id}` — update a category
- `DELETE /admin/api/library/categories/{id}` — delete a category
- `GET /admin/api/library/cards` — list all cards
- `POST /admin/api/library/cards` — create a card
- `PUT /admin/api/library/cards/{id}` — update a card
- `DELETE /admin/api/library/cards/{id}` — delete a card

Cards with `send_to_landing_main_screen = true` appear in the home page quick-actions grid. Up to 6 featured cards are displayed.

---

## Landing page integration

The library homepage widget shows featured cards (those with `send_to_landing_main_screen = true`) grouped by the `quickActions` categories (`productivity`, `research`, `health`). These are rendered as suggestion chips that auto-fill the main task input on click.
